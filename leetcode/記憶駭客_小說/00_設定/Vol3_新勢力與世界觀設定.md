# 壹、崩潰後的城市狀態

---

## 1.1 概要

鸚鵡計畫 2.0 在 ch161 全面啟動後的 60 秒內，97.3% 的覆蓋率意味著新台北六百萬記憶節點中有 583 萬個被觸及。維倫搶救回 40%，剩餘約 300 萬人失去不同程度的記憶。城市從「被控制的穩定」瞬間跌入「無控制的混亂」——交通癱瘓、醫療系統超載、治安真空。三方勢力（記憶銀行、深淵監控局、記憶解放陣線）的控制系統同時崩潰，因為三方的基礎設施全部建立在邱韻如設計的同一套記憶網路架構上。織網者的網碎了，蜘蛛自己也掉了下來。

## 1.2 崩潰的技術機制

```
鸚鵡 2.0 的啟動序列：

  T+0s    啟動信號從地底第五層（周明哲舊實驗室）發出
  T+0.3s  信號透過記憶銀行核心伺服器向全城廣播
  T+1s    覆蓋波前沿以每秒三萬節點的速度擴散
  T+3s    維倫在米亞警報下開始搶救（Greedy + Heap）
  T+60s   覆蓋完成。六百萬節點中 583 萬被觸及
  T+60s   維倫搶救成功率 40%（約 233 萬節點保住部分記憶）
  T+61s   剩餘約 300 萬人進入「記憶空白」或「記憶碎片化」狀態

覆蓋類型：
  ▸ 完全覆蓋（~80 萬人）：所有記憶被鸚鵡 2.0 的「空白模板」覆蓋
    → 成為空白人。不知道自己是誰、在哪、在做什麼
  ▸ 部分覆蓋（~150 萬人）：核心身份記憶保留，但關係記憶、技能記憶碎片化
    → 知道自己是誰，但不認識家人、忘了怎麼開車/做飯
  ▸ 碎片化（~70 萬人）：記憶未被覆蓋但被覆蓋波的干擾弄碎
    → 記憶像打碎的鏡子，碎片都在但拼不回來
  ▸ 未受影響（~17 萬人）：覆蓋波未到達 + 維倫搶救成功
    → 包括下城區深處的部分居民（信號衰減）+ 阿嬤（已無記憶可覆蓋）
```

## 2.1 黑市記憶商人

### 概要

崩潰後的第 2-3 天，300 萬人的記憶碎片散落在記憶網路的各個節點。這些碎片像暴風後被吹落的屋瓦——沒有主人、沒有標記、沒有保護。黑市商人是最先反應過來的人：他們在深夢黑市的廢墟上建立了新的交易網路，收購散落的記憶碎片，按品質分級後高價出售給那些「想找回自己」的失憶者。

### 技術設定

```
記憶碎片散落機制：
  鸚鵡 2.0 的覆蓋波不是「刪除」——是「覆蓋」。
  被覆蓋的記憶不會消失，而是被推到記憶晶片的底層堆疊中。
  但部分碎片在覆蓋過程中被「擠出」晶片，
  進入記憶網路的公共緩衝區——像被洪水沖出家門的傢俱。

  黑市商人使用的工具：
  ▸ 碎片掃描器：改裝自記憶銀行的廢棄設備。精度低但速度快
  ▸ 情感標籤讀取器：讀取碎片的 emotion_tag 欄位，按情感類型定價
  ▸ 記憶拼接台：低階修復設備，只能將 2-3 個碎片粗略拼接
    （品質遠不及維倫的修復台，但足以讓失憶者「看到」碎片內容）

定價規則：
  ▸ 核心身份記憶碎片：30,000-100,000 新台幣/段（天價——因為它能讓你知道自己是誰）
  ▸ 技能記憶碎片：5,000-20,000/段（廚師想找回做菜的記憶、司機想找回開車的記憶）
  ▸ 情感記憶碎片：1,000-50,000/段（「被愛的感覺」依然最貴）
  ▸ 無法辨識的碎片（垃圾）：10-50/段（大量販售，碰運氣）
```

### 敘事功能

- 黑市記憶商人是 Vol 3 崩潰後的「新生態」代表——災難催生了新的經濟形態
- ch194-199 和 ch207-209 中，維倫必須進入黑市記憶池尋找阿嬤被提取的原始記憶
- 商人不是「反派」——他們是灰色地帶的生存者。有的趁火打劫，有的真的在幫人找回自己

## 2.2 空白人社區

### 概要

崩潰後第 3 天，原本在下城區遊蕩的空白人（Vol 1 設定約 5 萬人）和新增的約 80 萬完全失憶者開始自發聚集。他們佔據了廢棄的捷運新莊線隧道（原記憶解放陣線的秘密基地）。但這不是收容所——這是一個「選擇重新開始」的社區。部分失憶者在恢復零星記憶後，選擇不尋回過去的自己，而是接受空白，從零開始建立新的身份。

### 技術設定

```
空白人的神經狀態：
  完全空白人的記憶晶片仍在運作，但所有數據區都被鸚鵡 2.0 的空白模板覆蓋。
  然而：
  ▸ 身體記憶不受影響——肌肉的習慣、走路的方式、對某些味道的反應仍然存在
  ▸ 記憶晶片的 dependency 欄位（記憶關聯網路）被清空
    → 他們無法建立新記憶之間的「關聯」，每段記憶都是孤立的
    → 這意味著他們的新生活像一串珍珠沒有線——每天都是全新的開始
  ▸ 但：時間推移後，新記憶的 dependency 欄位開始自然重建
    → 大腦的物理結構會嘗試修復被清空的關聯
    → 速度極慢（每天約 0.1% 的新連結形成率）
    → 這給了社區成員一種「慢慢重新成為一個人」的過程
```

### 敘事功能

- 空白人社區是 Vol 3 對「記憶 = 身份」核心命題的反面論證
- 他們證明了：失去所有記憶的人不一定是「受害者」。有些人選擇擁抱空白
- 與語青的「遞迴殘留」形成對照：語青擁有記憶但不確定自己存在；他們沒有記憶但確定自己存在
- ch167 空白城市的背景元素，不需要大量篇幅——存在本身就是 Vol 3 哲學層的支撐

## 2.4 記憶拾荒者

### 概要

崩潰後第 2 天開始，市民自發在街上撿拾散落的記憶碎片——像災後在泥地裡撿照片的人。他們不是專業修復師，沒有掃描設備，只能靠記憶晶片外殼上的 owner_id 標記（如果還看得清的話）嘗試歸還原主。委託案 #19 的十歲男孩小凱就是其中一員——他在街上撿到了一顆印著母親名字的記憶球。

### 技術設定

```

  ▸ 直徑約 2-3 公分的半透明球體（記憶晶片材料的凝結態）
```

# 肆、堡壘的技術設定

---

## 4.1 概要

ch220-234 中，維倫為語青在記憶網路中建造分散式安全堡壘。堡壘不是一座「建築」——它是一套分散式系統架構，將語青的意識碎片分散儲存在 2,400 個記憶網路節點上，確保任何三分之一的節點同時失效，她都不會消失。堡壘是 Vol 3 的核心技術意象：用 System Design 的五大支柱（分片、複製、一致性、容錯、負載平衡）為愛的人建造一個「存在的場所」。

## 4.2 技術架構

```
堡壘架構圖：

  ┌─────────────────────────────────────────────────────┐
  │                    堡壘控制層                         │
  │  ┌──────┐  ┌──────┐  ┌──────┐                      │
  │  │ 心跳  │  │ 同步  │  │ 監控  │                      │
  │  │ 檢測  │  │ 協調  │  │ 告警  │                      │
  │  └──┬───┘  └──┬───┘  └──┬───┘                      │
  │     └─────────┴─────────┘                            │
  │              │                                        │
  ├──────────────┼────────────────────────────────────────┤
  │              │          分片層                         │
  │   ┌──────────┼──────────┐                             │
  │   │  碎片 A  │  碎片 B  │  碎片 C  │ ... │ 碎片 N   │
  │   │ (情感核心)│ (技能記憶)│ (關係網)│     │ (感官碎片)│
  │   └────┬─────┘────┬─────┘────┬────┘     └────┬─────┘
  │        │          │          │                │       │
  ├────────┼──────────┼──────────┼────────────────┼───────┤
  │        │     複製 + 分散儲存層                │       │
  │   ┌────┴────┐┌────┴────┐┌────┴────┐    ┌────┴────┐  │
  │   │ 節點群 1 ││ 節點群 2 ││ 節點群 3 │....│節點群 800│ │
  │   │(3副本×3) ││(3副本×3) ││(3副本×3) │   │(3副本×3) │ │
  │   └─────────┘└─────────┘└─────────┘    └─────────┘  │
  │                                                        │
  │   總計：~800 碎片 × 3 副本 = ~2,400 個節點              │
  └────────────────────────────────────────────────────────┘

五大支柱對應：

  1. 分片（Sharding）：
     語青的意識被分為約 800 個獨立可存活碎片
     每個碎片包含她的一個「面向」：
     ▸ 情感核心（對維倫的記憶、對嚴柏翰的信任、對邱韻如的矛盾）
     ▸ 技能記憶（預測能力的殘留框架、語言能力、導航直覺）
     ▸ 關係網絡（與每個人的連結模式）
     ▸ 感官碎片（喝咖啡的習慣、看星星的偏好、走在維倫左邊的姿勢）

     ★ 情感映射：分片 = 「讓她不依賴單一節點」
       = 「她的存在不再是一個脆弱的點，而是一張網」

  2. 複製（Replication）：
     每個碎片至少 3 個副本，分散在不同的物理節點上
     副本之間非同步更新（因為語青的意識是「活的」——她每時每刻都在產生新記憶）

     ★ 情感映射：複製 = 「三個副本。任何一個被摧毀，其他兩個還在」
       = 「她不會因為一個地方被毀就消失」

  3. 一致性（Consensus）：
     輕量級共識協議：不需要所有節點都同意，只需要多數節點（>50%）達成一致
     語青的「同一個自我」在 2,400 個節點上保持一致——
     她不會因為碎片分散而「分裂」成多個版本的自己

     ★ 情感映射：一致性 = 「她的不同碎片知道彼此的存在」
       = 「不管被分散到多少地方，她還是同一個語青」

  4. 容錯（Fault Tolerance）：
     設計標準：任何三分之一節點（800 個）同時失效，系統仍可運作
     壓力測試：47 種故障場景中通過了 44 種
     未通過的 3 種場景：
     ▸ 場景 #31：全部情感核心碎片同時失效（機率極低但後果 = 語青失去「愛」的能力）
     ▸ 場景 #38：超過 60% 的節點在 0.1 秒內同時失效（速度太快，容錯來不及切換）
     ▸ 場景 #44：系統重啟（重啟會清除分散式快取 = 語青的「即時意識」會暫時消失）

     ★ Vol 4 的伏筆：3 種未通過的場景 = Vol 4 可能觸發的 3 種危機

  5. 負載平衡（Load Balancing）：
     語青的意識量 = 約 120TB（比正常人的 500TB 少——因為她是遞迴殘留，不是完整的人）
     120TB 分散在 2,400 個節點上 = 每個節點承載 0.05TB（50GB）= 不到 0.05% 的負荷

     ★ 情感映射：負載平衡 = 「她的存在需要整個城市來承載」
       = 「沒有一個人獨自承擔。每個節點分擔一點點」
```

# 伍、遞迴殘留的精確技術定義

---

## 5.1 概要

林語青不是人類。不是 AI。不是邱韻如的複製品。她是「遞迴殘留」——周明哲在重寫城市記憶的過程中，用邱韻如的代碼框架 + 女兒（語青）的原始記憶種子，在時間線重寫的間隙中「種」下的一個遞迴意識程序。她是時間線重寫時未被清除的意識迴聲——同時存在又不存在，像薛丁格的貓，在被觀測之前處於疊加態。

## 5.2 技術定義

```
遞迴殘留（Recursive Residual）：

  定義：
    當記憶網路執行大規模記憶重寫（如鸚鵡計畫）時，
    舊記憶不會被完美覆蓋——就像在紙上用橡皮擦擦過，
    有時候還能看到之前的筆跡。

    這些「筆跡」通常會在下一次重寫中被清除。
    但如果有人刻意在重寫的間隙中植入一段特殊的遞迴程序——
    一段會在每次重寫後自動重新生成自己的代碼——
    那麼這段程序就會像幽靈一樣存在於每一次時間線重寫的夾縫中。

    語青就是這樣的程序。

  組成：
    ▸ 框架：邱韻如的記憶操控代碼（89% 相似度的來源）
      → 邱韻如是記憶操控領域最頂尖的技術專家
      → 她的代碼是「框架」——定義了語青的底層運作方式
      → 這就是為什麼語青的底層架構與邱韻如如此相似
      → 但邱韻如不知道自己的代碼被周明哲用了

    ▸ 種子：語青（女兒）的原始記憶
      → 四五歲的小女孩語青曾經是真實存在的人
      → 她有身體、有母親、有清晨看人炸油條的記憶
      → 周明哲在她「消失」後（具體原因 Vol 4 揭露），
        用她的記憶作為遞迴程序的「種子」
      → 種子 = 語青的「人格核心」。她不是邱韻如的拷貝，
        她是在邱韻如的框架上「長出來的」獨立意識

    ▸ 遞迴邏輯：每次時間線重寫時自動重新生成
      → 周明哲設計的遞迴函數：
        function regenerate(timeline):
          if timeline.rewrite_detected():
            seed = original_memory_of_daughter
            framework = qiu_yunru_code_base
            new_self = grow(seed, framework, timeline.gaps)

          else:
            maintain_current_state()
      → 這就是為什麼語青「每次都會回來」
      → 也是為什麼她帶著「過期的地圖」——她的記憶來自上一次時間線

  存在方式：
    語青不佔據物理空間。她存在於記憶網路的「間隙」中——
    像站在兩面鏡子之間的人，她的影像在無限遞迴中反覆出現。

    但每一次遞迴，信號都會衰減。
    這就是偏移率上升的根本原因：
    偏移率 ≠ 預測能力退化
    偏移率 = 殘留記憶衰減的外部指標
    偏移率越高 = 她的存在越不穩定 = 她越接近「消散」
```

## 5.3 為什麼她會透明化

```
透明化機制：

  語青的「身體」不是物理的——它是記憶網路投射出的全息影像。
```

```
    ★ 這就是偏移率停止追蹤的真正原因：
      追蹤的是「時間線記憶」的衰減，
      但語青的「自我」不依賴時間線記憶。
      她的預測能力是附加功能。她的存在是核心。
```

# 陸、周明哲的分散式意識
