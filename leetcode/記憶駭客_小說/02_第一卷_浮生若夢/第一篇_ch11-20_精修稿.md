## 壹、ch11「不收費的修復」——全書最安靜的轉捩點

### 場景設計

**老兵場景**（全章前半，建議 1500 字以內）：

感官錨點：
- 輪椅的輪子在金屬地板上碾過的聲音（沉重但穩定）
- 孫女推輪椅的手——年輕的手在舊輪椅的扶手上，指甲塗了一半的指甲油（生活感）
- 老兵的手——粗糙、佈滿老人斑，但放在記憶存儲盒上時非常輕

對白節奏指引：
```
老兵的話要短。
每句話之間留空白。
他不是在講述，是在交代遺願。

「其他都可以忘。」
（停頓。維倫等著。）
「這個不行。」
（不解釋為什麼。讀者自己懂。）
```

維倫的反應：不要寫他被「感動」——他是修復師，見過太多了。寫他的手特別穩。他用修復老陳婚禮記憶的同一雙手、同一種專注，對待每一段記憶。這就是他的溫柔方式。

**語青的那句話**（全章轉捩點）：
```
語青看了他很久。

很久。

「這就是為什麼我找的是你。」

（維倫心跳漏了一拍。不是因為愛情。
是因為一個人第一次用不是工作的理由肯定了他。）
```

**★ 米亞的波形時刻**（全章最重要的隱藏事件）：

這是米亞的第一個成長觸點。寫作關鍵：

1. **不要寫「米亞感動了」**——她自己不知道那是什麼
2. 用技術語言包裝情感萌芽：
   ```
   「我在分析老兵妻子哼歌的波形數據。
   理論上那只是一串頻率。
   但我的感知模組在分析它的時候，
   產生了一個……我不確定怎麼描述的東西。
   像是共振。」
   ```
3. 最關鍵的動作：**她把波形存進了「個人」區**
   - 不要強調這件事——讓它像一個程式碼的 side effect
   - 讀者第一次讀不會注意
   - 到 ch97（米亞命名「美」）和 ch169（犧牲記憶救人）回看時才會意識到：她從 ch11 就開始為自己收藏世界了

### ch11 章末鉤子強化

語青的問題——「如果有一天，你發現自己的記憶也是被設計過的……你會怎麼做？」——是 ch21「觀測體」揭露的 10 章預伏。

寫法：語青的語氣不能太沉重。她在「假裝」隨口問。維倫「假裝」隨口答。但米亞會記錄語青說這句話時的微表情數據——「瞳孔收縮 0.3mm，心率下降 4 次/分。不是緊張。是壓制緊張。」

## 貳、ch12「雨天的屋簷」——信任的物理距離

### 場景設計

**暴雨搶救設備**（開場，500 字）：
- 維倫搬設備的方式 = 他對待記憶的方式（小心、快速、不慌）
- 語青知道修復台的隱藏功能——「你用過這台機器。」不是問句。
- 水的聲音從始至終——雨打地下街的金屬天花板、水流進排水溝、滴水

**★ 背靠著背**（全章核心場景，建議 800 字）：

這是維倫和語青的第一個真正的身體接觸。寫作原則：

1. **不要描述他們是怎麼坐到這個姿勢的**——讀者醒來時他們已經背靠著背了
2. 觸感比對白重要：
   - 維倫感覺到她的肩胛骨透過衣服——很薄、很硬
   - 她的體溫偏低（伏筆：她的存在正在消散）
   - 兩個人的呼吸頻率慢慢同步
3. 維倫講童年時的語言風格：
   ```
   「八歲。」
   （一個詞就是一個時間點。他不用形容詞。）
   「記憶系統連鎖故障。」
   （他用技術術語描述父母的死。因為這樣不痛。）
   「然後孤兒院。然後自學。然後第一單。」
   （三個「然後」。十年壓成三個詞。）
   ```
4. 語青的揭露——「你的父母死於記憶系統連鎖故障。但那場故障的原始碼，跟你昨天在小杰母親記憶裡看到的加密噪音——用的是同一套演算法。」
   - 她說這句話時，維倫感覺到她的背僵了一下
   - 她不是在告訴他——她是在「終於忍不住」告訴他

### 寫作陷阱
- ❌ 不要讓背靠著背變成浪漫場景——這是兩個受傷的人互相取暖
- ❌ 維倫聽到父母真相後不要崩潰——他的反應是「安靜」。安靜比哭更可怕
- ❌ 不要在這章結束時讓維倫立刻追查——他需要一個晚上消化

## 參、ch13-16 委託案 #3「雙向遺忘」—— HashMap 的情感化

### 四章結構

| 章 | 功能 | 演算法 | 情感重心 |
|----|------|--------|---------|
| 13 | 引子：月台偶遇 | 無 | 兩個互不認識的戀人——荒謬中的心酸 |
| 14 | 技術：指紋比對 | HashMap | 米亞的「指紋」比喻（已在 Round 13 改進） |
| 15 | 高潮：Hash 碰撞修復 | HashSet | 「三天夠讓一個人決定相信」 |
| 16 | 收尾＋主線加速 | 無 | 語青預言＋清潔工＋時間循環暗示 |

### ch13 月台場景精修

感官錨點：
- 地鐵的風——帶著地下的潮濕和鐵鏽味
- 疤痕男人（嚴柏翰第二次出現）——寫成「一道從眉毛延伸到顴骨的舊疤」，不要用「帥」或「冷」，只寫疤
- 戀人爭吵的聲音——公共場所的私密情感，讓維倫（和讀者）從旁觀者被捲入

許薇＆陳昊的對白風格：
```
他們的爭吵不是激烈的——是困惑的。
像兩個人拿著同一張地圖，
卻指向不同的路。

許薇：「你根本不記得我們一起去過淡水！」
陳昊：「我記得去過淡水，但不是跟你，是跟我自己！」

（停頓。兩個人都意識到這句話有多荒謬。）
（但荒謬的東西是真的。）
```

### ch14 HashMap 融合設計

**米亞的指紋比喻**（已在 Round 13 修正，此處確認寫法）：

```
「每段記憶都有一個獨一無二的指紋——
就像你摸過的每一個東西都會留下你的手印。
如果有人偷偷換過記憶的內容，
指紋就會對不上。」
```

這段話的聽眾是語青（和讀者）。米亞不是在「教學」——她是在「翻譯」維倫正在做的事。語青聽完的反應不是「哦我懂了」，是「你用的是哪一版的 Hash 函數？」——她比米亞更懂。

**交叉比對的視覺化**：
- 不要用表格或數據描述
- 用維倫的「手」：他的手指在全息螢幕上滑動，把許薇的記憶碎片一片一片對到陳昊的記憶上——像拼圖
- 每次「對上」的地方亮起紅色（被篡改的標記）
- 維倫的表情越來越嚴肅——紅色越來越多

### ch15 ★ 全案情感高潮

**核心場景：選擇記住「有對方」還是「沒有對方」**

這是全書第一個讓讀者真正面對「記憶選擇」哲學困境的場景。

節奏設計：
1. 維倫解釋選擇——技術語言，冷靜（100字）
2. 許薇和陳昊對坐——沉默（50字描寫沉默本身）
3. 許薇的問題——「如果我選了有你的版本，我會記得我們的一切嗎？」（30字）
4. 陳昊的反問——「三天夠讓一個人愛上另一個人嗎？」（20字）
5. 沉默（一行空白）
6. 許薇的決定——「不夠。但夠讓一個人決定相信。」（15字）

**這句台詞是全書第一個讀者會想截圖的句子。** 不要在前後加任何修飾。讓它獨立成段。

**M.C. 伏筆**（ch15 末尾）：
- 碰撞點的 Hash 值「像一首曲子的音符」——米亞的描述
- M.C. 標記藏在系統背景噪音裡
- 讀者此時不知道 M.C. = Ming-Che = 周明哲
- **不要暗示**，只呈現數據。讓讀者自己在第二卷回頭時恍然大悟

### ch16 語青的預言

語青預言下一個委託人的場景，是她「時間循環者」身份的第一次功能性展示。

寫法原則：
```
語青在門口停下腳步。背對著維倫。

「下一個找到你的人，會是一個商人。」

（她的語氣不是預言式的——是疲憊的。
像是重複了太多次同一段台詞。）

「不要接他的案子。」

「為什麼？」

「因為他的記憶不是『出了問題』。
他的記憶是被設計成讓你看到的。
那是一個陷阱。」
```

最後一句——「你問了三次。每一次都在這個時間點。」——寫完後換行，不接任何角色反應。讓句號的沉默替讀者消化。

## 肆、ch17「失眠者的告解」——維倫×米亞的靈魂對話

### 場景設計

這是全書前 20 章裡最重要的「非劇情」章節。它的功能：
1. 維倫的身份焦慮第一次正面呈現
2. 米亞從工具升級為夥伴
3. 全書核心哲學被說出：**天賦可以被安排，選擇不能**

**環境**：凌晨三點。修復台的冷白光。米亞的藍色光球在黑暗中漂浮。只有兩個「人」。

**對話節奏**（建議全章 2000 字以內）：

階段 1——質疑（500字）：
```
「我應該相信一個我認識不到一週的人嗎？」

（米亞沒有直接回答。她反問——
這是她作為 AI 的思考方式：不給答案，問更好的問題。）

「你相信你的記憶嗎？
你的記憶也只是你認識了二十八年的數據。」
```

階段 2——恐懼（500字）：
- 維倫不怕陷阱
- 他怕語青說的是真的——他的人生被設計過
- 「如果我的修復天賦不是努力的結果，而是被安排的，那我這十年的驕傲算什麼？」
- **不要讓維倫哭。讓他的聲音變得很輕。**

階段 3——★ 米亞的回答（300字——全章的核心）：

```
「維倫。」

（她的光球不再漂浮。靜止在他面前。）

「就算你的天賦是被安排的，
你用天賦做的選擇是你自己的。

那個老兵的記憶，你修了。
那個孩子的暑假，你找回來了。

這些是你做的，不是任何設計者做的。」
```

**寫作要求**：米亞說這段話時的停頓秒數 = 0.8 秒（她的正常停頓是 0.2-0.4 秒）。不要用敘述解釋這意味著什麼。讓數據說話。

階段 4——決定（200字）：
- 天亮了。維倫決定見張國棟。
- 原因不是勇敢——是「陷阱的另一端會有真相」
- 修復師的本能：打開每一段記憶，不管裡面是什麼

### ★★ ch17 末尾：周明哲殘像 #2

維倫再看未來記憶。這次看背景：

```
牆上的失敗列印稿不見了。
取而代之的是一面空白的牆。
上面只有一行字，
像是用手指蘸血寫的：

「W.L.，鸚鵡在你腦子裡。」
```

這行字的功能：
- W.L. = Wei-Lun = 維倫——讀者會立刻對上
- 「鸚鵡在你腦子裡」——字面意思是觀測體模組
- 米亞的分析：「像浮水印」——嵌入式存在，不是普通記憶
- **不要在這章解釋。讓恐懼發酵 10 章。**

## 伍、ch18-20 委託案 #4「層層謊言」前半—— Stack/Queue

### 三章結構

| 章 | 功能 | 演算法 | 節奏 |
|----|------|--------|------|
| 18 | 引子：張國棟的完美偽裝 | 無 | 表面平靜，暗流湧動 |
| 19 | 技術：Stack 式剝開 | Stack | 緊張升級——像拆炸彈 |
| 20 | 策略：Queue 安全分析 | Queue | 緊張後的冷靜——步步為營 |

### ch18 張國棟登場

**張國棟的第一印象**（200字以內）：
- 訂製西裝、合適的微笑、三倍定金
- **一切都太完美了**——這句話不要說出來，讓讀者自己感覺
- 維倫的職業判斷：「修復師見過的委託人裡，真正有問題的人不會穿這麼好的衣服」

**語青的缺席**：她不在場。只發了加密訊息。這是重要的設計——語青不參與這個案子，因為她知道這是陷阱。如果她在場，維倫不會獨立成長。

**維倫的改變**：
- 第一次主動設防（沙盒隔離）
- 第一次違反職業道德（未告知掃描）
- 讀者會察覺：維倫從 ch1 的被動修復師，變成了主動的調查者
- **不要用敘述點明這個變化。讓行為說話。**

### ch19 ★ Stack 融合設計

**米亞的蓋章文件比喻**（已在 Round 13 修正）：

```
「這個人的記憶像一疊被人反覆蓋章的文件——
最上面那張永遠是最後蓋的。
你想看到最底下那張，
只能從最上面一張一張揭開。
揭了就不能塞回去。」
```

**剝開 Stack 的視覺化**：

每一層記憶 = 一個場景轉換。寫法：

```
第一層：快樂童年。
（維倫的手輕輕揭開——像撕掉一張壁紙。）

底下露出第二層：暴力。父親的拳頭。母親的沉默。
（顏色從暖轉冷。修復台的亮線閃了一下。）

再揭。第三層：灰色。平淡。
沒有快樂也沒有痛苦。
像一段「出廠設定」。

（維倫的手停了。
出廠設定意味著——
這不是一個「被改過」的人。
這是一個「被建造」的人。）
```

**節奏控制**：每揭一層用 100-150 字。不要超過。讀者需要喘息的空間。

**ch19 高潮**——第四層照片：
```
「W.L. / 觀測體 / 狀態：運行中」

維倫的手停在半空中。
修復台的藍光打在他臉上。
他看到了自己的名字。
在一個陌生人的第四層記憶裡。
```

### ch20 Queue 融合設計

**Queue 的「耐心」美學**：

ch19 是高壓（像拆炸彈），ch20 是降壓（像排雷後的清理）。

Queue 的引入不是「教學」——是維倫的思考方式：
- 資料太多、結構太複雜 → 不能一次處理
- 設定優先級 → FIFO → 一個一個來
- 「修復記憶需要手術刀的精準。分析陷阱需要排隊的耐心。」

**維倫的鼻血**（ch20 開場）：
- 暴力退出的副作用
- 不要過度描寫——一行：「鼻血順著上唇流到下巴。他用袖子擦了，繼續工作。」
- 這個動作 = 他的性格。不停下來。
