## 第 49 章：第二階段

### 場景佈局

```
【時間】法庭裁定後第十五天。十二月初。
【地點】貝葉的家 / Oracular 總部（交叉剪接）
【氛圍】暴風雨前的最後一絲平靜。很快就要碎掉的平靜。
```

### 劇情推進

```
◎ 貝葉線——
  Oracular 遵從法庭裁定，開始公開先驗修改紀錄。
  但——公開的方式很奇怪。
  不是一次全部公開。是分批的。
  每天公開一小部分。
  像在餵食。不是在透明。

  小拉分析公開的紀錄——
  「貝姊。這些紀錄——有東西不對。」
  「什麼？」
  「被公開的部分——都是小幅度的修改。
   0.1σ 以內的微調。
   幾乎看不出來。
   但——我比對了我們的數據——
   大幅度修改的區域——沒有被公開。」

  他們在選擇性公開。
  公開的是無關緊要的部分。
  真正的操控——被藏起來了。

◎ 蘇立恆線——
  他在分析 Oracular 系統的深層架構。
  他發現——在法庭裁定之後——
  Oracular 的系統不是在「等待公開」。
  它在做另一件事。

  系統裡有一個新的程序在跑。
  名字：Oracle Prime。

  「這不是公開程序。
   這是——重新佈署程序。」

  他追蹤 Oracle Prime 的結構——
  發現它的核心——
  是一個巨大的馬可夫鏈。

  全台灣的先驗分布——
  不是獨立被修改的。
  它們透過一個轉移矩陣互相連結。
  修改一個區域的先驗——
  會像骨牌一樣影響所有相鄰區域。

  「這不是在修改幾個參數。
   這是一個活的系統。
   它在自我演化。」
```

### 機率概念

```
【馬可夫鏈 Markov Chain】
- 狀態空間：全台灣的先驗分布（每個區域 = 一個狀態）
- 轉移矩陣 P：P_ij = 從狀態 i 轉移到狀態 j 的機率
- 馬可夫性質：下一步只取決於現在，不取決於過去
- Oracle Prime 把全台灣的先驗連成了一張網——互相影響

【轉移矩陣的隱喻】
- 不是靜態的操控。是動態的。
- 像一個活的有機體——你切掉一處，其他地方會補上。
```

### 章末鉤子

```
蘇立恆打電話給貝葉。
「他們不是在公開。
 他們是在爭取時間。

## 第 50 章：轉移矩陣

### 場景佈局

```
【時間】同一天。深夜。
【地點】馬可瑜家（臨時指揮部）。
【氛圍】所有人都在。像第三篇的蔥油餅會議——但氣氛更緊。
```

### 劇情推進

```
◎ 團隊緊急會議。
  出席：貝葉、蘇立恆（本人到場——第一次在馬可瑜家出現）、
        馬可瑜、高思遠。
  theta 趴在沙發上。尾巴沒有搖。狗也感覺到了氣氛。

  蘇立恆展示 Oracle Prime 的架構：

    ┌─ Oracle Prime 結構 ───────────────────┐
    │                                       │
    │ 核心：馬可夫鏈                        │
    │ 狀態數：152（全台 152 個校準區域）   │
    │ 轉移矩陣：152 × 152                  │
    │                                       │
    │ 特性：                                │
    │ - 每個區域的先驗 = 一個狀態           │
    │ - 轉移矩陣定義了區域之間的影響       │
    │ - 系統每 6 小時自動執行一次轉移      │
    │ - 方向：朝向預設的「目標分布」收斂   │
    │                                       │
    │ 目標：達到穩態分布 π                  │
    │ π = πP                               │
    │ 一旦達到穩態——                        │
    │ 所有區域的先驗鎖定在操控後的狀態     │
    │ 不可逆。                              │
    │                                       │
    └───────────────────────────────────────┘

  高思遠看著那張圖。
  他的臉色——變了。

  「穩態分布。」
  「對。」
  「如果系統達到穩態——」
  「先驗會被鎖死。
   因為在穩態下——
   π = πP。
   系統不再變化。
   你修改任何一個區域——
   轉移矩陣會把它拉回穩態。
   自動修復。自動校準。
   不需要人。」

  馬可瑜推眼鏡：
  「像一個自動恆溫器。
   你把溫度調高——
   恆溫器會自動調回來。
   除非你拔掉恆溫器。」

  蘇立恆：「對。
   除非我們找到先驗寫入端口——
   Oracle Prime 的核心——
   然後關掉它。

   物理位置：Oracular 總部。B3 實驗室。
   地下三層。需要實體存取。」

◎ 馬可瑜的計算——

  「系統多久會達到穩態？」

  她拿過蘇立恆的數據。
  開始算。

  用遍歷性定理（Ergodic Theorem）——
  如果馬可夫鏈是不可約的且非周期的——
  它必然會收斂到穩態。

  「按照目前的轉移速率——
   每 6 小時一次轉移——
   收斂所需的步數——大約 12 步。
   12 × 6 = 72 小時。」

  72 小時。
  三天。

  「三天之後——系統達到穩態。
   操控變得不可逆。」
```

### 機率概念

```
【穩態分布 Stationary Distribution】
- π = πP → 穩態是轉移矩陣的不動點
- 一旦達到穩態，系統不再變化
- 穩態 = 操控的「完成態」

【遍歷性定理 Ergodic Theorem】
- 如果馬可夫鏈是不可約的（所有狀態互相可達）
  且非周期的（不會卡在循環裡）→ 必然收斂到唯一穩態
- 時間平均 = 空間平均

【72 小時倒數】
- 12 步 × 6 小時/步 = 72 小時
- 三天。終極倒數開始。
```

### 章末鉤子

```

他拿出手帕。

「沒事。老毛病。」

但馬可瑜——精算師的眼睛——

「我們有七十二小時。」

## 第 51 章：路徑

### 場景佈局

```
【時間】倒數第 68 小時。
【地點】多地。
【氛圍】佈署。準備。分頭行動。
```

### 劇情推進

```
◎ 計劃制定。
  要關閉 Oracle Prime，必須實體進入 B3 實驗室。
  問題：費雪封鎖了所有常規路線。
  Oracular 總部周圍布滿了安保——
  不是暴力的那種。
  是——偵測的那種。
  透鏡會自動標記接近總部的人。
  如果你的透鏡顯示你「不應該在這裡」——
  安保會被通知。

  貝葉的解決方案：
  「我們不能走預設路線。
   因為費雪的安保系統——
   會預測我們的路徑。
   它用的是什麼？」

  蘇立恆：「馬可夫鏈。
   它根據你之前的行動模式——
   預測你下一步會去哪。
   轉移機率。
   你去過的地方越多次——
   它越確定你下次還會去。」

  貝葉：「那——如果我們的路徑是真正隨機的？」

  蘇立恆看著她。

  「Random Walk。」

  貝葉在白板上畫了一張台北的簡化地圖。

  「Random Walk——隨機漫步。
   在每一個路口——
   我們擲硬幣。
   正面——左轉。反面——右轉。
   沒有模式。沒有歷史。
   完全隨機。

   費雪的馬可夫預測系統——
   無法預測真正隨機的路徑。
   因為——隨機沒有轉移矩陣。
   P(下一步) = 均勻分布。
   每個方向——等機率。」

  蘇立恆：「你要我們用擲硬幣來決定逃跑路線？」

  貝葉微笑了。第一次在這場危機中微笑。
  「不是逃跑。是隨機漫步。
   聽起來比較有學術尊嚴。」

◎ 任務分配：
  蘇立恆 + 貝葉：用 Random Walk 穿越城市，進入 B3。
  馬可瑜：留在外面。監控 Markov Chain 的收斂進度。
          計算剩餘時間。實時更新。
  高思遠：他要做另一件事——
          他要去找柏朗。
          「你們負責系統。我負責人。
           柏朗——需要有人跟他說話。
           三十年了。該說的——不能再等了。」
```

### 機率概念

```
【隨機漫步 Random Walk】
- 每一步的方向 = 隨機（等機率）
- 沒有記憶。沒有模式。不可預測。
- 1D Random Walk：S_n = X₁ + X₂ + ... + X_n（每步 +1 或 -1）
- 隨機漫步 vs 馬可夫預測 = 真隨機 vs 基於歷史的預測
- 如果路徑是真隨機的 → 任何預測都是猜

【反制策略的數學基礎】
- 費雪的安保 = 馬可夫鏈預測（基於歷史行為模式）
- 貝葉的反制 = 真隨機路徑（消除歷史依賴）
- 這就是為什麼隨機性是一種「自由」——不可被預測 = 不可被控制
```

## 第 52 章：收斂

### 場景佈局

```
【時間】倒數 60 小時。
【地點】台北街頭（Random Walk 進行中）/ 馬可瑜的監控站
【氛圍】時間在跑。城市在醒來。他們在城市裡隨機移動。
```

### 劇情推進

```
◎ Random Walk 實況——
  貝葉和蘇立恆在台北的街道上。
  每一個路口——擲硬幣。
  正面左。反面右。

  路線完全不可預測。
  他們自己也不知道下一步去哪。
  這是故意的。

  走過永和。走過中和。走過三重。
  凌晨的城市——很空。
  路燈。便利商店。偶爾一台計程車。

  蘇立恆擲硬幣的方式很特別——
  他不看結果。
  他讓硬幣落在手背上。
  然後用另一隻手蓋住。
  「你不看嗎？」
  「看了會影響我的判斷。
   如果我看到正面——
   但我想走右邊——
   我可能會再擲一次。
   那就不是真隨機了。」
  「所以你——」
  「我蓋著。然後直接走。
   不讓偏好汙染隨機性。」

  貝葉看著他。
  「你連擲硬幣都這麼嚴謹。」
  「……職業病。」

◎ 馬可瑜的監控——
  她在家裡。三個螢幕。
  左邊：Markov Chain 的收斂進度。
  中間：貝葉和蘇立恆的位置（GPS）。
  右邊：城市的即時數據。

  她在追蹤收斂：

    ┌─ Oracle Prime 收斂進度 ────────────┐
    │                                     │
    │ 步數已完成：3 / 12                 │
    │ 剩餘步數：9                        │
    │ 每步間隔：6 小時                   │
    │ 預計穩態時間：54 小時後            │
    │                                     │
    │ 收斂指標：‖π_n - π‖ = 0.341      │
    │ 穩態門檻：‖π_n - π‖ < 0.001      │
    │                                     │
    │ 趨勢：收斂中。速率穩定。          │
    │ 不可逆閾值：‖π_n - π‖ < 0.010    │
    │ 預計到達不可逆閾值：36 小時後     │
    │                                     │
    └─────────────────────────────────────┘

  「36 小時後到達不可逆閾值。
   54 小時後完全穩態。
   但——36 小時才是真正的死線。
   因為一旦低於 0.010——
   即使我們關掉 Oracle Prime——
   系統也會在慣性下繼續收斂。」

  她傳訊息給貝葉：
  「真正的期限是 36 小時。不是 72。」

◎ 高思遠線——
  他獨自坐計程車去 Oracular 總部。
  沒有告訴任何人。
  他要去見柏朗。

  他在車上。看著窗外。
  台北的黎明。
  天邊有一條金線。

  他咳了一下。
  拿出手帕。
  紅色——比上次多了。
  他看了一眼。折起來。塞回口袋。

  「師父不能沒有學生。
   但學生——也不能沒有師父。
   三十年了。
   該上最後一課了。」
```

### 機率概念

```
【馬可夫鏈的收斂】
- ‖π_n - π‖ → 0（n → ∞）
- 收斂速率取決於轉移矩陣的第二大特徵值
- 不可逆閾值：收斂到某個程度後，即使停止，慣性會繼續

【Random Walk 的實踐】
- 真隨機 = 不讓人類偏好汙染
- 蘇立恆蓋住硬幣 = 確保隨機性的嚴謹
- 隨機路徑 = 自由（不可預測 = 不可控制）
```

### 章末鉤子

```
馬可瑜的第二則訊息：
「收斂速率在加快。
 不是等速的。
 他修改了轉移矩陣——
 加大了對角元素。
 系統正在自我強化。

 新預估：不可逆閾值——28 小時。」

28:00:00
時間比預期少了 8 小時。
柏朗在加速。
```

## 第 53 章：Poisson 的意外

### 場景佈局

```
【時間】倒數 24 小時。
【地點】台北街頭——接近 Oracular 總部的外圍。
【氛圍】黎明。城市開始有人。風很大。十二月的風。
```

### 劇情推進

```
◎ Random Walk 中的意外事件——Poisson Process。

  隨機漫步不是在真空中走的。
  城市是活的。會發生事情。
  意外的事情。
  在時間軸上隨機出現的事件——
  就是 Poisson Process。

  事件一（幫助）：
  走到一個巷口。一個年輕人攔住他們。
  「你們是……陳貝葉？」
  「……你是誰？」
  「我叫林安。我是林岳的弟弟。」

  林岳——第二篇裡被「校準失敗」的同事。
  他的弟弟。

  「我哥——他在醫院。他不記得很多事了。
   但他記得一個人。
   他記得你。
   他叫我……如果有機會遇到你——
   告訴你一件事。」

  「什麼事？」

  「他說——B3 的備用通道。
   從地下停車場的 P2 層。
   消防逃生梯。
   安保系統不覆蓋那裡。
   因為——那是他設計的逃生動線。

### 機率概念

```
【Poisson Process】
- 時間軸上的隨機事件
- λ = 平均速率（事件/時間）
- P(N(t) = k) = (λt)^k e^{-λt} / k!
- 事件之間的等待時間 ~ Exp(λ)

【指數分布的無記憶性 Memoryless Property】
- P(T > s+t | T > s) = P(T > t)
- 不管已經等了多久，下一次的等待時間分布不變
- 跟 Markov 性質對應：未來只取決於現在，不取決於過去
```

  旁邊有那張紙條。五摺的。

  「不急。」

## 第 59 章：隨機（尾聲前的倒數第二章）

### 劇情推進

```
◎ 貝葉一個人走在台北的街上。
  傍晚。
  透鏡開著。個人先驗模式。

  世界看起來——跟以前不同了。
  不是更好。不是更差。
  是——更真。

  她的先驗是她自己的。
  P(rain tomorrow) = 0.62。
  但那是她的 0.62。不是 Oracular 的。
  也許跟 Oracular 的一樣。也許不一樣。
  不重要。
  重要的是——那是她的。

  她走到一個路口。
  停了。
  口袋裡——
  那張紙條。五摺的。
  蘇立恆的 Type II Error。

  打開。
  看了一眼。

  然後——
  她在紙條的背面寫了一行字。

  「後驗 ∝ 先驗 × 似然。
   先驗是你的。
   似然是我的。
   後驗——是我們的。」

  她把紙條折回去。
  五摺。
  放回口袋。

  繼續走。
