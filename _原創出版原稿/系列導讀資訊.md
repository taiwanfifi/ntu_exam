## 閱讀順序建議

### 情境一：準備資格考（統計＋機率＋演算法）

**建議順序：白噪音 → 機率預言師 → 記憶駭客**

原因：
- 白噪音是暖身。統計學是機率的應用層，概念最直覺（你看得到數據、畫得出圖）。用懸疑劇的節奏把「為什麼要學統計」這件事內化——讀完你會記得「常態分布是堡壘」「相關不等於因果但有些真相是一把刀」，這些比課本定義更容易在考試時被你的大腦調出來
- 機率預言師是核心。機率是統計的理論基礎，也是演算法分析的前置知識。「先驗是偏見，似然是遭遇，後驗是成長」——如果你讀完這本還不懂貝氏定理在幹嘛，那大概沒有東西能教你了
- 記憶駭客是終局。演算法需要前面兩本的數學成熟度。而且這本最長（320 章），當你已經被前兩本養出閱讀慣性，才不會被篇幅嚇跑

你能學到什麼（考試層面）：
- 不是學會解題，是學會「為什麼這個概念存在」。資格考最難的不是計算，是理解一個定理的動機。這三本小說做的事情就是把動機寫成故事
- 白噪音讓你記住統計工具的「使用情境」
- 機率預言師讓你理解推導背後的「哲學邏輯」
- 記憶駭客讓你感受演算法的「設計思維」——為什麼用 Segment Tree 而不是暴力解，不是因為比較快，是因為問題的結構長那個樣子

---

### 情境二：台大資工學生

**建議順序：記憶駭客 → 機率預言師 → 白噪音**

原因完全不同：
- 資工學生已經會演算法，所以記憶駭客對他們來說不是在學新東西，而是在經歷一種「原來我每天在寫的 code 可以這樣被理解」的震撼。XOR 不只是位元運算，是兩個人「只有不同的部分才能還原彼此」的愛情。Segment Tree 不只是區間查詢，是極權體制的隱喻。這本會改變他們看待自己專業的方式
- 機率預言師填補資工學生最常見的弱點——統計與機率的直覺。很多資工人可以推導 MLE 但不知道它在「說什麼」。這本用故事把直覺補上
- 白噪音最後讀，因為它最「非技術」，是一個社會派故事。資工學生讀完前兩本後，用這本來體驗「技術被用在人身上」的倫理衝擊

資工學生能學到什麼：
- 記憶駭客：你寫的每一行 code 都有哲學意義。選擇 O(n log n) 而非 O(n²) 不只是效率，是一種「用有限資源面對無限問題」的人生態度
- 機率預言師：ML 課教你調參數，這本教你為什麼 prior matters——而且不只在模型裡，在人生的每一個決策裡
- 白噪音：當你有能力用數據分析一個人的行為模式時，你的責任邊界在哪裡

---

### 情境三：一般大學生（非理工背景、純粹有興趣）

**建議順序：白噪音 → 記憶駭客 → 機率預言師**

- 白噪音先讀，因為它最像一般人會看的小說——懸疑、推理、有 CP 感。數學都藏在「諮商手記」裡，正文讀起來就是一個好看的犯罪故事。門檻最低，但讀完你會發現自己不知不覺懂了什麼是常態分布
- 記憶駭客第二，因為它的情感後座力最強（催淚）。就算你不懂演算法，米亞的 0.4 秒延遲、阿嬤的鹹豆漿、語青消散時的 21.8 度微溫——這些不需要技術背景就能被打動。演算法的部分你可以當「很酷的設定」來讀
- 機率預言師最後，因為它門檻最高。但如果你被前兩本圈粉了，你會有動力啃這本。而且讀完這本，你會成為那種「在蔥油餅攤前突然想到貝氏定理」的人

一般大學生能學到什麼：
- 理科不是你以為的那種冷冰冰的東西
- 數學是一種看世界的方式，不是一種折磨你的考試工具
- 白噪音教你：數據會說謊，但也會替沉默的人說話
- 記憶駭客教你：遺忘不是失去，記憶不是擁有
- 機率預言師教你：沒有人是客觀的，但承認自己的偏見（prior）就是成長的開始

---

**一句話總結：** 考試的人從最簡單的開始（白→機→記），資工的人從最熟悉的開始（記→機→白），一般人從最好看的開始（白→記→機）。三條路最終都會走完整個圓。
