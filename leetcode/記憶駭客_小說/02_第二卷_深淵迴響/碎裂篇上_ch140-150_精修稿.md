### ch141 他們是戀人——催淚彈與擁抱

**核心意象**：血染的實驗服

**核心場景**：

```
錄像繼續。

遊行到了下午三點。
標語還在搖晃。
嚴柏翰還在最前面。

然後畫面抖了一下。

鏡頭搖晃。
聲音突然變了——
不再是口號。
是尖叫。

催淚彈。

第一顆落在隊伍右側。
白煙像一隻巨大的手掌從地面升起。
人群開始跑。

第二顆。第三顆。
白煙吞沒了標語。
吞沒了舉起的拳頭。
吞沒了所有人的臉。

鏡頭在白煙中搜索——
找不到嚴柏翰。

然後畫面穩定了。
鏡頭拉近。

他在地上。

臉朝下。
黑色外套的背面有一道長長的撕裂——
像被什麼東西劃開的。
血從撕裂處滲出來。
在柏油路面上蔓延。

他不動了。

然後——

一個女人從白煙裡衝出來。
她在跑。
鎮暴盾牌在她面前排成一道牆——
她不是繞過去的。
她是撞過去的。

她摔了一跤。
膝蓋著地。
爬起來。
繼續跑。

她跑到嚴柏翰身邊。
跪下去。
把他翻過來。

他的臉上全是血和催淚液。
她用自己的袖子擦他的臉。

她的實驗服——
白色的實驗室外套——
沾滿了他的血。

她把他的頭抱進懷裡。
一隻手捂住他左眼上方的傷口——
血從她的指縫間流出來。

四秒。

錄像結束。
月台牆壁恢復黑暗。

安靜。

**設計解析**：

- 催淚彈場景的節奏 = 快→急停。前半段快速（尖叫、白煙、跑），後半段急停（她抱住他，鏡頭停留四秒）。四秒 = 讀者的呼吸被按住
- 邱韻如「撞過」鎮暴盾牌而不是繞過 = 她的行動 DNA。她不繞路。她直撞。這個細節呼應 ch156-157 她按下刪除鍵時「手指沒有顫抖」= 同一個人。做最痛的事時，最直接
- 實驗服上的血 = 核心意象。「白色的實驗室外套沾滿了他的血」= 她的身份（科學家）和她的愛（戀人）在這一刻融合。實驗服不是戰鬥服——她穿著工作服衝進了戰場
- 語青「握著左手腕」= 替母親的動作做鏡像。她跟邱韻如一樣——面對最大的痛苦時，不哭。只有身體的無意識動作洩露了一切
- 嚴柏翰「摸疤痕」= 身體記得母題的又一次響起。ch35「手記得」、ch140「嘴唇記得」、ch141「疤痕記得」——三種不同的身體記憶，三次不同的真相揭露
- 嚴柏翰用嘴型重複「邱韻如」= 他的嘴唇在 ch140 無聲重複了遊行口號。現在他的嘴唇在重複一個人的名字。從口號到名字 = 從信念到愛情。嘴唇記得的東西比大腦記得的更深
- 「她是你的戀人」由語青說出 = 女兒替母親做了一次自我介紹。這是整個二代關係中最安靜的連結

**寫作指令**：

1. 催淚彈場景 ≤400 字。不要寫太多鎮壓細節——這不是社會運動小說。只需要：催淚彈、白煙、他倒地、她衝過來。四個動作
2. 邱韻如衝過防線的動作用短句碎句。每個動作一行。不要合併。她在跑。她摔了。她爬起來。她繼續跑。= 讀者跟著她的動作呼吸
3. 「實驗服沾滿了他的血」= 獨立成段。這句是 ch141 的關鍵視覺。讓它燒進讀者的腦海
4. 鏡頭停留「四秒」= 寫出來。不要寫「很久」。四秒 = 精確。精確的時間比模糊的時間更有重量
5. 月台反應段不要寫維倫的內心。維倫在 ch141 是見證者，不是主角。主角是嚴柏翰
6. 語青蹲下來看嚴柏翰的眼睛 = 呼應 ch153（記憶中邱韻如蹲下來教小語青折紙鶴）。母女的姿態是遺傳的
7. 嚴柏翰的最後反應不要寫哭。不要寫任何情緒詞。只有嘴唇在動。聲音消失 = 語言崩潰 = 他的理性被擊穿了
8. ≤3,000 字。催淚彈段 + 月台反應段 + 語青揭露段。三段結構

### ch142 鸚鵡計畫——Topological Sort 篡改鏈

**核心意象**：一張按順序展開的滅口清單

**核心場景**：

```
安全屋。
維倫打開從記憶銀行取得的鸚鵡計畫文件。

米亞將文件投射在月台牆壁上。

第一頁是目錄。

鸚鵡計畫。
代號：Parrot。
啟動日期：2079.11.03。
目標：消除新台北所有市民對 2079.11.02 遊行事件的記憶。
覆蓋方式：替換式植入（「什麼都沒發生」的虛假日常記憶）。
預計覆蓋人數：2,847,000 人。
實際覆蓋人數：2,770,000 人（覆蓋率 97.3%）。

維倫讀到「2,847,000 人」時，
他的手停在半空。

差不多三百萬。
新台北的全部人口。

他們——把全城所有人的記憶都改了。
```

**Topological Sort 教學場景**：

```
「米亞，文件裡有執行記錄嗎？」

「有。完整的執行序列。」

米亞展開第二頁：

鸚鵡計畫——執行依賴鏈

步驟 1：偽造周明哲的死亡紀錄
        （前置條件：無）

步驟 2：關閉周明哲的公開研究終端
        （前置條件：步驟 1 完成）

步驟 3：替換記憶銀行核心伺服器中的遊行監控紀錄
        （前置條件：步驟 2 完成——終端關閉後才能修改）

步驟 4：切斷所有新聞媒體的遊行報導索引
        （前置條件：步驟 3 完成——先改源頭再斷傳播）

步驟 5：分區覆蓋——從遊行核心區域向外圈逐層擴散
        第一圈：遊行參與者（3 萬人）
        第二圈：現場目擊者（12 萬人）
        第三圈：當日在場居民（60 萬人）
        第四圈：透過媒體得知的市民（270 萬人）
        （前置條件：步驟 3+4 完成——先消除源頭證據，再消除個人記憶）

步驟 6：植入替代記憶——「2079.11.02 是普通的星期四」
        （前置條件：步驟 5 完成——先清空再填入）

步驟 7：清除執行痕跡——刪除本計畫的操作日誌
        （前置條件：步驟 1-6 全部完成——最後刪除自己）

維倫看著這七個步驟。

「米亞。」

「我看到了。」

「這是——」

「拓撲排序。」

米亞的聲音很平。
但她說完之後停頓了 1.7 秒。
對她來說這是很長的沉默。

「每一步依賴前一步的結果。
  順序不能亂。
  先偽造死亡，才能關閉終端。
  先關閉終端，才能修改源頭。
  先修改源頭，才能斷傳播。
  先斷傳播，才能覆蓋個人記憶。
  最後——刪除自己的痕跡。

  完美的拓撲排序。
  沒有環路。
  沒有死鎖。
  每一步的入度為前序步驟的出度。

  這不是一場暴力。
  這是一場工程。」
```

**演算法：Topological Sort（篡改依賴鏈）**

```
A 線：鸚鵡計畫的執行 = 一張有向無環圖（DAG）
     每一步是一個節點
     依賴關係是有向邊
     拓撲排序找出唯一可行的執行順序：
       1→2→3→4→5→6→7
     不可能在步驟 3 之前執行步驟 5
     不可能在步驟 6 之前執行步驟 4
     每一步的入度（依賴數量）決定了它在序列中的位置

     關鍵教學點：
     拓撲排序只在 DAG 上有意義。
     如果有環——就會死鎖。
     鸚鵡計畫之所以成功，
     是因為設計者確保了「沒有任何步驟互相依賴」。
     這需要極高的系統架構能力。

B 線：篡改歷史 = 一場精心計算的工程。
     它不是暴力。
     是按照依賴關係一步一步、從核心到外圍的系統性覆蓋。
     先殺死周明哲（源頭），
     再清除傳播（媒體），
     再覆蓋個人（三百萬人），
     最後刪除自己的存在。
     完美。乾淨。安靜。
     像什麼都沒發生。

     ——事實上，這就是他們想要的效果：
     「什麼都沒發生。」

同步點：米亞說「這不是一場暴力。這是一場工程。」
       A 線的「拓撲排序 = 無環有序執行」
       = B 線的「篡改歷史 = 按依賴順序系統性消除」
       讀者在恐懼中理解了拓撲排序的核心概念：
       有些事情必須按順序做，順序本身就是權力。
```

**維倫的反應**：

```
維倫站在投射的文件前面。

他的手指沿著依賴鏈的箭頭移動——
從步驟 1 到步驟 7。

他走完了整條鏈。
然後他的手停在步驟 5。

「第一圈：遊行參與者。三萬人。」

三萬人。
昨天他在錄像裡看到了他們的臉。
標語。拳頭。口號。
「記憶屬於每一個人。」

他們喊完了口號。
然後被催淚彈打倒。
然後——被刪除了記憶。

他們忘記了自己曾經站起來。

維倫的手從牆上放下來。

「這已經執行過了。」

不是問句。是確認。

「97.3%。」米亞說。

維倫轉向嚴柏翰。

嚴柏翰坐在月台角落。
他沒有看投射。
從錄像結束之後他就沒有看任何東西。
他在看自己的手。

那雙十年前舉著標語的手。
現在空空的。
```

**寫作指令**：

1. 鸚鵡計畫的七步依賴鏈用清單式呈現。不要用散文。清單的冷靜 = 篡改的系統性。像讀一份工程文件——讀者應該感覺到「設計這個計畫的人不是瘋子，是工程師」
2. 米亞停頓 1.7 秒 = 她在消化這個計畫的規模。她的停頓秒數遞進表中，1.7 秒大約等於「超出程式設定」的思考深度。她不評價——她只描述。但她的停頓比評價更有力
3. 「這不是一場暴力。這是一場工程。」= 本章的核心台詞。獨立成段。讓這句話的重量沉下去
4. 維倫的反應聚焦在「步驟 5」——因為步驟 5 是人。不是系統。不是終端。是三萬個活生生的人。他的手停在那裡 = 他作為修理師的本能：人不是可以被「覆蓋」的數據
5. 嚴柏翰在本章幾乎不說話。他看著自己的手 = 那雙舉過標語的手。現在什麼都不記得。他是步驟 5 的第一圈裡面的一個人
6. 不要在本章揭露鸚鵡 2.0。97.3% + 殘留 2.7% = 留給 ch143 的鉤子
7. ≤2,800 字

## 參、ch145 阿嬤的那碗豆漿——呼吸章 + 情感錨點

### 章節定位

```

### 與 ch83 的精確呼應

```

  ch83：身體記得一個「少掉的人」（語青）

### ch146 「我要回去」——嚴柏翰的決心

**核心意象**：傍晚的光線

**核心場景**：

```
看完遊行錄像和鸚鵡計畫之後。
嚴柏翰沉默了一整天。

一整天。

他沒有說話。沒有吃東西。
他坐在月台角落——
同一個角落——
從早上到傍晚。

語青遞過去的水。他沒有接。
維倫遞過去的食物。他沒有碰。
米亞試圖投射嚴柏翰的生理數據。
他用手擋住了光球。

「不要。」

那是一整天裡他說的唯一一個字。

──

傍晚。
地面上的光透過通風口滲入月台。
不是月光。是夕陽。
橘色的。
在灰色的月台上畫出一道斜斜的光柱。

嚴柏翰站了起來。

他站起來的動作——
不像 ch134 那樣突然。
不像被折彎的刀彈直。

是慢慢站起來的。
像一座被壓了很久的建築
一根柱子一根柱子地把自己撐直。

他走到維倫面前。

夕陽的光柱正好打在他的左半邊臉上。
那道疤在橘色的光裡像一條被照亮的河。

他看著維倫。

維倫看著他。

  我要知道我是誰。」
```

**設計解析**：

- 「一整天」= 參照嚴柏翰情感崩潰模型。否認（3秒）→ 驗證（10分鐘）→ 無力 → 沉默 → 接受。他用一整天走完了五個階段。然後站起來
- 站起來的動作「慢慢的」vs ch134 的「彈直」= 兩次站起來的區別。ch134 他站起來是為了逃離（暴怒→摔門）。ch146 他站起來是為了面對（決心→行動）。ch134 是破壞性的。ch146 是建設性的
- 疤痕在夕陽中「像一條被照亮的河」= 全書第一次（也是唯一一次）疤痕被描寫為美的。之前疤痕 = 傷害、記憶、痛苦。此刻疤痕在光裡 = 它不再是傷口，而是地圖——一條通往他真實身份的路
- 「我要恢復原始記憶。我要知道我是誰。」= 兩句話。句號。不是問句。不是請求。是宣告。嚴柏翰的語言 DNA：短句、結論在前、不用形容詞
- 他走到「維倫面前」而不是語青面前 = 他選擇了信任維倫。不是因為語青不值得信任——是因為維倫是修理師。他需要一個修理師。他要修復自己

**寫作指令**：

1. 嚴柏翰沉默一整天 = 用時間跳躍。不要寫他沉默的過程。「從早上到傍晚」一句話。過程留空 = 讀者自己填充
2. 「不要。」= 他一整天唯一的字。放在拒絕米亞投射數據的時候。他不要被量化。他需要不被任何人打擾的時間
3. 夕陽的描寫 ≤3 行。不要把通風口的光寫得太浪漫。這不是 ch135 的溫柔月光。這是決戰前的最後一道光
4. 嚴柏翰的台詞之後不要接任何人的反應。以他的話結尾。留空。ch147 的開頭再寫維倫和米亞的回應
5. ≤1,800 字。決心不需要很多字。嚴柏翰不是會演講的人。他站起來、走過去、說一句話。完

### ch147 容量的極限——DP/Knapsack 問題定義

**核心意象**：一個數字：34%

**核心場景**：

```
米亞用了兩個小時掃描嚴柏翰的神經結構。

結果出來了。

她投射了一張腦容量示意圖——
兩個色塊像兩片大陸板塊擠在一張太小的地圖上。

藍色 = 十年探員記憶
  案件、審訊、追蹤、格鬥訓練、監控局通訊協議、
  同事的臉、獎章、升遷、獨處的夜晚。
  佔用容量：62%。

紅色 = 被覆蓋的原始記憶（鸚鵡計畫覆蓋層下方）
  台大研究生時期、實驗室、邱韻如、遊行、被捕、
  一切被刪除前的嚴柏翰。
  佔用容量：72%。

「62% + 72% = 134%。」
米亞說。

「你的大腦只有 100% 的容量。
  超出 34%。
  如果同時保留兩套記憶——
  神經突觸會在 72 小時內開始崩潰。
  不可逆的損傷。」

嚴柏翰看著那張圖。

「換句話說——」

維倫替他說完：

「你必須放棄一部分。」

沉默。

米亞繼續：

「這是一個背包問題。」

她的語氣恢復了技術模式——
像是用理性覆蓋住剛才的沉默。

「你的大腦是一個背包。
  容量上限：100%。
  你有兩組物品——
  每一段記憶都有『重量』（佔用的神經容量）
  和『價值』（對你的人格和身份的重要性）。

  你必須選擇——
  在不超過 100% 容量的前提下，
  保留價值最高的記憶子集。

  放棄的部分——
  會被永久清除。
  不可恢復。」
```

**演算法：DP/Knapsack（問題定義）**

```
A 線：0/1 背包問題
     容量 W = 100%（腦容量上限）
     物品集合 = 嚴柏翰的所有記憶段
     每段記憶 i 有：
       weight[i] = 神經容量佔用百分比
       value[i] = 對人格的重要性（0-100 的人格依賴分數）

     約束：選取的記憶總 weight ≤ 100%
     目標：最大化總 value

     0/1 = 每段記憶要嘛完整保留，要嘛完整放棄
           不能「保留一半的案件記憶」
           記憶不是可以被切割的——它是一個整體

B 線：「什麼值得記住」= 人生的背包問題
     你的人生容量有限。
     你無法同時擁有所有的經歷。
     有時候你必須選擇——
     帶走什麼，留下什麼。
     而留下的東西——
     不是消失了。
     是你選擇讓它消失。

     嚴柏翰面對的不只是記憶的取捨——
     是「我是誰」的取捨。
     保留探員記憶 = 他是現在的嚴柏翰
     保留原始記憶 = 他是十年前的嚴柏翰
     他無法同時是兩個人。

同步點：米亞說「你的大腦是一個背包」
       A 線的「容量有限必須選擇」
       = B 線的「人生有限必須取捨」
       讀者不需要懂 DP——
       他們只需要想像：如果是你，
       你會保留哪些記憶？
```

**嚴柏翰的第一反應**：

```
嚴柏翰看著腦容量示意圖。
藍色和紅色。
兩個自己。

「如果我放棄所有藍色——」

「探員記憶全部清除。
  你會失去十年的辦案經驗、
  所有學到的技能、
  所有認識的人。」

米亞停了一下。

「包括——跟維倫的記憶。」

嚴柏翰的目光從示意圖移到維倫身上。

一秒。

然後移回來。

「如果我放棄所有紅色——」

「原始記憶永久清除。
  你永遠不會知道自己是誰。
  邱韻如——會徹底從你的生命中消失。
  連名字都不剩。」

嚴柏翰的手——
又開始握拳了。
握了。鬆開。握了。鬆開。

第三個選項。

「能不能——兩邊都保留一部分？」

「可以。」米亞說。
「但你必須逐段評估。
  每一段記憶的重量和價值。
  然後做出選擇。」

嚴柏翰閉上眼睛。

「開始。」
```

**設計解析**：

- 米亞說「包括跟維倫的記憶」時特別停頓 = 她知道這句話的重量。她不是在報告數據——她是在提醒嚴柏翰代價
- 嚴柏翰看維倫「一秒」= 最短的計量單位。但足夠做一個判斷。他在那一秒裡決定了：維倫的記憶不能放棄
- 「握了。鬆開。握了。鬆開。」= 呼應 ch35 的身體記得——那時他的手「反覆握緊鬆開」是因為記起了邱韻如。這裡同樣的動作 = 他在兩個自己之間搖擺
- 「第三個選項」= 嚴柏翰不接受二選一。他要兩邊都保留一部分。這是他的性格——果斷但不極端。他找第三條路
- 「開始。」= 一個字。ch148 的觸發器

**寫作指令**：

1. 腦容量示意圖的描寫 ≤150 字。用色塊比喻。藍色 vs 紅色。不要用太多技術名詞
2. 米亞的背包問題解釋 ≤200 字。「你的大腦是一個背包」一句話定義問題。然後列三個要素：容量、重量、價值。不需要公式
3. 嚴柏翰的兩個「如果」= 快問快答。不要拖。讀者需要快速理解兩個極端選項的代價
4. 嚴柏翰看維倫的「一秒」不要加任何解釋。一秒。然後移回來。讀者自己讀出含義
5. 「開始。」之後空行。ch147 結束。ch148 開始
6. ≤2,200 字

### ch148 什麼值得記住——★★★★★ 逐一秤量記憶

> **寫法原則**：這是 Vol 2 最重要的演算法教學場景，也是最重要的情感場景。每一段記憶的「秤量」= 一次 AB 線同步。技術上是背包問題的物品評估。情感上是「你的人生中什麼才真正重要」。
> **節奏**：極慢。每一段記憶佔 200-300 字。停下來。看嚴柏翰的反應。再下一段。

**演算法：DP/Knapsack（0/1 背包——逐一評估）**

```
A 線：0/1 背包的核心操作
     for each item i:
       if weight[i] > remaining_capacity:
         skip（裝不下）
       else:
         decide: include or exclude?
         include: value += value[i], capacity -= weight[i]
         exclude: keep current state

     嚴柏翰的每一段記憶 = 一個 item
     每一次「保留」或「放棄」= 一次 include/exclude 決策

     讀者不需要看到代碼——
     他們看到的是嚴柏翰的手在發抖

B 線：每一段記憶被秤量的過程
     不是「哪個有用」——
     是「哪個是我」

     有用的記憶可以重新學習
     但「我是誰」的記憶不能

     嚴柏翰必須在「有用」和「是我」之間做選擇

同步點：每一次嚴柏翰說「放棄」或「保留」
       = A 線的 include/exclude
       = B 線的「這是我的一部分」或「這不再是我」
```

**逐一秤量——按順序列出所有被評估的記憶段**：

```

嚴柏翰搖頭。

**設計解析**：

- 三十七個同事的名字 = 最具體的痛。不是「同事」這個概念——是三十七個名字。他讀了兩遍。然後放棄。這比任何抽象的描述都有力
- 嚴柏翰要求「壓縮」紅色記憶 = 第三個選項。他不接受「全留藍色或全留紅色」，也不接受「藍色全棄」。他找到了第三條路：壓縮原始記憶的精度，騰出空間保留藍色中最重要的三樣。代價 = 邱韻如的臉可能不完全清楚。他接受了。因為他選擇了「記得她」而不是「看清她」

### 核心意象：迷宮的入口

### 核心場景

```
安全屋。
維倫坐在修復台旁邊。
嚴柏翰躺在修復台上。

這張修復台——
維倫在上面修過鋼琴師的根。

修過退役探員的迷宮。

嚴柏翰閉著眼睛。

維倫看了米亞一眼。

米亞：「記憶重組方案已上傳。
  三樣保留項已鎖定。
  壓縮比率 = 原始記憶從 72% 壓縮至 65%。
  預計損失：細節模糊度增加 15-20%。
  情感核心保留率：96%。」

維倫：「恢復的方式呢？」

米亞：「Backtracking。」

她投射了嚴柏翰被覆蓋的記憶結構——
像一座被多層混凝土封住的地下城。

最外層 = 監控局疊加的忠誠人格強化。
第二層 = 邱韻如植入的探員身份。
第三層 = 鸚鵡計畫覆蓋的十一月二日記憶。

「四層覆蓋。
  從最外層開始——
  逐層剝開。
  每剝一層，檢查：
    路通不通？
    如果通——繼續深入。
    如果不通——回退到上一層，換一條路。

  這就是 Backtracking。
  系統性地嘗試所有路徑。
  碰壁就退。
  記錄已探索的狀態。
  直到找到通往最深層的路。」
```

**演算法：Backtracking**

```
A 線：Backtracking 的核心機制
     State = 當前記憶層（4 層）
     Choice = 每一層有多個可能的切入點
     Constraint = 某些切入點被防禦機制堵住
     Goal = 到達最深層——原始記憶

     偽代碼：
     def restore(layer):
       if layer == 最深層:
         找到了。開始恢復。return True
       for entry in layer.possible_entries:
         if not blocked(entry):
           if restore(next_layer(entry)):
             return True
           else:
             undo(entry)  # 回退
       return False  # 所有入口都不通

B 線：回溯 = 回到過去。
     不是時間旅行——
     是在被覆蓋的記憶層中
     一步一步走回原點。
     每一步「回退」不是失敗——
     是「知道了這條路不通」。

     嚴柏翰要回到自己——
     不是跳回去。
     是一層一層地揭開
     別人蓋在他身上的東西。

同步點：維倫啟動修復程序的瞬間——
       A 線的 restore(layer_1) 開始執行
       = B 線的「嚴柏翰開始回家」
       Backtracking 的每一步「回退」
       = 每一層假記憶被剝離
       = 嚴柏翰離真正的自己又近了一步
```

**啟動的瞬間**：

```

他閉上眼睛。

「米亞，開始。」

嚴柏翰的身體微微抖了一下。

維倫的意識進入了嚴柏翰的記憶空間——

黑暗。
層層疊疊的黑暗。
像站在一座沒有燈的大廈的頂樓
往下看——
每一層都是封閉的。
每一層都有鎖。

維倫深呼吸。

「第一層。監控局強化層。
  正在掃描入口......」

他開始走。

（ch150 在此結束。
  嚴柏翰的記憶恢復六階段
  = ch151-157 碎裂篇下半的內容。）
```

## 柒、寫作陷阱（10 條）

1. **ch141 催淚彈場景不要寫成動作片**。這不是戰爭場景——這是一段被埋葬的歷史的最後一幀。重點不是「多少催淚彈」，是「她衝過去抱住他」。一個動作 > 一百個爆炸。催淚彈 ≤3 顆。白煙 ≤3 行。然後直接到她衝過來

2. **ch142 拓撲排序不要寫成教科書**。七步依賴鏈用列表呈現——讓格式做教學的工作。米亞的解釋 ≤5 行。「這不是暴力。這是工程。」一句話比一段解釋有力十倍

3. **ch143 鸚鵡 2.0 的恐怖不要靠「解釋」**。不要讓角色長篇分析 2.0 的影響。「所有人的核心身份記憶」這十個字就夠了。說完之後 = 沉默。讓讀者自己想像。他們的想像比你的描述更恐怖

4. **ch144 不要在本章解決 89% 的問題**。89% 是伏筆，不是炸彈。它在 ch144 只是「被發現」。真正的爆炸在 Vol 3。如果在 ch144 就討論「語青是不是人」——太早。讓不安停留在空氣裡

5. **ch145 阿嬤不要超過三句話**。她的力量在於「少」。如果讓她說太多——她就變成了一個「角色」。她不是角色。她是一碗豆漿

6. **ch146 嚴柏翰的決心不要用鋪墊**。不要寫他「想了很久」「經過內心掙扎」。他沉默了一整天（用一句話帶過）。然後站起來。說了一句話。決心的力量來自「突然」——即使它不是突然的

7. **ch147 米亞的背包問題解釋不要超過 200 字**。「你的大腦是一個背包。容量有限。每段記憶有重量和價值。你必須選擇。」四句話。讀者已經理解了。剩下的讓 ch148 的具體秤量來教

8. **ch148 秤量記憶的順序不可打亂**。案件 → 獎章 → 技能 → 同事 → 獨處。從「最容易放棄」到「最難放棄」。如果先寫同事再寫案件——情感遞進會斷裂

9. **ch149 茉莉花的選擇不要讓嚴柏翰「理解」它**。他不知道茉莉花是什麼。不知道跟誰有關。他只是被手帶著選了它。如果他在 ch149 就說出「這是邱韻如的味道」——ch152 的記憶恢復就失去了衝擊力。他不知道。他的手知道。保持這個斷裂

10. **ch150 不要在迷宮入口就給讀者看到什麼**。維倫進入嚴柏翰的記憶空間。黑暗。層層封閉。然後 = 章節結束。不要在 ch150 就開始第一層的剝離——那是 ch151 的事。ch150 只有「他開始走」。一步。然後讀者被留在黑暗裡
