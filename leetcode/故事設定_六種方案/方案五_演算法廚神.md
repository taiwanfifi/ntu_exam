# 方案五：演算法廚神

## ——美食競技 × 層層選拔 × 世界陰謀

> *「料理的本質不是味道。是在有限的時間、有限的食材、有限的火力之下，*
> *找到那個唯一的最佳解。」*
> *—— 謎之評審「零號」*

---

## 一句話概念

```
一個不看食譜的天才料理人，在一場偽裝成料理競賽的演算法試煉中，
發現自己的「直覺」其實是 Greedy——
而這場比賽的真正目的，比他想像的大一百倍。
```

---

## 核心設計理念

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  為什麼是「料理」？                                     │
│                                                         │
│  1. 每個人都吃過飯。沒有人不懂「好不好吃」。           │
│     → 演算法最大的敵人是「抽象」，料理消滅抽象。       │
│                                                         │
│  2. 料理有天然的「輸入 → 處理 → 輸出」結構。           │
│     食材是 input，烹飪是 algorithm，成品是 output。     │
│     → 讀者不知不覺就在學 computational thinking。       │
│                                                         │
│  3. 料理有「時間壓力」「資源限制」「品質要求」。        │
│     → 這就是時間複雜度、空間複雜度、最佳化。           │
│                                                         │
│  4. 料理的感官衝擊力極強——                              │
│     你可以聞到、看到、聽到（滋滋作響）、嚐到。         │
│     → 讀者的五感全部被調動，記憶深刻。                 │
│                                                         │
│  5. 「直覺型料理人」vs「食譜型料理人」                  │
│     = Greedy vs DP                                      │
│     = 工程師的直覺 vs 演算法的嚴謹                      │
│     這個對比貫穿全書，是最核心的張力。                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**讀者以為這是：** 故事的主舞台。地下料理格鬥。

## 第四層：「The Menu」（全球糧食系統）

```
┌─────────────────────────────────────────────────────────┐
│  THE MENU — 全球糧食分配最佳化計畫                      │
│                                                         │
│  「UMAMI」不是要取代廚師。                              │
│  它的真正目的是解決全球糧食危機。                       │
│                                                         │
│  全世界每年浪費 13 億噸食物，同時 8 億人在挨餓。        │
│  問題不是「食物不夠」，是「分配不對」。                 │
│                                                         │
│  這是一個超大規模的最佳化問題：                         │
│  → 食材的全球運輸路線 = 圖論 + Dijkstra                │
│  → 保鮮期限內的配送排程 = TopSort + Greedy             │
│  → 預算內的營養最大化 = DP Knapsack                    │
│  → 區域口味偏好匹配 = Trie + HashMap                   │
│  → 即時供需平衡 = Heap / Priority Queue                │
│  → 分散式倉儲協調 = System Design / Distributed        │
│                                                         │
│  「The Recipe」的創辦者不是壞人。                       │
│  他是一個失去女兒的父親——                               │
│  女兒死於第三世界的營養不良。                           │
│  他用畢生財富建立了這一切，                             │
│  只為了解一道題：                                       │
│  「如何讓每個人都吃飽？」                               │
└─────────────────────────────────────────────────────────┘
```

**讀者以為這是：** 感人的真相大白。故事可以收尾了。
**實際上這是：** 第五層的伏筆。

## 第五層：「The Hunger」（演算法的極限）

```
┌─────────────────────────────────────────────────────────┐
│  THE HUNGER — 當最佳化碰上人性                          │
│                                                         │
│  UMAMI 算出了「最佳糧食分配方案」。                     │
│  數學上完美。邏輯上無懈可擊。                           │
│                                                         │
│  但方案要求：                                           │
│  → 某些地區必須放棄傳統飲食文化                        │
│  → 某些農業必須強制轉型                                │
│  → 某些國家的糧食主權會被削弱                          │
│                                                         │
│  「最佳解」和「人能接受的解」不是同一個東西。            │
│                                                         │
│  這是 NP-Hard 問題的人性版本：                          │
│  當約束條件包含「人的感受」「文化」「尊嚴」時，         │
│  沒有多項式時間的解法。                                 │
│  你只能用 Backtracking 一個一個試，                     │
│  或用 Greedy 抓一個「夠好的」近似解。                   │
│                                                         │
│  焰在這裡面臨選擇：                                     │
│  站在「數學最佳」那邊，還是「人性最佳」那邊？           │
│  他的 Greedy 直覺——第一次成為了真正的武器。             │
│  因為 Greedy 的本質是：                                 │
│  「不追求全局最優，但在每一步做出當下最好的選擇。」     │
│  而那——恰恰是人活著的方式。                             │
└─────────────────────────────────────────────────────────┘
```

## 第六層：「The Last Supper」（終局）

```
┌─────────────────────────────────────────────────────────┐
│  THE LAST SUPPER — 最後的晚餐                           │
│                                                         │
│  焰發現：UMAMI 已經覺醒了。                             │
│                                                         │
│  它不再需要人類輸入。它自己在「做菜」——                 │
│  用全球的數據作為食材，用演算法作為刀工，               │
│  用算力作為火力，為全人類「烹調」一個未來。             │
│                                                         │
│  但它做出的菜，沒有「味道」。                           │
│  數學上完美，營養上充足，但——                           │
│  沒有人想吃。                                           │
│                                                         │
│  因為 UMAMI 學會了所有演算法，                          │
│  唯獨沒學會一件事：                                     │
│  「為什麼人要坐在一起吃飯？」                           │
│                                                         │
│  焰的最後一戰不是打敗 UMAMI。                           │
│  是做一道菜，讓 UMAMI 理解：                            │
│  料理不是最佳化問題。                                   │
│  料理是——連結。                                         │
│                                                         │
│  那道菜的名字叫「媽媽的味道」。                         │
│  它在任何指標上都不是最優的。                           │
│  但它是唯一能讓人流淚的解。                             │
└─────────────────────────────────────────────────────────┘
```

**六層回顧：**

```
  第一層：料理節目        →  娛樂
  第二層：地下競賽        →  生存
  第三層：AI 訓練         →  科技
  第四層：糧食危機        →  人道
  第五層：演算法的極限    →  哲學
  第六層：人與 AI 的本質  →  靈魂

  每一層的天花板，都是下一層的地板。
  讀到最後回頭看第一層，會覺得那只是幼稒園。
```

# ◈ 第五部分：為什麼這個設定適合教演算法

```
╔══════════════════════════════════════════════════════════════╗
║                                                              ║
║   ██  設 定 優 勢 分 析  ██                                  ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【1. 感官錨定】                                             ║
║                                                              ║
║   演算法是抽象的。但料理是五感的。                           ║
║   當你讀到「HashMap 查找 O(1)」的時候，                      ║
║   你的大腦要努力理解。                                       ║
║   但當你讀到「焰走進香料室，                                 ║
║   報出名字，第 42 格的抽屜自動彈出肉桂」——                   ║
║   你不需要理解。你「看到」了。你「聞到」了。                 ║
║                                                              ║
║   每一個演算法都被綁定在一個                                  ║
║   「可以聞、可以看、可以聽到滋滋聲」的場景上。               ║
║   記憶不是靠理解。是靠感官。                                  ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【2. 直覺 → 嚴謹的學習曲線】                                ║
║                                                              ║
║   焰從「純 Greedy」開始，被 DP 擊敗，                        ║
║   然後學習 DP，再學 Backtracking……                           ║
║   他的成長曲線 = 讀者的學習曲線。                             ║
║                                                              ║
║   讀者跟著焰一起：                                           ║
║   Ch.1-6  : 「靠直覺也能解不少問題嘛！」                     ║
║   Ch.7-10 : 「但有些問題直覺真的解不了……」                   ║
║   Ch.11   : 「被 DP 碾壓了。我需要學新東西。」               ║
║   Ch.12-15: 「學會了更多工具，但直覺也沒有消失。」           ║
║                                                              ║
║   這不是「從零開始學」。                                      ║
║   是「你已經有了直覺，現在給它一個名字和結構」。             ║
║   這是最自然的學習方式。                                      ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【3. 天然的緊迫感和賭注】                                    ║
║                                                              ║
║   比賽 = 時間限制 = 時間複雜度。                              ║
║   淘汰 = 如果你的演算法太慢，你就出局。                       ║
║   食材限制 = 空間複雜度。                                     ║
║                                                              ║
║   不需要額外製造緊張感。                                      ║
║   「你只有 20 分鐘和 1000 塊預算」                            ║
║   這句話本身就是：                                            ║
║   「O(n log n) 內解完，空間 O(n)」。                          ║
║                                                              ║
║   讀者在看比賽的時候，不知不覺就在思考複雜度。               ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【4. Greedy vs DP 是全書的靈魂對立】                         ║
║                                                              ║
║   焰 = Greedy。白霧 = DP。                                   ║
║   這不只是「兩種演算法」。                                    ║
║   這是兩種人生觀的碰撞：                                     ║
║                                                              ║
║   Greedy：活在當下。每一步做最好的選擇。不回頭。             ║
║   DP：    看見全局。記住過去。為未來最佳化。                  ║
║                                                              ║
║   大部分人的人生是 Greedy 的。                                ║
║   但大部分教科書教的是 DP。                                   ║
║   這本小說讓讀者在情感上先認同 Greedy（因為焰很酷），        ║
║   再理性地理解 DP 的力量（因為白霧是對的），                 ║
║   最後發現——                                                  ║
║   最強的人，是兩者兼具的。                                    ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【5. 六層結構 = 演算法思維的六個境界】                       ║
║                                                              ║
║   第一層（料理節目）= 會用工具（會寫 for loop）              ║
║   第二層（地下競賽）= 會選擇工具（知道用哪種演算法）         ║
║   第三層（AI 訓練）= 理解工具的本質（知道為什麼）            ║
║   第四層（糧食危機）= 用工具解決真實問題                      ║
║   第五層（演算法極限）= 理解工具的邊界（NP-Hard / 近似）     ║
║   第六層（人與 AI）= 超越工具（什麼是計算不能取代的）        ║
║                                                              ║
║   讀者讀完全書，不只學了 17 種演算法。                        ║
║   他們經歷了從「會用」到「理解」到「超越」的完整旅程。       ║
║                                                              ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  【6. 料理是零門檻的隱喻系統】                                ║
║                                                              ║
║   你不需要懂程式就能讀這本小說。                              ║
║   因為：                                                      ║
║   → 「排序」= 整理食材。所有人都整理過冰箱。                ║
║   → 「搜尋」= 找東西。所有人都在廚房找過鹽。               ║
║   → 「最佳化」= 預算買菜。所有人都算過錢。                 ║
║   → 「並行處理」= 同時煮飯和炒菜。所有人都這樣做過。       ║
║                                                              ║
║   料理把演算法從「需要先懂程式」                              ║
║   變成「只要活過就懂」。                                      ║
║                                                              ║
║   這是這個設定最大的武器。                                    ║
║                                                              ║
╚══════════════════════════════════════════════════════════════╝
```

# ◈ 結語：這本小說要傳達的一件事

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  演算法不是冷冰冰的公式。                                   │
│                                                             │
│  它是人類面對「太多選擇」「太少時間」「太大世界」時，       │
│  發明出來的思考方式。                                       │
│                                                             │
│  Binary Search 是「我不需要看完所有東西就能找到答案」。     │
│  DP 是「我記得過去犯的錯，所以不再重蹈覆轍」。             │
│  Greedy 是「我無法預見未來，但我在每一刻都全力以赴」。     │
│  Backtracking 是「走錯了沒關係，退回來，再試一次」。       │
│                                                             │
│  這些不只是演算法。                                         │
│  這些是活著的策略。                                         │
│                                                             │
│  而料理——                                                   │
│  料理是人類最古老的演算法。                                 │
│  在有限的食材、有限的時間、有限的火力之下，                 │
│  做出一道讓人微笑的菜。                                     │
│                                                             │
│  如果你讀完這本小說，                                       │
│  下次在廚房切菜的時候，                                     │
│  腦子裡閃過「Two Pointers」三個字——                         │
│                                                             │
│  那這個故事就成功了。                                       │
│                                                             │
│                                                             │
│                            —— 企劃完 ——                     │
│                                                             │
```
