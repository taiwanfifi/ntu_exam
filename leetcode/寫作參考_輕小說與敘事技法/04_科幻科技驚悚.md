# 科幻 / 科技驚悚 — 硬核科學 + 技術概念嵌入敘事

> 這些作品示範了如何將硬核科學/技術概念嵌入引人入勝的敘事中。

---

## 1. 《三體》(The Three-Body Problem) ⭐⭐

| 項目 | 內容 |
|------|------|
| **作者** | 劉慈欣 |
| **類型** | 硬科幻 |
| **規模** | 三部曲：《三體》(2006)、《黑暗森林》(2008)、《死神永生》(2010)，共 ~88 萬字 |
| **獎項** | 雨果獎（2015，亞洲首位） |
| **維基** | https://zh.wikipedia.org/zh-tw/三体_(小说) |

### 寫作技法分析

**硬科學 = 劇情引擎**
- 軌道力學、量子物理、宇宙學不是裝飾，而是推動劇情的核心動力
- 「三體問題」本身就是一個著名的不可解物理/數學問題——小說把抽象問題變成了生存危機

**「黑暗森林」理論 = 賽局理論**
- 這本質上是一個囚徒困境的宇宙版本
- 讀者在享受太空歌劇的同時，不知不覺學會了賽局理論的核心概念

**後人類敘事突破**
- 劉慈欣作為工程師出身的作家，反覆「毀滅又重建」世界觀
- 常識和邏輯被刻意扭曲——迫使讀者重新審視基本假設

**文革背景的情感錨點**
- 在真實歷史創傷中植根科幻設想，創造情感真實性

### 可借鑑的技法
> **「科學概念 = 劇情引擎」是最核心的借鑑**：
> - 「三體問題」→ 換成「NP-Complete 問題」
> - 「黑暗森林理論」→ 換成「演算法賽局」（兩個AI對弈，用的演算法策略決定生死）
> - 劉慈欣的方法：把抽象學問變成生存問題。

## 2. Cryptonomicon ⭐⭐ 密碼學小說之王

| 項目 | 內容 |
|------|------|
| **作者** | Neal Stephenson |
| **類型** | 歷史小說 / 技術驚悚 |
| **規模** | 100+ 節，雙時間線 |
| **平台** | Avon Books (1999) |
| **維基** | https://en.wikipedia.org/wiki/Cryptonomicon |
| **Goodreads** | https://www.goodreads.com/book/show/816.Cryptonomicon |

### 寫作技法分析

**小說裡嵌入真正可運行的算法**
- 包含一個真正的撲克牌密碼系統（Solitaire cipher），由密碼學家 Bruce Schneier 應 Stephenson 之邀設計
- 小說中甚至有一段 Perl 加密腳本
- 讀者可以真的拿書裡的方法來加密訊息

**雙時間線結構**
- 二戰密碼破譯者（包括虛構的圖靈）← 1940s
- 他們的孫輩建立離岸數據天堂 ← 1990s
- 兩條線都圍繞密碼學展開

**數學概念的敘事類比法**
- 質數、模運算、Van Eck phreaking 等概念都透過故事類比來解釋
- 不是「現在我來教你質數分解」，而是角色在生死關頭需要用到質數分解

### 可借鑑的技法
> **「在小說中嵌入可運行的程式碼」是我們可以直接學的**：
> - 每個章節的演算法不只是概念，而是真正可以跑的 Python 程式碼
> - 讀者可以邊讀邊開 terminal 實際執行
> - 我們已經在「演算法偵探筆記」中做了類似的事（pseudo code），可以更進一步

## 3. Digital Fortress（數位乘碼）

| 項目 | 內容 |
|------|------|
| **作者** | Dan Brown |
| **類型** | 科技驚悚 / 密碼學 |
| **規模** | 128 章（超短章節！） |
| **平台** | St. Martin's Press (1998)；中譯本可得 |
| **維基** | https://en.wikipedia.org/wiki/Digital_Fortress |

### 寫作技法分析

**128 個超短章節 = 停不下來的節奏**
- 每章只有 2-4 頁
- 讀者總覺得「再看一章就好」，然後一口氣讀完整本
- **Dan Brown 的專利節奏法**

**密碼學概念嵌入動作場景**
- 公鑰加密、質因數分解、暴力破解等概念在追逐和緊急情況中被解釋
- 不是靜態教學，而是「如果你不懂這個概念，主角就會死」

### 可借鑑的技法
> **超短章節法**：
> 每個「演算法概念」可以是一個 2-4 頁的迷你章節。
> 讀者的心理：「才兩頁，再看一章」→ 不知不覺學完 10 個概念。

## 4. Warcross（戰爭遊戲）

| 項目 | 內容 |
|------|------|
| **作者** | Marie Lu |
| **類型** | YA 科幻 / 駭客小說 |
| **規模** | 31 章（第一部）；續作 Wildcard 也是 31 章 |
| **平台** | G.P. Putnam's Sons (2017) |
| **獎項** | NYT 暢銷書；Kirkus 2017 最佳青少年書 |
| **Goodreads** | https://www.goodreads.com/book/show/41014903-warcross |

### 寫作技法分析
- **駭客技能 = 超能力**：程式設計能力被呈現為一種令人嚮往的力量
- **VR 錦標賽結構**：天然的「關卡」遞增結構
- **年輕的女性駭客主角**：打破「程式設計 = 宅男」的刻板印象

### 可借鑑的技法
> **讓「會寫程式」看起來很酷**：
> 學姊的 ICPC 金牌設定已經在做這件事了。繼續強化。

## 5. Ready Player One（一級玩家）

| 項目 | 內容 |
|------|------|
| **作者** | Ernest Cline |
| **類型** | 科幻 / 反烏托邦 / 遊戲 |
| **規模** | 39 章，3 個 Level |
| **獎項** | NYT 暢銷書；Spielberg 改編電影 |
| **維基** | https://en.wikipedia.org/wiki/Ready_Player_One |

### 寫作技法分析
- **「知識挑戰」= 劇情推進**：主角必須精通特定領域的知識才能前進
- **彩蛋獵人結構**：解謎 → 獲得線索 → 下一關
- **學習動機的終極示範**：在這個世界裡，「有知識」= 「有權力」

### 可借鑑的技法
> **「知識 = 力量」的世界觀設定**：
> 在我們的故事裡，「懂演算法」= 「能破案」。
> 越難的案件需要越進階的演算法知識。
> 這就是 Ready Player One 的結構。

## 6. Flatland 平面國

| 項目 | 內容 |
|------|------|
| **作者** | Edwin A. Abbott |
| **類型** | 數學小說 / 諷刺 |
| **規模** | 2 部分，~22 節 |
| **年代** | 1884 年（！）出版至今仍在版 |
| **免費** | https://www.gutenberg.org/ebooks/201 |
| **維基** | https://en.wikipedia.org/wiki/Flatland |

### 寫作技法分析
- **「漸進式維度擴展」**：0D → 1D → 2D → 3D，每一步都建立在前一步之上
- **主角的「不可置信」= 讀者對新概念的抗拒**：Square 看到三維世界時的震驚，就是學生第一次理解遞迴時的感覺
- **140 年前的作品，至今仍是數學教育的經典**

### 可借鑑的技法
> **漸進式概念擴展的完美範本**：
> - O(n) 的世界 = Flatland（線性、平面）
> - O(log n) 的世界 = 發現了「維度」的秘密
> - O(1) 的世界 = 直接跳到目標（像 hash map）
> - 每個新概念 = 進入一個新的「維度」

## 7. 《質數的孤獨》

| 項目 | 內容 |
|------|------|
| **作者** | Paolo Giordano（理論物理學博士） |
| **類型** | 文學小說 / 數學隱喻 |
| **規模** | ~15 章，跨越數十年 |
| **獎項** | 義大利 Premio Strega 文學獎（最高榮譽） |
| **博客來** | https://www.books.com.tw/products/CN10740337 |

### 寫作技法分析
- **單一數學概念撐起整部小說**：「孿生質數」——靠近但永遠無法相鄰的數字——作為兩個主角關係的核心隱喻
- **數學概念 = 情感結構**：不只是「用數學做比喻」，而是數學本身就是情感的結構
- **作者本身是物理學博士**，確保數學隱喻的準確性

### 可借鑑的技法
> **「一個概念，一種情感」**：
> - Binary Search = 在混亂中尋找分界線（做決定的焦慮）
> - Hash Map = 每個人都有一個獨一無二的位置（歸屬感）
> - Recursion = 回到過去的自己（成長）
> - Stack = 越疊越高的壓力，只有一個出口（釋放）
