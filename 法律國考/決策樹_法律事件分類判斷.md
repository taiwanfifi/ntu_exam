> 這份講義寫給有理工、程式設計或 ML 背景的非法律轉職考生。

## 第一章：Master Decision Tree — 事件發生後的第一步判斷

### 1.1 整棵決策樹的全貌

法律人拿到一個案件，腦子裡跑的分類器長這樣：

```
event_occurred(事件)
│
├─ Step 1: classify_domain(事件)
│   ├─ involves_state_vs_person(事件) == True
│   │   ├─ 公法領域 (Public Law)
│   │   ├─ is_criminal_punishment(國家行為) == True
│   │   │   └─ return 刑法 + 刑事訴訟法
│   │   └─ is_criminal_punishment(國家行為) == False
│   │       └─ return 行政法
│   │           ├─ is_administrative_disposition(行為) → 訴願 → 行政訴訟
│   │           ├─ is_factual_act_causing_damage(行為) → 國家賠償
│   │           ├─ is_administrative_contract(行為) → 行政契約爭議
│   │           └─ is_administrative_legislation(行為) → 法規命令審查 / 釋憲
│   └─ involves_state_vs_person(事件) == False
│       ├─ 私法領域 (Private Law)
│       ├─ involves_business_org(事件) → 商事法（公司/票據/保險/證交）
│       ├─ involves_employment(事件)  → 勞動法
│       ├─ involves_family(事件)      → 民法親屬繼承編
│       └─ default → 民法 + 民事訴訟法
│
├─ Step 2: check_multi_trigger(事件)
│   └─ 同一事件是否同時觸發多個法律體系？
│       └─ 幾乎總是 YES → 進入「多重法律分析」（第二章）
│
└─ Step 3: determine_procedure(目標結果)
    └─ 根據你想要的結果，選擇對應的程序（第五章）
```

### 1.2 公法 vs 私法的分界（Root Node）

**判斷標準：這個法律關係裡面，有沒有「國家公權力」出場？**

```python
def classify_domain(event):
    parties = extract_parties(event)
    if any(is_state_authority(p) for p in parties):
        if state_acting_as_private_entity(event):
            return "私法"   # 例外：國家用「私人身分」行為
        return "公法"
    return "私法"
```

**關鍵 edge case：**

| 情境 | 表面上看 | 實際分類 | 原因 |
|------|---------|---------|------|
| 台電跟你簽電費契約 | 國家 vs 人民 | 私法 | 供電契約是私法行為 |
| 公立醫院看病 | 國家 vs 人民 | 私法 | 醫療行為非行使公權力 |
| 健保局核定給付 | 國家 vs 人民 | 公法 | 健保局在行使公權力 |
| 公立學校退學處分 | 國家 vs 人民 | 公法 | 影響受教育權（釋字382） |

想像「國家」這個 class 有兩個 mode：`sovereign`（行使公權力→公法）和 `private`（作為私人主體→私法）。同一個機關在不同 mode 下做的事，分類不同。

### 1.3 公法内部再分類

```python
def classify_public_law(event):
    if involves_criminal_penalty(event):
        return "刑法 + 刑事訴訟法"
    return classify_admin(event)

def classify_admin(event):
    action = identify_action_type(event)
    mapping = {
        "行政處分": "訴願 → 行政訴訟",           # 具體個別的公權力決定
        "事實行為": "國家賠償法",                  # 警察開槍、施工損鄰
        "行政契約": "行政訴訟法§8 一般給付訴訟",   # BOT契約
        "法規命令": "法規命令審查 / 釋憲",
        "行政指導": "原則上不具法律效果"
    }
    return mapping.get(action)
```

### 1.4 私法内部再分類

```python
def classify_private_law(event):
    if involves_commercial_org(event):
        if is_listed_company(event):
            return "公司法 + 證券交易法"   # 證交法 override 公司法
        return "公司法 / 票據法 / 保險法"  # 依具體情節
    if involves_employment(event):     return "勞動法"
    if involves_family(event):         return "民法親屬編 + 繼承編"
    if involves_IP(event):             return "智慧財產法"
    return "民法總則 + 債編 + 物權編 + 民事訴訟法"  # default
```

### 1.5 多重觸發是常態（Multi-Label Classification）

法律分類不是 single-label classification，而是 **multi-label classification**。一個事件可以同時被貼上「刑法」「民法」「行政法」「保險法」好幾張標籤。

```python
def analyze_event(event):
    labels = []
    if has_criminal_element(event):    labels.append("刑法+刑訴")
    if has_civil_damage(event):        labels.append("民法+民訴")
    if has_admin_action(event):        labels.append("行政法+行訴")
    if has_insurance(event):           labels.append("保險法")
    if has_labor_element(event):       labels.append("勞動法")
    if has_commercial_element(event):  labels.append("商事法")
    return labels  # 通常 len(labels) >= 2
```

## 第二章：同一事件的多重法律體系分析

### 2.1 案例 1：車禍撞傷人

> 小明開車闖紅燈，撞傷騎機車的小華。小華右腿骨折，住院兩個月。

```
labels = ["刑法+刑訴", "民法+民訴", "保險法", "行政法"]
```

**[Label 1] 刑法：過失傷害罪（刑§284）**
- 構成要件：過失 + 傷害結果 + 因果關係
- 闖紅燈 = 違反注意義務 = 有過失
- 法律效果：一年以下有期徒刑、拘役或十萬元以下罰金
- **告訴乃論**（刑§287）→ 可和解後撤告；告訴期間六個月

**[Label 2] 民法：侵權行為損害賠償（民§184 I 前段）**

```
damage = {
    "醫藥費": 實支實付, "看護費": 住院期間×每日費用,
    "工作損失": 月薪×不能工作月數,
    "勞動能力減損": 有後遺症→霍夫曼計算法,
    "精神慰撫金": 民§195→法院酌定,
    "交通費+車輛維修": 實際支出
}
# 時效：知有損害及賠償義務人時起二年（民§197）
```

**[Label 3] 保險法：強制汽車責任保險**
- **不問過失**（no-fault）→ 不管誰對誰錯，受害人都能請求
- 給付項目：傷害醫療最高 20 萬、殘廢最高 200 萬、死亡 200 萬
- 優先順序：先請求強制險 → 不足再向肇事者民事求償

**[Label 4] 行政法：交通裁罰**
- 闖紅燈 → 道交條例§53 → 罰鍰 1,800-5,400 元（行政處分）

**升級情境：**

```
if victim_dies:    →  過失致死（刑§276）+ 非告訴乃論 + 扶養費殯葬費
if hit_and_run:    →  肇事逃逸（刑§185-4）→ 一年以上七年以下
if drunk_driving:  →  公共危險（刑§185-3）+ 吊銷駕照
```

### 2.2 案例 2：公司董事掏空公司

> A 上市公司董事長甲，透過假交易將公司資金 3 億元匯入自己控制的紙上公司。

```
labels = ["公司法", "證交法", "刑法+刑訴", "民法+民訴"]
```

**[Label 1] 公司法：違反忠實義務（公司§23）**
- 董事負忠實義務及善良管理人注意義務 → 違反者對公司負損害賠償
- 股東可提代位訴訟（公司§214-§215）

**[Label 2] 證交法：特別背信罪（證交§171 I②）**
- 上市公司 → 證交法 override 普通刑法
- 刑度：三年以上十年以下，得併科 1,000 萬至 2 億罰金
- 犯罪所得超過一億 → 加重至七年以上（證交§171 II）

**[Label 3] 刑法：普通背信（刑§342）作為 fallback**
- 五年以下 → 但有證交法特別規定 → 被 override
- 如果不是上市公司 → 才 fallback 回普通背信

**[Label 4] 民法：不當得利返還（民§179）**
- 甲取得 3 億元無法律上原因 → 獨立於侵權行為的另一條請求權

**升級情境：**

```
if fake_accounting:     → 證交§171 I① 財報不實 + 刑法偽造文書
if insider_trading:     → 內線交易（證交§157-1）
if damage_to_investors: → 證交§20 損害賠償 + 投保中心團體訴訟
```

### 2.3 案例 3：員工在公司偷東西

> 倉管人員阿強，利用職務之便，將公司倉庫裡的筆電 10 台搬回家。

**關鍵判斷：竊盜 vs 業務侵占**

```python
def classify_theft(perpetrator, item):
    if has_lawful_possession(perpetrator, item):  # 本來就持有
        if in_course_of_duty(perpetrator):
            return "業務侵占（刑§336 II）"   # 六月至五年
        return "普通侵占（刑§335）"           # 五年以下
    return "竊盜（刑§320）"                    # 本來沒持有→偷拿
```

阿強是倉管 → 對筆電有「業務上持有關係」→ **業務侵占**，非竊盜。

**[Label 1] 刑法：業務侵占（刑§336 II）** → 非告訴乃論（公訴罪）
**[Label 2] 民法：侵權（民§184）+ 不當得利（民§179）** → 請求權競合
**[Label 3] 勞動法：** 解僱依據勞基法§12 I④ → 不需預告、不需資遣費

**升級情境：**

```
if 公務員: → 貪污治罪條例§4（無期徒刑或十年以上）
if 偷營業秘密: → 營業秘密法§13-1（五年以下+罰金）
```

### 2.4 案例 4：網路上罵人

> 小美在 Facebook 公開寫道：「王經理是個騙子，專門騙客戶的錢！」

**關鍵判斷：公然侮辱 vs 誹謗**

```python
def classify_defamation(statement):
    if contains_specific_factual_claim(statement):  # 指摘具體事實
        return "誹謗（刑§310）"        # 「騙客戶的錢」= 具體事實
    return "公然侮辱（刑§309）"         # 「白癡」= 純粹謾罵
```

**[Label 1] 刑法：加重誹謗（刑§310 II）** → 文字散布，二年以下，告訴乃論
- 抗辯：刑§310 III 能證明為真 / 刑§311 善意言論 / 釋字509 真實惡意原則

**[Label 2] 民法：侵害名譽權（民§195）** → 精神慰撫金 + 回復名譽處分
- 刑事無罪 ≠ 民事免賠（民事證明標準較低）

**[Label 3] 憲法：言論自由 vs 名譽權**

```python
def check_defense(statement, speaker):
    if speaker.can_prove_truth(statement):              return "不罰"
    if speaker.has_reasonable_belief_of_truth(statement): return "不罰"  # 釋字509
    return "成立誹謗"
```

**升級：** 職場罵人 → 職場霸凌（職安法§6）；罵公務員 → 言論自由保障較強

```python
def buyer_remedies(severity):
```

### 2.6 案例 6：房東趕房客

> 租約還有一年，房東要收回房子給兒子住，要求房客一個月內搬走。

```
labels = ["民法（租賃）", "租賃專法", "刑法（如果強制）"]
```

**[Label 1] 民法：租賃契約的效力**
- 民§450：定期租賃，期限屆滿前原則上不得終止
- 「給兒子住」不是法定終止事由
- 房東片面終止 → 構成債務不履行 → 房客可請求損害賠償
- 如果房客被迫搬離 → 可請求搬遷費用、租金差額等

**[Label 2] 租賃住宅市場發展及管理條例（租賃專法）**
- 2018 年上路，保障房客權益
- 房東只能在法定事由下提前終止：
  - 房客欠租超過二個月
  - 房客違法使用（毀損、違法轉租等）
  - 房屋有重大修繕必要
- 「給兒子住」不在法定事由內 → 不能終止

**[Label 3] 如果房東自力救濟（強制手段）**

```
房東換鎖 → 刑法強制罪（§304）三年以下 + 民法侵權
房東斷水電 → 民法債務不履行 + 租賃專法§16 罰鍰
房東闖入搬東西 → 刑法侵入住宅（§306）一年以下
→ 房客應對：報警 + 提告強制罪 + 民事損害賠償
```

**處理流程決策樹：**

```
房東想趕房客
├─ 租約有約定提前終止條款？
│   ├─ YES → 按約定走（但仍不能違反強制規定）
│   └─ NO → 不能提前終止
├─ 房客有違約事由？
│   ├─ YES → 依法定程序催告 → 期滿未改善 → 終止
│   └─ NO → 只能等租約到期
└─ 房東硬來？→ 房客報警 + 提告 + 求償
```

### 2.7 案例 7：醫療糾紛（手術失敗）

> 老王動膝蓋手術，術後感染惡化，最終截肢。

```
labels = ["民法", "刑法", "醫療法", "消保法(爭議)"]
```

**[Label 1] 民法：雙軌請求權**

```python
def medical_liability(case):
    """醫療糾紛有兩條平行的請求權路線"""
    route_A = {
        "基礎": "契約責任（民§227 不完全給付）",
        "性質": "醫療契約 = 委任契約",
        "注意": "善良管理人注意義務",
        "時效": "知悉起15年（§125）"
    }
    route_B = {
        "基礎": "侵權責任（民§184 I 前段）",
        "要件": "過失侵害身體健康權",
        "時效": "知悉起2年（§197）→ 比契約短"
    }
    # 兩條路的共同關鍵：醫師有沒有違反「醫療常規」
    return [route_A, route_B]
```

**[Label 2] 刑法：過失致重傷**
- 截肢 = 重傷害（刑§10 IV）
- 如果醫師有過失 → 過失致重傷（刑§284 後段）
- 但醫療行為的過失認定非常嚴格 → 必須是違反醫療常規

**[Label 3] 醫療法§82：醫療刑責明確化**
- 醫療機構及醫事人員因執行業務致生損害 → 以故意或**違反醫療常規**為限
- 這是立法者為醫師設定的保護門檻

**[Label 4] 消保法的爭議**
- 醫療行為是否適用消保法§7 無過失責任？
- 通說及實務：**醫療行為不適用消保§7**
- 理由：醫療行為有高度不確定性，課予無過失責任將導致防禦性醫療

**舉證責任特殊處理：**

```
一般侵權：原告舉證（病患必須證明醫師有過失）
    ↓
但醫療資訊高度不對稱（病歷都在醫院手上）
    ↓
實務見解：適度減輕病患舉證責任
    ├─ 方法1：舉證責任轉換（由醫師證明自己沒過失）
    ├─ 方法2：表見證明（依經驗法則推定過失）
    └─ 方法3：醫師說明義務（民§184 II → 醫療法§63 告知同意）
        └─ 未盡告知義務 → 推定有過失
```

### 2.8 案例 8：公務員違法拒絕核發許可

> 阿國向市政府申請建築許可，已檢附所有文件，承辦人拖三個月不核發，也不說明理由。

```
labels = ["行政法", "國家賠償法", "公務員懲戒"]
```

**[Label 1] 行政法：行政怠惰 → 課予義務訴訟**

```
阿國的救濟路徑：
│
├─ Step 1: 申請（已完成）
│
├─ Step 2: 機關怠惰不作為
│   └─ 滿兩個月不為處分 → 視為拒絕（訴願法§2）
│       └─ 可以提訴願
│
├─ Step 3: 訴願（向上級機關）
│   ├─ 訴願有理由 → 命原機關作成處分
│   └─ 訴願無理由 → 進入行政訴訟
│
└─ Step 4: 課予義務訴訟（行訴§5）
    ├─ §5 I：怠為處分 → 命機關作成處分
    └─ §5 II：駁回申請 → 命機關依判決意旨作成處分
```

**[Label 2] 國家賠償法（如果造成損害）**
- 因遲延核發導致工程延誤 → 阿國有實際損失
- 國賠§2 I：公務員執行職務行使公權力，因故意或過失不法侵害人民權利
- 程序：先向賠償義務機關書面協議 → 協議不成 → 向**民事法院**起訴

**[Label 3] 公務員懲戒**
- 公務員怠於執行職務 → 可能被記過或其他懲處
- 情節重大 → 移送懲戒法院

### 2.9 案例 9：勞資糾紛（欠薪 + 非法解僱）

> 小李工作三年，公司積欠三個月薪水，老闆口頭說「明天不用來了」。

**多重違法分析：**

| 違法事項 | 法條 | 效果 |
|---------|------|------|

## 第三章：法律適用的優先順序規則

### 3.1 核心概念

```
法律適用 ≈ Method Resolution Order (MRO) in OOP
```

當多個法律都「看起來可以用」時，法律系統有明確的解析順序。

### 3.2 規則一：特別法優先於普通法（Class Inheritance Override）

```python
class 普通法:
    def 損害賠償(self): return "民法§184 一般侵權行為"
    def 僱傭關係(self): return "民法§482 僱傭契約"
    def 買賣瑕疵(self): return "民法§354 瑕疵擔保"

class 特別法(普通法):
    def 損害賠償(self): return "消保§7 無過失責任"    # override
    def 僱傭關係(self): return "勞基法（最低工資、工時上限）"  # override
    def 買賣瑕疵(self): return "消保§19 七天鑑賞期"   # override
```

**常見 Override 關係：**

| 普通法（base） | 特別法（derived） | Override 條件 |
|---------------|------------------|--------------|
| 民法 | 消保法 | 消費關係（B2C） |
| 民法 | 勞基法 | 勞動關係 |
| 民法 | 土地法 | 不動產交易 |
| 公司法 | 證交法 | 上市櫃公司 |
| 刑法 | 貪污治罪條例 | 公務員犯罪 |
| 刑法 | 陸海空軍刑法 | 軍人犯罪 |
| 刑法背信罪 | 證交法特別背信 | 上市櫃董監 |
| 行政程序法 | 稅捐稽徵法 | 稅務行政 |

**重點：特別法只 override 它有規定的部分，沒規定的照用普通法。** 就像 Python 的 class inheritance：子類別只覆寫它定義的 method，其餘繼承父類別。

### 3.3 規則二：後法優先於前法（Software Version Update）

```
舊法 = v1.0    新法 = v2.0
如果 v1.0 和 v2.0 衝突 → 以 v2.0 為準（立法者最新意志）
```

**但刑法有重要限制：不溯及既往（罪刑法定原則）**

```python
def apply_criminal_law_version(act_time, old_law, new_law):
    if act_time < new_law.effective_date:
        if new_law.penalty < old_law.penalty:
            return new_law   # 從輕原則（刑§2 I 但書）
        return old_law       # 從舊原則（不溯及既往）
    return new_law           # 行為在新法後 → 用新法
```

### 3.4 規則三：上位法優先於下位法（CSS Specificity）

```
法律金字塔：
Level 0: 憲法     → !important     ← 一切法律不得牴觸
Level 1: 法律     → #id           ← 立法院三讀通過
Level 2: 法規命令  → .class        ← 行政機關依授權發布
Level 3: 行政規則  → element       ← 機關內部規定
Level 4: 自治條例  → inline        ← 地方議會通過
```

衝突解決：下位法牴觸上位法 → 下位法無效。法官可直接拒絕適用違法的法規命令；但不能自己宣告法律違憲，須聲請大法官解釋。

### 3.5 規則四：法律適用的 Fallback Chain

```python
def find_applicable_rule(issue):
    """民法§1-§2 的法源適用順序"""
    if rule := search_statute(issue):           return rule    # 法律明文
    if rule := search_custom(issue):            return rule    # 習慣
    if rule := search_legal_principles(issue):  return rule    # 法理
```

**完整 Resolution Order：**

```
1. 強行規定 → 不能用契約排除（最低工資、法定利率上限）
2. 契約約定 → 當事人自治
3. 特別法任意規定 → 特別法的 default 值
4. 普通法任意規定 → 民法的 default 值
5. 習慣 → 業界慣例
6. 法理 → 誠信原則、比例原則等一般原則
```

這就像程式裡的 config resolution：環境變數 > 命令列參數 > 設定檔 > 預設值。

### 3.6 實戰：判斷法律適用順序的完整流程

```python
def resolve_all_applicable_laws(event):
    """
    面對一個法律事件，完整的法律適用判斷流程
    """
    # Step 1: 列出所有可能適用的法律（multi-label）
    candidates = find_all_applicable_laws(event)

    # Step 2: 檢查特別法與普通法的關係
    for special, general in find_override_pairs(candidates):
        # 特別法有規定 → 用特別法
        # 特別法沒規定 → fallback 回普通法
        candidates.resolve_override(special, general)

    # Step 3: 檢查法律位階
    for higher, lower in find_hierarchy_conflicts(candidates):
        # 下位法牴觸上位法 → 下位法無效
        candidates.invalidate(lower)

    # Step 4: 檢查新舊法
    for newer, older in find_temporal_conflicts(candidates):
        # 後法優先，但刑法注意從舊從輕
        candidates.resolve_temporal(newer, older)

    return candidates.sorted_by_priority()
```

**一個完整的判斷範例：員工在上市公司偷營業秘密**

```
Step 1 候選法律：
  刑§320竊盜 / 刑§336侵占 / 營業秘密法§13-1 / 證交法 / 勞基法§12 / 民§184

Step 2 特別法判斷：
  營業秘密法§13-1 override 刑法竊盜/侵占（針對營業秘密）
  證交法 → 如果涉及重大資訊 → 可能觸發內線交易
  勞基法§12 override 民法僱傭（解僱規定）

Step 3 位階檢查：
  所有候選法律都是法律層級 → 沒有位階衝突

Step 4 新舊法檢查：
  營業秘密法 2013/2020 修正 → 適用最新版本

最終結論：
  刑事 → 營業秘密法§13-1（主要）
  民事 → 民§184 + 營業秘密法§12 損害賠償
  勞動 → 勞基法§12 解僱
```

## 第四章：身分不同，規則就不同 — Polymorphism in Law

### 4.1 核心概念

同一個 method call，因為 object 的 type 不同，行為不同。法律也一樣：**同一行為 + 不同身分 = 不同法律**。

```python

class Doctor(Person):
    def commit_theft(self):
```

### 7.1 五步驟分析法（Five-Step Algorithm）

```python
def analyze_any_legal_event(event):
    # Step 1: 識別當事人（誰 vs 誰？）→ 決定公法 or 私法
    parties = identify_parties(event)
    # Step 2: 識別法律關係（什麼關係？）→ 決定法律領域
    relationships = identify_relationships(parties)
    # Step 3: Multi-label scan（觸發哪些法律？）
    laws = [law for law in ALL_LAWS if law.applies(event)]
    # Step 4: 排序（Resolution Order）
    laws.sort(key=lambda l: (l.specificity, l.date, l.hierarchy), reverse=True)
    # Step 5: 確定程序（走什麼路？）
    procedure = determine_procedure(desired_outcome)
    return {"laws": laws, "procedure": procedure,
            "deadline": check_limitations(laws), "court": find_court(procedure)}
```

### 7.2 法律分類的 Feature Importance

如果把法律分類想成一個 ML model，哪些 feature 對分類結果影響最大？

```
Feature Importance Ranking（法律分類）：

1. 當事人身分          → Importance: ★★★★★
   公務員 vs 一般人 → 直接決定適用哪套法律

2. 行為類型            → Importance: ★★★★★
   侵害身體 vs 侵害財產 vs 侵害名譽 → 不同罪名/請求權

3. 關係類型            → Importance: ★★★★☆
   消費關係/勞動關係/家庭關係 → 決定特別法是否介入

4. 行為人主觀狀態      → Importance: ★★★★☆
   故意 vs 過失 → 影響罪名和刑度

5. 損害結果            → Importance: ★★★☆☆
   輕傷 vs 重傷 vs 死亡 → 影響升級與否

6. 行為場域            → Importance: ★★★☆☆
   網路 vs 實體 / 職場 vs 公共場所 → 影響程序和適用法

7. 標的物性質          → Importance: ★★☆☆☆
```

### 7.3 常見誤區 Checklist

| 常見錯誤 | 正確理解 |
|---------|---------|
| 借錢不還 = 詐欺 | 借錢不還是民事，除非一開始就不打算還 |
| 車禍只是民事 | 同時涉及刑事+民事+保險+行政 |
| 公務員打人 = 普通傷害 | 執行職務中→國賠+刑法傷害 |
| 老闆欠薪只能認了 | 勞基法+勞檢+調解+訴訟 |
| 法律沒規定就不違法 | 刑法是（罪刑法定），民法有習慣和法理補充 |
| 撤告就完全沒事 | 撤刑事告訴 ≠ 放棄民事求償 |

### 7.4 刑事 vs 民事的核心差異速查

很多初學者搞不清楚刑事和民事的差別。用一張表徹底釐清：

| 比較項目 | 刑事 | 民事 |
|---------|------|------|
| 目的 | 處罰犯罪者 | 填補被害人損失 |
| 對立當事人 | 國家（檢察官）vs 被告 | 原告 vs 被告 |
| 證明標準 | 超越合理懷疑（beyond reasonable doubt） | 優勢證據（preponderance of evidence） |
| 舉證責任 | 檢察官（控方） | 原告 |
| 法律效果 | 自由刑、罰金、拘役 | 損害賠償、回復原狀 |
| 可否和解了事 | 告訴乃論可以；非告訴乃論不行 | 隨時可以和解 |
| 律師費用 | 可申請法扶（低收入） | 自己負擔（但勞動訴訟有法扶） |
| 上訴審級 | 三級三審 | 三級三審（小額一審終結） |

**實務上的重要連動：**

```python
def criminal_civil_interaction():
    """刑事和民事經常連動"""
    # 1. 刑事附帶民事（最常見的連動方式）
    #    → 省裁判費、同一法官審理
    #    → 但如果刑事無罪，附帶民事通常會被移送民事庭

    # 2. 刑事判決對民事的影響
    #    → 刑事有罪確定 → 民事法官「原則上」會採認事實
    #    → 但刑事無罪 ≠ 民事不用賠（證明標準不同！）

    # 3. 告訴乃論的「談判籌碼」效果
    #    → 被害人握有「撤告權」→ 可以用來談和解金額
    #    → 這是實務上最常見的策略
```

### 7.5 考試實戰：看到題目的前 30 秒

國考二試拿到一個案例題，你應該在前 30 秒完成的「心理決策樹」：

```
讀完題目
│
├─ 1. 識別當事人身分
│   「這題的人物有誰？各自什麼身分？」
│   → 影響適用法律（第四章的 Polymorphism）
│
├─ 2. 識別事件類型
│   「核心是什麼？侵權？契約？行政？」
│   → 決定主要法律領域（第一章的 Decision Tree）
│
├─ 3. Multi-label 掃描
│   「除了主要法律，還有沒有其他法律被觸發？」
│   → 跨科考點就在這裡（第二章的案例分析）
│
├─ 4. 找出爭點
│   「題目的關鍵問題是什麼？學說/實務有爭議嗎？」
│   → 這是拿分的重點
│
└─ 5. 決定答題架構
    「用什麼順序寫？」
    → 通常：請求權基礎 → 構成要件檢驗 → 法律效果
```

**一個實戰範例：**

> 甲酒後駕車撞傷行人乙，乙送醫後因醫師丙疏忽導致傷勢惡化致死。

30 秒內你的腦中應該跑出：
1. 身分：甲（一般駕駛）、乙（被害人）、丙（醫師）
2. 事件：車禍 + 醫療過失 → 刑法 + 民法 + 保險 + 行政
3. 甲的刑責：公共危險§185-3 + 過失致死§276（因果中斷？）
4. 丙的刑責：業務過失致死§276 + 醫療法§82
5. 爭點：因果關係是否中斷？甲和丙是否共同侵權（民§185）？
