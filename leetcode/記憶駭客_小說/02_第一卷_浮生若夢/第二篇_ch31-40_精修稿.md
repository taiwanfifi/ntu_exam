## 壹、ch31-32 嚴柏翰登場——兩隻刺蝟

### ch31「獵犬」場景設計

**開場節奏**：不要拖。嚴柏翰在第一頁就出現。

```
一輛黑色無標識車停在工作室門口。
車門打開。

穿黑色長風衣。
左眼有一道從眉毛延伸到顴骨的舊疤。

他沒有出示證件。
他報出了維倫最近接手的四個案子的編號。
```

**語言 DNA 對照**：
- 嚴柏翰的台詞：短。硬。結論在前。不用形容詞
  - ✅「你要嘛是傻瓜，要嘛是嫌疑人。」
  - ❌「我觀察你很久了，你的案子引起了我的注意——」（太長、太軟）
- 維倫的反應：不動聲色。修復師見過太多人——不會被權力嚇到
  - ✅「你是來抓我的？」（反問，不退縮）
  - ❌「你是誰？你怎麼知道我的案子？」（太被動）

**ch31 的暗線任務**：
1. 讓讀者回想 ch10、ch13 的「暗處觀察者」—— 這就是那個人
2. 嚴柏翰的疤痕 = 他最重要的謎。不要在這裡解釋來源
3. 「我不相信死人能做這些事。你呢？」—— 周明哲的名字第一次被一個重要角色正面提出

### ch32「不情願的握手」

米亞在兩人之間充當緩衝：

## 貳、ch33-37 委託案 #6「大雪中的殺意」——Binary Search 的最佳舞台

### 結構設計（5 章 = 5 幕完美戲劇弧）

```
ch33：揭露（嚴柏翰脫下面具→ Binary Search 展開）
ch34：阻擋（防禦機制觸發→ 97.3% 匹配 = 同一系統）
ch35：轉向（身體記憶搜尋→ 茉莉花 → ★★★ 哭泣）
ch36：證據（照片牆→ 十年前的鸚鵡標誌）
ch37：重建（紙條→「你愛邱韻如」→ 三人重新評估全局）
```

### ch33 Binary Search 的 AB 線融合

**A 線（技術）**：
```
三十四年的記憶庫。
直接搜索太慢了。

但記憶庫是按時間排序的。

先看中間點——17 年前。太早。
再看 8 年前的中間點。太晚。
縮小到 10~11 年前——

「我把他整個人生對折再對折，
在無數個日夜的縫隙裡，
找到了那個大雪紛飛的瞬間。」
```

**B 線（情感）**：
- 嚴柏翰脫下面具：「我的左眼。這道疤。我不記得怎麼來的。」
- 維倫打開記憶庫的動作 = 嚴柏翰把最脆弱的部分交給一個認識不到一週的人
- 同步點：Binary Search 每次「對折」= 一段被跳過的人生。每跳過一段，嚴柏翰的呼吸都微微變化

**米亞的比喻**（已有，不需新增）：「人生對折再對折」——這個比喻在 ch33 由維倫自己說出，不需要米亞翻譯。因為 Binary Search 的概念在這裡已經完全融入劇情。

**寫作指令**：
1. Binary Search 的過程不超過 300 字。讀者需要的是「對折再對折」的意象，不是步驟
2. 搜尋結果：大雪、模糊的臉、刀、血、悲傷——每個關鍵詞單獨成行
3. 不要在 ch33 就讓讀者猜到模糊的臉是邱韻如——保持懸念

### ch35 ★★★ 身體記憶搜尋——全卷核心母題的確立

**場景結構**（三段式）：

**第一段：方法論（100 字）**
```
如果數位記憶被鎖死了——
還有另一種記憶。

肌肉的反射。
皮膚對溫度的反應。
聞到某種味道時的心跳變化。

這些不存在於記憶晶片裡。
無法被篡改。
```

**第二段：Binary Search 變體（200 字）**
```
大範圍 → 中範圍 → 小範圍

視覺 vs 聽覺 vs 嗅覺 → 嗅覺
花香 vs 食物 vs 化學品 → 花香
玫瑰 vs 茉莉 vs 桂花 → ——

茉莉。

嚴柏翰的心率飆升到 150。
```

**第三段：★★★ 崩潰（150 字以內）**
```
他的手不自覺地握成拳頭。
然後鬆開。
反覆握緊、鬆開。

像是在記得什麼——
握住某個人的手，然後放開。

「我不記得她。」
（停頓。整個工作室安靜得像一座墓。）
「但我的手記得。」
```

**為什麼這個場景是全卷核心**：
- 「身體記得」= 全書母題的首次完整表達
- 之後 ch59 邱韻如的手在空中停住、ch70 十分鐘母女重聚、ch149 茉莉花選擇、ch293 花園澆花——全部呼應這一刻
- 嚴柏翰哭了 = 讀者第一次看到他碎裂。不是慢慢崩塌——是一瞬間

**寫作陷阱**：
1. ❌ 不要描寫嚴柏翰「忍住」然後「忍不住」——直接哭。他不是那種會「忍」的人。他是不知道自己會哭
2. ❌ 不要讓維倫安慰他——維倫什麼都沒說。沉默比任何話都有力
3. ❌ 不要讓語青解釋邱韻如是誰——她只說了「她叫邱韻如」然後就被打斷了（嚴柏翰睜開眼睛）
4. ✅ 「我的手記得」—— 這句台詞獨立成段。前後各留一行空白

### ch36 照片牆——物理證據的力量

```
他拿了一把錘子砸開了那面牆。

牆後面是另一面牆——
上面貼滿了照片。
一個女人的照片。
每一張都是她。

穿實驗服的。
在咖啡廳笑的。
在陽光下回頭的。
```

**寫法規則**：照片牆的描寫用列舉法——每張照片只寫一行。不要形容「她很美」或「嚴柏翰很心痛」。讓照片說話。

**ch36 章末紙條**：
```
「柏翰，如果你找到了這面牆——
記住三件事：
一、你不是探員。
二、你愛邱韻如。
三、鸚鵡計畫是你親手設計的。」
```

**為什麼紙條有力**：因為這是嚴柏翰寫給未來的自己的。他預感到自己會被洗腦——在那之前，他為自己留了一條繩子。

### ch37 三人重新評估——鸚鵡計畫的初衷

```
「鸚鵡計畫的初衷不是篡改。
是保護。
你們想建立一個系統，
讓所有人的記憶都可以被備份和保護。

但有人——從內部——
把它改成了武器。」
```

**寫作指令**：語青透露這個真相時，不要讓她有「終於說出來了」的解脫感。她的語氣應該是平靜的、疲倦的——像重複了太多次。

## 參、ch38 蛋炒飯——★★★ 全卷最溫暖的場景

### 場景結構（四段式）

**第一段：教做飯（300 字）**
- 語青打蛋像做精密手術，敲了七下才裂
- 米亞實時評論黏稠度、蛋殼
- 語青面不改色：「在另一條時間線裡我不會犯這種錯。」
- 米亞：「在另一條時間線裡你也不會做飯。」

**第二段：火柱事件（100 字）**
- 語青把火開到最大 → 火柱
- 嚴柏翰拿滅火器的速度比拔槍快
- 「記憶修復師和深淵探員聯手，只為了做一碗蛋炒飯。」

**第三段：吃飯（150 字）**
- 蛋有點焦、飯有點生
- 四個人圍著小桌子
- ★ 嚴柏翰吃了一口，停頓，又吃第二口：「不難吃。」

**第四段：洗碗（100 字）**
- 語青在洗碗。維倫站在旁邊拿抹布
- 默默地洗
- 「謝謝你教我做飯。在所有的時間線裡——你從來沒有教過我做飯。」

### 寫作指令

1. **「不難吃」= 嚴柏翰弧線中最重要的日常台詞**。不要在這句話前後加解釋。讓讀者自己感受到：這個冷硬的男人用兩個字表達了他所有的信任
2. **蛋炒飯的描寫不超過 50 字**——重點不是食物，是一起吃飯的人
3. **洗碗場景 = 維倫×語青關係的第三次身體接近**（ch7 抓手臂、ch12 背靠著背、ch38 並肩洗碗）。不要有任何身體接觸。兩個人的距離 = 一條抹布的寬度
4. **「你從來沒有教過我做飯」**—— 這句話的力量在於：在所有的時間線裡，他們都在忙著拯救世界。沒有人教過她做飯。這一次不同了
5. **米亞的吐槽不超過 3 句**。這是喜劇和溫暖的段落，不是吐槽專場
6. **阿嬤不出現在 ch38**——她的豆漿段落在 ch57.5。兩個呼吸段不要重複元素

## 陸、演算法融合：Binary Search 的 AB 線

### 兩次 Binary Search 的對比

| | ch33 記憶庫搜尋 | ch35 身體記憶搜尋 |
|---|---|---|
| A 線 | 三十四年記憶→17年前→8年前→10-11年前 | 視覺vs聽覺vs嗅覺→花香→茉莉 |

| 同步點 | 「大雪紛飛的瞬間」= 模糊的臉 | 茉莉 = 心率 150 = 「我的手記得」|

### AB 線通用規則確認

1. ✅ 每 3 段 A 線穿插 1 段 B 線
2. ✅ A 線突破 = B 線情感釋放
3. ✅ 米亞翻譯（ch33 不需要——Binary Search 概念已經完全融入劇情）
4. ✅ ch35 是全卷 AB 線融合的最佳範例——搜尋的對象不是數據，是愛
