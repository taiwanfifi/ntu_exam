## 壹、五個核心角色的語言 DNA

---

### 程維倫

```
語言特徵：
  句式：短。常常不完整。少用形容詞。
  比喻系統：用修理的語言形容一切——
    「她的笑容碎了一個角。」
    「這段關係有三處裂紋。」
    「他的信任需要校準。」
  情緒表達：嘴巴說得越少，手做得越多。
    → 平靜時：正常說話
    → 焦慮時：句子變短
    → 憤怒時：只剩動作
    → 心碎時：一個字——「不。」

  他不說「我愛你」。
  他說「不」（ch214）。
  他說「因為這一次，有妳」（ch233）。
  他為她寫代碼到天亮（ch226）。
  他的手在雨中抓住她（ch299）。

  標誌性口頭禪：
  - 「修得好。」（完成修復後的自我確認）
  - 「……嗯。」（在消化重大信息時）

  禁忌：他不會長篇大論。不會解釋自己的感受。
        如果你寫了維倫說超過三句話，重寫。
```

### 林語青

```
語言特徵：
  句式：精確。像在寫程式碼——沒有冗餘。
  比喻系統：用時間線和數據的語言——
    「在另一條時間線裡，那顆星偏了零點三度。」
    「你的偏移率已經超過我的預測了。」
  情緒表達：藏在事實陳述裡。
    → 她說「你的手心溫度升高了 0.7 度」——她在說「我知道你在害怕」
    → 她說「我記得所有在你身邊看過的東西」——她在說「我愛你」
    → 她說「省略號」——她在說「我不想說再見」

  身體語言比台詞更重要：
  - 耳根微紅（尷尬/害羞）
  - 手指無意識摸小腹（身體記得母親）
  - 指尖透明（存在正在崩塌）

  標誌性口頭禪：
  - 「你每次都會。」（殘留記憶的迴聲）
  - 「在另一條時間線裡……」（她唯一允許自己感性的句式）

  禁忌：她不會用「我覺得」開頭。不會主動談論自己的感受。
        她的感受只通過米亞的數據分析或維倫的觀察才「被看到」。
```

### 嚴柏翰

```
語言特徵：
  句式：軍事化。祈使句。名詞。沒有副詞。
  比喻系統：幾乎不用比喻。他說的都是事實或命令。
    唯一例外：談到邱韻如時，他會用感官描述——
    「她聞起來像茉莉花。」
  情緒表達：通過沉默和動作。
    → 他表達信任的方式是「不難吃。」
    → 他表達痛苦的方式是擦槍（擦三遍以上=崩潰中）
    → 他表達愛的方式是「帶我去見她。」

  標誌性特徵：
  - 「資料不會騙人。」→ 到 Vol 2 被自己的設定反噬
  - 「不難吃。」= 最高讚美
  - 左眼舊疤在陰影裡若隱若現 = 他的過去在注視現在

  禁忌：他不會說「我害怕」。不會解釋自己為什麼做某件事。
        他做了。然後別人理解。或不理解。他不在乎。
```

---

### 米亞（M.I.A.）

```
語言特徵：
  句式：快速、精確、帶百分比。吐槽密度高。
  比喻系統：用技術語言做人文觀察——
    「你們的 CPU 利用率令人遺憾。」
    「你的社交能力跟一塊磚差不多，磚至少還能砌牆。」
  情緒表達：通過計算精度和反應速度。
    → 正常狀態：0.1 秒回應，精確到小數點後四位
    → 困惑/感動：停頓 3.7 秒（對 AI 來說是永恆）
    → 犧牲後（ch218）：慢 0.4 秒——吐槽延遲是她的傷疤
    → 重啟後（ch294）：慢 1.2 秒——但「在乎」最先恢復

  成長的語言標記：
  Vol 1：「理論上，這個反應不在我的出廠設定裡。」（否認）
  Vol 2：「美。」（命名）
  Vol 3：「我在乎你們。不需要找理性的藉口了。」（擁抱）
  Vol 4：「好。那我信你。」（超越——AI 選擇了信念而非數據）

  標誌性口頭禪：
  - 「理論上……」（但實際上她在乎）
  - 數字和百分比穿插在一切對話中
  - 全書最後一個吐槽永遠帶著溫暖

  禁忌：她不會直接說「我愛你」。她用數據和犧牲來表達。
        她的感動永遠有一個「延遲」——那個延遲就是她的人性。
```

### 邱韻如

```
語言特徵：
  句式：優雅、冷靜、有教養。像一把裹了天鵝絨的刀。
  比喻系統：不用比喻。她用的是定義和邏輯——
    「重寫記憶就能消除一切痛苦。」（被植入的信念）
    「什麼是真正屬於我的？」（信念崩塌後的反問）
  情緒表達：靜止。
    → 她崩潰的方式不是哭——是停止動作。
    → 她最痛的時候，是安靜地坐在地上反覆問同一句話。
    → 她做出選擇的方式是站起來，聲音比之前更穩。

  她的三個核心台詞概括了她的整個弧線：
  1.「重寫記憶就能消除一切痛苦。」（被植入的信念）
  2.「什麼是真正屬於我的？」（崩塌）
  3.「這是我自己的選擇。真正的。第一次。」（重生）

  禁忌：她不會求饒。不會道歉。不會解釋。
        她做的事要嘛是計算過的，要嘛是她「第一次」的真實。
```

## 肆、演算法潛移默化準則

```
核心原則：讀者讀完一個場景後，應該「理解」了一個演算法的直覺，
          但不會覺得自己剛剛上了一堂課。

實現方法：

1. 用生死情境引入
   ✗「Greedy 演算法是每一步選擇局部最優解。」
   ✓「六十秒。每一秒他只能救一個人。他不看後面的。只看眼前的。然後救它。」
   → 讀者自然理解了 Greedy，但他們感受到的是心碎。

2. 讓米亞做科普但用吐槽包裝
   ✗「Priority Queue 將元素按優先級排列。」
   ✓ 米亞：「我把所有碎片按『你現在最應該救誰』排了個隊。最上面那個最急。
     你不需要看排隊理由。你只需要看最上面那一個。」
   → 技術概念被情感包裝，讀者學到了 Heap 但覺得自己只是在看搶救。

3. 用比喻而非定義
   ✗「Trie 是一種前綴樹結構。」
   ✓「所有被周明哲標記過的記憶都有一個共同的開頭——
     像一本字典裡所有以『周』字開頭的詞條。」
   → 讀者建立了直覺，不需要知道 Trie 這個詞。

4. 讓角色活出演算法
   ✗ 在旁白中解釋 Backtracking。
   ✓ 維倫在黑暗中追逐語青的碎片——每條路走到底，碰壁就退回分岔點，
     標記已走的路，嘗試下一條。他一個碎片都不想放棄。
   → 讀者跟著維倫「體驗」了 Backtracking。

5. 不要在高潮場景中教學
   高潮時演算法應該是隱形的——像呼吸一樣自然。
   教學放在引入階段（每個演算法的第一次出場）。
   之後的出場只需要一個觸發詞就夠了——
   「米亞把碎片排進了佇列。」
   讀者已經知道佇列是什麼了。不需要再解釋。
```

## 伍、寫作時的十條戒律

```
1. 維倫說超過三句話，砍掉兩句。
2. 語青的感情藏在數據裡，不藏在台詞裡。
3. 嚴柏翰的沉默比對白更有力。
4. 米亞的停頓秒數就是她的心跳。
5. 阿嬤每次出場不超過 200 字。但讀者應該想哭。
6. 演算法像空氣——讀者呼吸了但不覺得在學習。
7. 「身體記得」的場景用觸覺描寫，不用抒情散文。
8. 周明哲出場時，讓讀者先恨他再理解他。
9. 每一章的最後一句必須讓人想翻下一頁。
10. 如果一段台詞刪掉之後場景不受影響——刪。
```

### 場景 1：ch149 三樣東西（嚴柏翰的背包問題）

```
【節奏】極慢。像是在水下說話。
【字數參考】1500-2000 字

  建議結構：
  ① 修復台的冷光。嚴柏翰閉著眼。維倫在旁邊列出所有記憶和它們的「重量」。
  ② 嚴柏翰逐一放棄。不要一次列完——每放棄一個，給讀者 3-4 行的安靜空間。
  ③ 最後三樣的揭露：
     - 第一樣：「跟你的友誼。」——維倫的手停了一下。
     - 第二樣：「辦案的工具。不是為了案子。是因為這是唯一我用自己雙手學會的東西。」
     - 第三樣：不說。沉默。維倫問：「第三樣呢？」
       嚴柏翰吸了一口氣：「一種味道。茉莉花。」
       （他不知道為什麼。讀者知道。這是讓讀者哭而角色不知情的設計。）
  ④ 嚴柏翰最後一句：「你是這十年裡唯一真的東西。」

### 場景 4：ch287 你連壞人都做不好

### 場景 5：ch289-290 省略號

```
【節奏】ch289 = 加速的心碎。ch290 = 完全靜止。
【字數參考】ch289 約 800 字，ch290 約 400 字

  ch289 建議：
  ① 語青握著維倫的手。描寫觸感——「幾乎握不到了。她的手指像光做的。」
  ② 核心台詞：「我把你忘不掉的東西，寫進了你不知道的地方。」
     展開：你的手。你的心跳。你聽到聲音時轉頭的方式。
     ——這段要像在刻碑。每個器官是一行字。
  ③ 「數位記憶會消失。但身體記得的——比演算法更永恆。」
     ——這是全書主題的最終陳述。放在消散前一刻。

  ch290 建議：
  ① 「這不是再見。」「那是什麼？」「省略號。」
     ——三句話佔三行。每行之間留空白。讓讀者自己填充那個停頓。
  ② 她笑了。描寫：半透明的臉上，光和淚同時閃爍。
  ③ 然後她消失了。
     不寫「她消失了」。寫：
     空氣中還有她的溫度。
     維倫的手還維持著握住的姿勢。
     手裡什麼都沒有。
     但手心是熱的。
  ④ 最後一行。章結束。沒有鉤子。只有溫度。

  【陷阱】不要寫「維倫哭了」。他不會哭。他的手會抖。
  【陷阱】不要解釋「省略號」是什麼意思。讀者懂。
```

```
