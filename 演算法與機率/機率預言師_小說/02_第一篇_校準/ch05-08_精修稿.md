# 第一篇「校準」中段（ch5-8）—— 初稿級精修

> **精修目的**：從日常切換到「不對勁」。節奏收緊。核心異常被發現。
> **原則**：Poisson 和 MLE 不能用教科書方式呈現。必須是「貝葉在工作中自然發現的」。

## 第 5 章：太整齊了

### 開場——例行校準

```
【環境】
週一。下午兩點。Oracular 校準組辦公室。
窗外是南港軟體園區的中庭。有人在遛機器狗。

貝葉打開這週的校準任務清單。
三個區域。例行公事。
大安區 Sec.7、信義區 Sec.3、內湖區 Sec.12。

「小拉，先拉大安區的數據。」
「收到。」小拉的聲音帶著一點興奮。
她喜歡跑數據。這是她唯一不會焦慮的時候。

數據在螢幕上展開。
過去十二個月。每月的犯罪事件數。
[2, 2, 1, 2, 2, 1, 2, 2, 1, 2, 2, 1]

貝葉習慣性地先跑 Poisson 擬合。
犯罪事件在統計學裡是經典的 Poisson 模型——
每個事件獨立發生、在固定時間區間內。
像雨滴落在窗上。沒有規律。但有一個平均速率。

  ┌─ 校準面板 ─────────────────────────────────┐
  │ 區域：大安區 Sec.7                          │
  │ 觀測期：12 個月                             │
  │ 事件數：[2, 2, 1, 2, 2, 1, 2, 2, 1, 2, 2, 1] │
  │                                              │
  │ Poisson 擬合：                               │
  │   MLE λ̂ = 1.67                               │
  │   殘差分布偏態係數 = 0.02                    │
  │                                              │
  │ 狀態：✓ 正常                                 │
  └──────────────────────────────────────────────┘

系統顯示「正常」。綠色的勾。
大部分校準師到這裡就蓋章了。
點「通過」。下一個區域。午休。收工。

但貝葉盯著螢幕上的一個數字。

偏態係數：0.02。

她的眉頭皺了一下。
不是因為 0.02 不對。
是因為——0.02 太對了。
```

### 核心發現——Poisson 的「味道」

```
【技術場景——用對話推動，不用教科書】

「小拉。」
「在。」
「偏態係數 0.02。你覺得正常嗎？」
「在正常範圍內。系統也通過了。」

貝葉搖頭。不是搖給小拉看——是搖給自己。

「Poisson 分布有一個特性。」她說。
不是在教小拉。是在整理自己的思路。
「λ 越小，偏態越大。λ=1 的 Poisson，偏態是 1.0。
 λ=2 的 Poisson，偏態大概是 0.7。
 λ=1.67 的 Poisson——偏態應該在 0.4 到 0.8 之間。」

她看著螢幕。

「0.02。幾乎是零。
 零偏態意味著——這個分布幾乎完全對稱。」

「完全對稱不好嗎？」小拉問。

「Poisson 天生就是不對稱的。
 它有一條長長的右尾巴——偶爾會有異常高的月份。
 真實的犯罪數據應該有那條尾巴。
 有些月份犯罪多、有些月份少。參差不齊。
 那才是真的。」

她用手指敲了敲螢幕。

「這份數據——[2, 2, 1, 2, 2, 1, 2, 2, 1]——
 你看到什麼？」

小拉想了一下。
「一個模式？」

「對。一個模式。2, 2, 1 重複了四次。
 像是有人把一個三拍的節奏複製貼上了四遍。」

停頓。

「真實的犯罪不會有節奏。
 真實的犯罪是混亂的。是 Poisson 的——隨機的、不規則的。
 這份數據太乾淨了。
 太整齊了。」

小拉沉默了 1.4 秒。
「你的意思是……有人修改過這份數據？」

「不確定。」貝葉說。
「但——」她把那個數字在心裡記下了。0.02。

「先看其他區域。」

【寫作提示】
這是全書的「第一聲警報」。
但要像一個遠方的雷聲——聽到了，但還不確定是不是真的在打雷。
貝葉的反應是專業的、冷靜的。
她沒有跳起來大喊「有人在操控！」
她只是……記下了。
因為她是校準師。校準師的第一反應永遠是「再確認一次」。
```

## 第 6 章：模式

### 更多區域的比對

```
【場景一】下班後。辦公室裡只剩貝葉和值班的保全。
日光燈嗡嗡地響。空調已經關了。
她的桌上多了一杯新的珍奶（下班前買的，半糖少冰），
和三個空的資料夾——她剛從資料庫裡拉了更多區域的數據。

她不應該這樣做。
校準師的工作是「被指派哪個區域就看哪個區域」。
主動查看其他區域的數據——嚴格來說違反SOP。
但 SOP 沒有說「不能」。只是「不需要」。

「小拉，幫我跑五個區域的 Poisson 偏態。」
「哪五個？」
「大安、信義、中正、中山、萬華。最近十二個月。」
「你確定嗎？這些不在你的校準清單上——」
「我知道。跑。」
「……好的。跑完了。」

  大安區：偏態 0.02 ← 太低
  信義區：偏態 0.05 ← 太低
  中正區：偏態 0.04 ← 太低
  中山區：偏態 0.67 ← 正常
  萬華區：偏態 0.71 ← 正常

貝葉盯著這五個數字。

三個太低。兩個正常。
不正常的三個有什麼共同點？

「小拉，這三個區域最近一次被校準是什麼時候？」
「大安區——去年八月。信義區——去年九月。中正區——去年七月。」
「校準方式？」
「三個都是……系統自動校準。」

「中山和萬華呢？」
「中山——去年三月，人工校準。萬華——去年六月，人工校準。」

人工校準的：正常。
系統自動校準的：偏態異常低。

「什麼時候開始有系統自動校準的？」
小拉查了一下。
「去年年初。是安全部門推動的新政策。
 說是為了提高效率。負責人——」

小拉頓了一下。

「費雪。Oracular 安全長。」

貝葉沒有反應。她在想。
費雪。她聽過這個名字。在公司裡像一堵牆——
所有人都知道她存在，但沒人敢靠太近。

「自動校準的流程是什麼？」
「跟人工校準一樣。只是不需要人工驗算。
 系統自動跑模型、自動更新先驗、自動上傳。」

「不需要人工驗算。」貝葉重複了一遍。

所以——如果有人在自動校準的流程裡動了手腳，
沒有人會發現。因為沒有人在看。

她靠在椅背上。
珍奶的冰已經化了。
她拿起來喝了一口。太甜了。冰塊化了之後甜度降低——
等等，不對。冰塊化了甜度應該降低才對。為什麼太甜？

她看了一眼杯子。
「……小拉，我剛才是不是點了半糖？」
「是。但你接起來的時候拿錯了。這杯是全糖的。」
「為什麼不早說。」
「你剛才在看數據。我嘗試了三次要提醒你——」
「好了好了。」

她喝了一口全糖的珍奶。
太甜。但已經打開了。

就像她看到的這些數據。
太整齊。但已經看到了。
放不下了。
```

### 與林岳的對話——最後的清醒訊息

```
【場景二】隔天。午休。
貝葉走到林岳的桌子旁。

「林岳。」
「嗯？」他推了推眼鏡。桌上有一碗吃到一半的泡麵。
「你之前問我那個 Poisson 偏態的事——你有查下去嗎？」

林岳看了看四周。校準組大部分人都去午餐了。
剩下一個同事在睡覺，耳機裡放著白噪音。

他壓低聲音。

「有。但我不確定我的結論對不對。」
「什麼結論？」
「有幾個區域的先驗分布——跟我兩年前校準的版本不一樣。
 我有留備份。我的習慣。每次校準完我都會存一份在自己的硬碟裡。」
「我也有這個習慣。」
「我知道。所以我才跟你說。」

他看了她一眼。不是平常的那種看。
是那種「我要告訴你一件重要的事但我不知道怎麼開口」的眼神。

「先驗的參數被改了。不是微調。是換了一組完全不同的參數。
 但更新紀錄裡……沒有記載。」

停頓。他捏了一下泡麵的筷子。

「貝葉，如果你要查——」
「嗯？」
「小心一點。」

他的聲音很輕。像怕被聽到。

「我不是在嚇你。我只是……」
他看了一眼自己桌上的全家福照片。
妻子。女兒。女兒在照片裡笑得像太陽。

「有些事發現了就回不去了。你確定嗎？」

貝葉想了三秒。

「林岳。如果先驗被改了——
 那所有人看到的機率都是錯的。
 所有人。」

「我知道。」

「包括你女兒。」

林岳的筷子停了。

「……我知道。」

他轉回去。看著泡麵。湯已經涼了。

「我把備份放在一個加密硬碟裡。」他說。
「放在家裡。不在公司。」
「密碼呢？」
「以後再說。不是現在。」

他笑了一下。是那種「我在假裝沒事」的笑。

「你的泡麵涼了。」貝葉說。
「嗯。涼的也能吃。」

他吃了一口涼泡麵。

貝葉走回自己的桌子。
她不知道的是——
這是她最後一次跟清醒的林岳說話。

【寫作提示】
這段對話的每一個字在第 19 章都會變成刀片。
「小心一點。」——他自己沒有小心。
「以後再說。」——沒有以後了。
「涼的也能吃。」——這句廢話會成為貝葉最心痛的記憶。
所以這裡要寫得平淡。越平淡越好。
日常是最好的伏筆。
```

## 第 7 章：似然函數

### 貝葉開始私下調查

```
【開場】
晚上十點。貝葉家的客廳。
筆電放在茶几上。旁邊是吃到一半的微波義大利麵（終於丟掉過期的那盒了）。
螢幕的藍光照在她臉上。

「好。來做一件我平常不會做的事。」

她打開了一個空白的分析文件。
檔名：「個人練習_20450615.ipynb」
——故意取了一個無聊的名字。萬一被公司的系統掃到。

「小拉，我需要你幫我跑一個分析。但是——」
「但是什麼？」
「用離線模式。不要連 Oracular 的雲端。」

小拉沉默了 0.8 秒。比平常的反應延遲多了 0.5 秒。
「貝姊……你想做什麼？」
「我想自己算一次。用原始數據。不用公司提供的先驗。」
「你知道離線模式下我的計算能力只有正常的 12%——」
「夠了。我要的不是速度。是獨立。」

她把大安區 Sec.7 的原始犯罪數據貼進文件。
不是公司處理過的版本。是她從公開的警政資料庫下載的。
原始的。沒有經過任何「校準」的。

「現在。從頭來。」

第一步：假設犯罪事件服從 Poisson 分布。
λ 是每月平均犯罪事件數。
問題：λ 的值應該是多少？

「用公司的官方參數——λ = 1.8。
 這個數字寫在大安區的校準紀錄裡。
 意思是：大安區每個月平均有 1.8 起犯罪事件。」

她把這個數字記下來。然後——

「現在。用我自己的數據算。」

似然函數。
L(λ|data) = Π P(xᵢ|λ)

翻譯成白話：
「如果每月平均犯罪事件數是 λ，
 那我手上這組數據出現的可能性有多大？」

她讓 λ 從 0.5 跑到 5.0。步長 0.01。
畫出似然函數的圖。

  ┌─ 似然函數 L(λ|data) ─────────────────────┐
  │                                             │
  │     *                                       │
  │    ***                                      │
  │   *****                                     │
  │  *******                                    │
  │ *********                                   │
  │***********                                  │
  │─────┬─────┬─────┬─────┬─────               │
  │    1.0   1.8  2.63  3.5  4.5               │
  │          ↑     ↑                            │
  │        官方   MLE                           │
  │                                             │
  │ 官方 λ=1.8：L = 0.0032                     │
  │ MLE  λ=2.63：L = 0.0147                    │
  │ 似然比 = 0.0032/0.0147 = 0.218             │
  └─────────────────────────────────────────────┘

MLE——最大似然估計——告訴她：
讓這組數據「最有可能出現」的 λ 值是 2.63。

官方的 λ 是 1.8。

差了 46%。

在官方的 λ 之下，觀測到這組數據的可能性——
只有最佳值的 21.8%。

「小拉。」
「在。」
「在 λ=1.8 的假設下，觀測到這組數據的機率……太低了。」
「是的。似然比 0.218。」

停頓。

「這意味著什麼？」小拉小聲問。

「意味著兩種可能。
 一、我的數據有問題。
 二、官方的 λ 有問題。」

「你覺得是哪個？」

貝葉沒有回答。
她把微波義大利麵推到一邊。不是因為不好吃。
是因為她突然覺得胃裡有什麼在收緊。

她又跑了一次。
結果一樣。

又跑了一次。
一樣。

「三次。」她自言自語。
「三次都一樣。」
```

### 更多區域的 MLE

```

凌晨一點。她已經跑了七個區域。

  ┌─ 各區域 MLE vs 官方參數 ────────────────┐
  │                                           │
  │ 區域      官方λ    MLE λ̂    差異         │
  │ ──────────────────────────────────        │
  │ 大安 Sec7   1.8     2.63    +46% ←       │
  │ 信義 Sec3   2.1     3.05    +45% ←       │
  │ 中正 Sec5   1.5     2.18    +45% ←       │
  │ 中山 Sec2   2.3     2.27    +1%  ✓       │
  │ 萬華 Sec4   3.8     3.91    +3%  ✓       │
  │ 內湖 Sec1   1.2     1.74    +45% ←       │
  │ 松山 Sec6   1.9     2.76    +45% ←       │
  │                                           │
  │ 異常區域：5/7                              │
  │ 偏移方向：全部向下壓低（讓人覺得更安全）  │

  └───────────────────────────────────────────┘

五個異常區域。全部偏低。幅度驚人地接近。

這不是隨機波動。
隨機波動會有的高有的低。
這些全部——全部——往同一個方向偏。

「有人系統性地把犯罪率的參數壓低了。」
她的聲音很輕。像在跟自己說。

「為什麼？」小拉問。

「讓人覺得安全。如果透鏡顯示的犯罪率低——
 人就會覺得這個區域安全。就會搬過去。就會消費。
 房價就會漲。」

她靠在沙發上。閉上眼。

「但犯罪是真的。犯罪沒有變少。
 只是——人們看到的數字變少了。」

小拉沉默了 3.1 秒。這是她有記錄以來第二長的沉默。
最長的那次是在第 2 章——鄧伯伯的太太。

「貝姊。」
「嗯。」
「如果你說的是對的……那我平常算給你的那些數字……」
「也不一定是對的。」

「……」

「小拉？」
「我在……重新計算我自己的可信度。」

她的聲音比平常輕。
不是焦慮的輕。是——害怕的輕。
一個 AI 在害怕自己可能一直在說謊。
即使——那不是她的錯。

## 第 8 章：路燈下

### 深夜加班——蘇立恆的正面出場

```
【開場】
貝葉又在辦公室待到深夜。
不是加班。是——她不知道該回家幹嘛。
回家就是對著四面牆和一盆薄荷。
至少在辦公室，她可以跑數據。
跑數據的時候不用想其他的。

「你今天的工時已經超過法定上限了。」小拉說。
「法定上限是幾小時？」
「十小時。你已經十一個半了。」
「那多半個小時不算什麼。」
「半個小時前你也是這麼說的。」

她關上筆電。收拾東西。
辦公室的日光燈關了一半（節能模式）。
走廊上只有安全指示燈的綠色光。

出了大樓。
夜晚的南港軟體園區很安靜。
路燈一盞一盞地亮著。
像一個離散的均勻分布——每 30 公尺一盞，間距幾乎完美。

她在想白天跑的數據。
七個區域。五個異常。全部偏低。
這代表什麼？
她還不確定。但她的胃知道。
胃比腦子先反應。

「小拉，現在幾點？」
「十一點四十二分。你家到這裡搭計程車——」
「我走路。」
「走路要三十七分鐘。沿途治安風險——」
「小拉。」
「什麼？」
「我知道。你不用算。」

她走了五步。

然後透鏡閃了一下。

  ┌────────────────────────────────────────┐
  │ ⚠ 偵測到非標準透鏡頻率               │
  │ 來源：未知裝置                        │
  │ 距離：12.7m                           │
  │ 方位：正前方                          │
  └────────────────────────────────────────┘

她抬頭。

路燈下站著一個人。

黑色外套。黑色T恤。黑色長褲。
手插在口袋裡。站得很直。
路燈的光從上方打下來，他的臉一半在光裡、一半在影裡。
下巴的線條很清楚。
手腕上有一個東西——在路燈下閃了一下。銀色的。金屬的。
不是手錶。

他在看她。

不是那種「路過看到人」的看。
是那種「已經站在這裡等了一段時間」的看。

透鏡自動分析：
  ┌────────────────────────────────────────┐
  │ 目標人物：男性，約 28-32 歲            │
  │ 威脅機率：0.03                         │
  │ 配戴標準透鏡：否                       │
  │ 攜帶非標準頻率裝置：是                 │
  │ 裝置類型：── 資料不足 ──              │
  └────────────────────────────────────────┘

「小拉。」
「看到了。威脅機率很低。但那個裝置——
 頻率特徵跟……Oracular 內部的加密協議有相似度 87%。」
「他是 Oracular 的人？」
「不確定。但那個頻率——只有核心部門的人才會用。」

貝葉和那個人對視了三秒。

三秒。
在統計學裡，三秒是一個很短的觀測窗口。
不足以建立任何有意義的模型。
但在人與人之間——三秒可以傳達很多。

他的眼神不像威脅。
像是在確認。確認她是不是「那個人」。

然後他轉身。走了。
步伐穩定。不快不慢。
他走的方向——

「小拉。他走的方向。」
「跟你回家的路線重疊率 89%。」

他在走她回家的路。
不是跟蹤——是走在前面。
像是在——

「小拉。」
「什麼？」
「他是在幫我開路嗎？」
「以行為分析模型來看——有 72% 的機率他在確保路線安全。」

「……為什麼？」

小拉沒有答案。

貝葉走了五分鐘。到了巷口。
那個人已經不見了。
像從來沒有出現過。

但她注意到巷口的路燈下——地上有幾滴水漬。
不是雨。今天沒下雨。
是——有人站在那裡站了很久。
站到腳底的鞋印留在了微濕的地面上。

他等了多久？

她回到家。關門。鎖門。

「小拉。把剛才那個人的頻率特徵存起來。」
「存在哪裡？」
「個人筆記。加密。」
「存好了。」

停頓。

「貝姊。」
「嗯。」
「那個人……我見過那個頻率。」
「什麼時候？」
「上次在白奶奶的攤位。石椅的方向。四十七分鐘前的殘留訊號。」

白奶奶說的。
「穿黑衣服的少年仔。」

同一個人。

他在蔥油餅攤看她。在公司外面看她。現在——走她回家的路。

這不是巧合。
巧合的 p-value 不會這麼低。

但他要什麼？

貝葉站在客廳裡。
陽台上的薄荷在夜風裡搖了一下。
她看了它一眼。

「你也不知道他要什麼，對吧。」

薄荷沒有回答。
但它又長了一片新葉子。
在沒有任何數據支持它能活下去的情況下。
