## 核心理念與使用方式

公法最容易失分的不是「不會」，而是「搞混」。這份文件把 18 組最易混淆的概念拆開比對，每組含：一眼判斷法表格、混淆原因、深度解析、案例對比、邊界案例、考試陷阱。


# 第一部分：行政行為分類混淆組

> 行政行為的分類是行政法的 type system。分類錯誤 = 後面所有分析全錯。

---

## 混淆組 1：行政處分 vs 事實行為

### 一眼判斷法

```
def classify(行政機關的行為):
    if 行為產生「法律效果」（創設/變更/消滅權利義務關係）:
        return "行政處分"    # 行程法§92
    else:
        return "事實行為"    # 沒有改變法律關係，只有事實上的影響
```

| 判斷基準 | 行政處分 | 事實行為 |
|----------|----------|----------|
| 法律效果 | 有（改變權利義務） | 無（只有事實影響） |
| 法律依據 | 行程法 §92 | 無統一定義 |
| 救濟方式 | 訴願 → 撤銷訴訟 | 一般給付訴訟 / 國賠 |
| 比喻 | git commit（改變了 repo 狀態） | git log（只是查看，不改變狀態） |

### 混淆原因

兩者都是「行政機關做了某件事」。差別只在於有沒有產生「法律效果」。但「法律效果」是個抽象概念，很多行為落在灰色地帶。

### 深度解析

**行政處分的四要素（行程法 §92 I）：** (1)行政機關的行為 (2)就公法上具體事件 (3)所為之決定或公權力措施 (4)對外直接發生法律效果。四要素缺一不可，事實行為通常缺第 4 個。

### 案例對比：微調一個條件翻轉結論

| 場景 | 分類 | 關鍵差異 |
|------|------|----------|
| 環保局「命令」工廠停工 | 行政處分 | 產生法律效果：工廠的營業權被限制 |
| 環保局「派員」到工廠採集水樣 | 事實行為 | 沒有法律效果：只是物理上的採樣動作 |
| 環保局公布空氣品質「警報」 | 事實行為 | 只是資訊提供，沒有改變任何人的權利義務 |
| 環保局公布空氣品質「命特定工廠減量」 | 行政處分 | 產生法律效果：特定工廠的排放權被限制 |

### 邊界案例

**交通號誌設置 = 處分還是事實行為？** 通說：號誌「設置」是一般處分（§92 II 對物），因產生遵守義務。但號誌「壞掉不修」是事實行為（不作為），走國賠 §3。

**道路命名？** 命名本身是事實行為（觀念通知），路名不改變權利義務。但因路名變更導致的門牌重編通知 → 行政處分。

### 考試陷阱

- 陷阱 1：「行政機關在公告欄張貼公告」不一定是事實行為。要看公告的內容：如果是「公告某地為禁建區」，那是一般處分；如果是「公告天氣預報」，那是事實行為。
- 陷阱 2：行政指導（行程法 §165）是事實行為，因為不具法律上拘束力。但如果行政指導實質上具有強制效果（例如不配合就刁難你），實務上可能被認定為行政處分。

## 混淆組 2：行政處分 vs 行政契約

### 一眼判斷法

```
def classify(行政行為):
    if 政府和人民之間有「合意」（雙方同意）:
        return "行政契約"    # 行程法§135以下
    else:
        if 政府「單方面」對人民做出決定:
            return "行政處分"  # 行程法§92
```

| 判斷基準 | 行政處分 | 行政契約 |
|----------|----------|----------|
| 意思表示 | 單方（政府說了算） | 雙方（需要合意） |
| 人民地位 | 被動接受 | 主動協商 |
| 法律依據 | 行程法 §92 | 行程法 §135-149 |
| 救濟方式 | 訴願 → 撤銷訴訟 | 直接提一般給付訴訟 |
| 比喻 | sudo command（系統管理員強制執行） | npm agreement（雙方同意授權條款） |

### 混淆原因

有些行政行為看起來像契約（需要人民申請、人民簽名），但本質上是處分。關鍵在於人民有沒有「協商餘地」。

### 深度解析

行政契約兩種類型（行程法 §137）：(1)**和解契約**：雙方互相讓步達成合意 (2)**雙務契約**：人民負擔義務，行政機關給予利益。核心判斷：**人民有沒有實質協商空間？** 有 → 契約；沒有 → 處分。

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 申請建築執照，政府核發 | 行政處分 | 你不能跟政府「談」建照的條件，政府依法審查後單方決定 |
| BOT 案（高鐵 BOT） | 行政契約 | 政府和廠商就營運條件、年限等相互協商 |
| 社會住宅租約 | 行政契約 | 政府和承租人就租期、租金等達成合意 |
| 公費醫學生簽約 | 行政契約（通說） | 學生和政府就服務年限、違約金等條件達成合意 |
| 核發醫師執照 | 行政處分 | 符合要件就發照，沒有協商空間 |

### 邊界案例

**公費醫學生 — 契約還是處分附款？** 實務傾向行政契約（學生簽契約書，有一定選擇權）。學說有認為是處分附款（學生無實質協商籌碼）。**考試答法：** 兩說並陳 + 指出關鍵差異：契約 → 一般給付訴訟；處分附款 → 訴願 + 撤銷訴訟。

### 考試陷阱

- 陷阱：「政府和人民簽了一份文件」不代表就是行政契約。如果那份文件只是人民的「申請書」，而政府的「核准」是單方決定，那還是行政處分。
- 陷阱：行政契約不能走訴願。如果題目問「不服行政契約的內容應如何救濟」，答訴願就錯了。

## 混淆組 3：法規命令 vs 行政規則

### 一眼判斷法

```
def classify(行政機關發布的規範):
    if 規範對象是「人民」（對外）:
        return "法規命令"     # 行程法§150
    elif 規範對象是「公務員/下級機關」（對內）:
        return "行政規則"     # 行程法§159
```

| 判斷基準 | 法規命令 | 行政規則 |
|----------|----------|----------|
| 規範對象 | 對外（人民） | 對內（公務員/下級機關） |
| 法律授權 | 需要法律授權 | 不需要（基於上級指揮監督權） |
| 發布方式 | 須刊登政府公報 | 下達即生效 |
| 法律依據 | 行程法 §150 | 行程法 §159 |
| 比喻 | public API（對外公開的介面） | internal docs（內部開發規範） |

### 混淆原因

兩者都是「行政機關發布的抽象規範」，外觀上都長得像一堆條文。差別在於規範的對象是誰。

### 深度解析

**法規命令（§150）：** 行政機關基於法律授權，對不特定人民就一般事項所作抽象之對外規定。關鍵詞：法律授權 + 對外 + 抽象。

**行政規則（§159）：** 上級對下級，規範機關內部秩序及運作之一般性規定。種類：組織性、作業性、裁量基準、解釋性規則。

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 財政部發布「所得稅法施行細則」 | 法規命令 | 基於所得稅法授權，規範人民的納稅義務 |
| 財政部發布「各級稅捐稽徵機關作業手冊」 | 行政規則 | 規範稅務員怎麼工作，不直接對人民 |
| 環保署發布「空氣品質標準」 | 法規命令 | 基於空污法授權，設定對人民有拘束力的標準 |
| 環保署發布「環境影響評估審查委員會組織規程」 | 行政規則 | 規範內部組織運作 |

### 邊界案例：解釋性行政規則的對外效力

**這是考試超級重點。**

解釋性行政規則（例如財政部的稅法函釋）雖然形式上是「對內」的行政規則，但實務上下級機關都照著辦，等於間接影響了人民。

這就產生了「行政自我拘束原則」：

```
if 行政機關一直按照某個行政規則做決定:
    該行政規則產生「間接對外效力」
    行政機關不能突然改變做法（違反平等原則）
    除非有正當理由變更
```

**大法官釋字 287 號**確認：解釋性行政規則雖非法規命令，但如果是對法律的正確解釋，自法律生效時即有其適用；如果是錯誤解釋，嗣後變更時應注意信賴保護。

### 考試陷阱

- 陷阱：「行政規則不需要法律授權」不代表行政規則可以違反法律。行政規則仍受法律優位原則拘束。
- 陷阱：法規命令如果沒有法律授權，不是「無效」，而是「違法」，可能被法院拒絕適用（大法官釋字 524 號）。
- 陷阱：裁量基準形式上是行政規則，但實質上會影響人民，要注意是否逾越母法授權。

## 混淆組 4：行政處分 vs 一般處分 vs 法規命令

### 一眼判斷法：對象確定性光譜

```
def classify(行為):
    if 對象是「特定人」:
        return "個別處分"           # 行程法§92 I
    elif 對象是「可得確定之多數人」or「對物」:
        return "一般處分"           # 行程法§92 II
    elif 對象是「不特定多數人」且內容是「抽象規範」:
        return "法規命令"           # 行程法§150
```

| 判斷基準 | 個別處分 | 一般處分 | 法規命令 |
|----------|----------|----------|----------|
| 對象 | 特定人（張三） | 可得確定之人或對物 | 不特定多數人 |
| 內容 | 具體事件 | 具體事件 | 抽象事項 |
| 存續性 | 通常一次性 | 可能持續 | 持續有效 |
| 救濟 | 訴願→撤銷訴訟 | 訴願→撤銷訴訟 | 法規審查/附帶審查 |
| 比喻 | 針對某 user 的 ban | 針對某 IP range 的防火牆規則 | 全站 Terms of Service |

### 混淆原因

一般處分介於個別處分和法規命令之間。關鍵在「可得確定」：對象不只一人，但可根據具體情境確定。

### 深度解析

一般處分兩種（§92 II）：(1)**對人**：依一般性特徵可得確定之多數人（如公告某路段禁停 → 經過的駕駛人）(2)**對物**：公物的設定/變更/廢止（如指定某建築為古蹟）。

### 案例對比：同一條路的三種行為

| 場景 | 分類 | 理由 |
|------|------|------|
| 罰張三闖紅燈 1800 元 | 個別處分 | 對象特定（張三）、事件具體（那次闖紅燈） |
| 公告中正路 1/1-1/31 禁止停車 | 一般處分 | 對象可得確定（該時段經過的所有駕駛人）、事件具體（特定時段） |
| 修改道路交通管理處罰條例提高罰鍰 | 法規命令 | 對象不特定（全國所有駕駛人）、內容抽象（通案適用） |

### 邊界案例

**都市計畫是什麼？**

都市計畫的性質是大法官釋字 156 號的經典爭議。釋字 156 號認為：主管機關變更都市計畫，如果直接限制一定區域內人民之權利，具有「行政處分」的性質。但後續實務和學說仍有爭議，有認為應屬法規命令或一般處分者。

```
// 都市計畫性質的判斷
if (計畫內容是針對「特定區域」的「具體限制」):
    偏向一般處分（釋字156號見解）
elif (計畫內容是「通盤規劃」的「抽象安排」):
    偏向法規命令
```

### 考試陷阱

- 陷阱：一般處分仍然是行政處分，適用行政處分的救濟程序（訴願前置）。不要因為它是「一般」的就以為不能訴願。
- 陷阱：交通號誌、禁止標誌等「物的一般處分」最常考。記住：設置行為本身是一般處分，不是事實行為。

## 混淆組 5：羈束處分 vs 裁量處分

### 一眼判斷法

```
def classify(法律規定):
    if 法條用「應」「shall」:
        return "羈束處分"    # 沒有選擇空間
    elif 法條用「得」「may」:
        return "裁量處分"    # 有選擇空間
```

| 判斷基準 | 羈束處分 | 裁量處分 |
|----------|----------|----------|
| 法條用語 | 「應」 | 「得」 |
| 行政機關空間 | 零（依法照做） | 有（可以選擇） |
| 司法審查密度 | 全面審查 | 原則上尊重行政裁量 |
| 比喻 | 固定演算法（input 決定 output） | ML 模型（有調參空間） |

### 混淆原因

有些法條用語曖昧，不明確寫「應」或「得」。而且裁量處分不代表政府可以隨便決定，裁量有界限。

### 深度解析

**裁量的三種瑕疵（行程法 §10）：**

| 瑕疵類型 | 定義 | 比喻 | 例子 |
|----------|------|------|------|
| 裁量逾越 | 超出法律授權範圍 | 變數超出 valid range | 法律規定罰 1-10 萬，罰了 15 萬 |
| 裁量濫用 | 考量不相關因素或違反一般法律原則 | 用錯誤的 feature 做決策 | 因為長官的私人恩怨，對當事人從重處罰 |
| 裁量怠惰 | 應裁量而不裁量 | 該調參卻用 default | 法律規定罰 1-10 萬，不分情節一律罰 10 萬 |

**裁量收縮至零（裁量收縮理論）：**

```
def should_exercise_discretion(情況):
    if 人民生命/身體面臨急迫危險:
        return "必須行使裁量（得→應）"
    if 不行使裁量將嚴重侵害基本權:
        return "必須行使裁量（得→應）"
    if 行政機關已建立長期慣例:
        return "原則上須遵循（行政自我拘束）"
    else:
        return "有裁量空間"
```

經典案例：警察看到有人要殺人，法律規定「得」採取必要措施，但此時裁量已收縮至零，警察「應」採取措施。如果警察不作為導致人死亡，國家要賠。

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 符合退休條件，機關「應」准許退休 | 羈束處分 | 法條明定「應」，無裁量空間 |
| 違規排放廢水，主管機關「得」處罰鍰 | 裁量處分 | 法條用「得」，可選擇罰或不罰、罰多少 |
| 違規排放廢水導致居民中毒 | 裁量收縮至零 | 雖然法條用「得」，但因危害重大，「得」→「應」 |

### 考試陷阱

- 陷阱：「不確定法律概念」和「裁量」不一樣。不確定法律概念（如「公共利益」「情節重大」）是法律要件的解釋問題，法院可以全面審查；裁量是法律效果的選擇問題，法院原則尊重。但兩者的界線在學說上有爭議（判斷餘地理論）。
- 陷阱：裁量處分的司法審查不是「不審查」，而是「只審查有無瑕疵」。

## 混淆組 6：行政處分的無效 vs 得撤銷 vs 廢止

### 一眼判斷法（用程式 error handling 比喻）

```
def classify(行政處分的問題):
    if 處分有「重大明顯瑕疵」（行程法§111）:
        return "無效"        # compile error：從頭就不能執行
    elif 處分「違法」但瑕疵未達重大明顯:
        return "得撤銷"      # runtime bug：能執行但有問題，可 rollback
    elif 處分「合法」但情況變了/政策改了:
        return "廢止"        # deprecation：本來正常但官方決定下架
```

| 判斷基準 | 無效 | 得撤銷 | 廢止 |
|----------|------|--------|------|
| 處分本身 | 違法（重大明顯） | 違法（一般瑕疵） | 合法 |
| 效力 | 自始無效 | 撤銷前有效 | 廢止前有效 |
| 誰可以主張 | 任何人、任何時間 | 有期限限制 | 行政機關依職權 |
| 信賴保護 | 不適用 | 可能要補償 | 通常要補償 |
| 法律依據 | 行程法 §111-112 | 行程法 §117 | 行程法 §122-126 |
| 程式比喻 | compile error | runtime bug → rollback | planned deprecation |

### 混淆原因

三者都涉及「行政處分出了問題」。但問題的性質不同，法律效果差很多。

### 深度解析

**無效法定事由（§111）：** 不能得知處分機關、應給證書未給、內容不能實現、行為構成犯罪、違背公序良俗、欠缺事務權限、**其他重大明顯瑕疵**（概括條款）。

**得撤銷 vs 廢止的決定性差異：**

```
// 得撤銷
撤銷.原因 = "處分自始違法";        // 一開始就有問題
撤銷.效力 = "溯及既往（原則）";      // rollback 到最初
撤銷.補償 = "看信賴保護";           // 行程法§120

// 廢止
廢止.原因 = "處分合法但情況變更";    // 本來沒問題
廢止.效力 = "向將來失效（原則）";     // 只影響未來
廢止.補償 = "原則要補償";           // 行程法§126
```

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 核發建照給一塊法律上根本不能蓋房子的國有保護區 | 無效 | 內容對任何人均不能實現（保護區不能蓋） |
| 核發建照時漏審了消防法規 | 得撤銷 | 違法但非重大明顯，撤銷前建照仍有效 |
| 核發建照後都市計畫變更，該地改為公園用地 | 廢止 | 原處分合法，但情況變了 |

### 邊界案例

**「重大明顯」的門檻？** 實務採「明白性理論」：一般理性人一看就知有問題 → 無效；需仔細調查 → 得撤銷。例：罰單對象已死亡三年。兩說：無效（對象不存在）vs 得撤銷（需查戶籍才知）。

### 考試陷阱

- 陷阱：無效的行政處分「自始」無效，不需要撤銷。如果題目問「無效的行政處分應如何撤銷？」，正確答案是「不需要撤銷，可以提確認訴訟確認其無效」。
- 陷阱：得撤銷的處分在被撤銷前仍然有效（構成要件效力）。人民不能因為覺得處分違法就拒絕遵守。
- 陷阱：撤銷是溯及既往（原則），廢止是向將來失效（原則）。但兩者都有例外。

# 第二部分：救濟管道混淆組

> 選錯救濟管道 = 程序不合法 = 直接被駁回，實體問題連討論的機會都沒有。

---

## 混淆組 7：訴願 vs 行政訴訟

### 一眼判斷法

```
def choose_remedy(不服行政處分):
    step1 = 訴願(向上級機關)          # 行政體系內部自我審查
    if 訴願結果仍不服:
        step2 = 行政訴訟(向行政法院)  # 司法體系外部審查
    return step1 → step2              # 原則：訴願前置
```

| 判斷基準 | 訴願 | 行政訴訟 |
|----------|------|----------|
| 審查機關 | 上級行政機關 | 行政法院（司法機關） |
| 審查範圍 | 合法性 + 合目的性 | 原則只審合法性 |
| 法律依據 | 訴願法 | 行政訴訟法 |
| 期限 | 30 日內 | 訴願決定後 2 個月內 |
| 性質 | 行政救濟（內部） | 司法救濟（外部） |
| 比喻 | 向主管投訴（公司內部） | 去勞動局申訴（外部機構） |

### 混淆原因

兩者都是「不服行政處分的救濟」，差別在行政體系內審 vs 司法體系審。

### 深度解析

**訴願前置原則：** 原則上先訴願再行政訴訟，讓行政機關自我修正。**例外：可直接行政訴訟的情況：**

```
def can_skip_訴願():
    exceptions = [
        "課予義務訴訟中，已經向機關申請被駁回或怠為處分",  # 行訴法§5
        "確認訴訟",                                      # 行訴法§6
        "一般給付訴訟",                                   # 行訴法§8
        "交通裁決事件（2012年後改制）",                     # 直接向地方法院行政訴訟庭
        "其他法律特別規定免訴願者",
    ]
    return exceptions
```

### 案例對比

| 場景 | 救濟路徑 | 理由 |
|------|----------|------|
| 收到違規罰單不服 | 訴願 → 撤銷訴訟 | 標準路徑：不服行政處分 |
| 申請營業執照被拒 | 訴願 → 課予義務訴訟 | 訴願前置仍適用 |
| 不服交通違規裁決 | 直接向地院行政訴訟庭 | 交通裁決事件免訴願 |
| 請求確認行政處分無效 | 直接提確認訴訟 | 確認訴訟免訴願 |

### 考試陷阱

- 陷阱：訴願的審查範圍比行政訴訟「廣」，因為訴願還可以審查「合目的性」（行政裁量是否適當），而行政訴訟原則只審「合法性」。
- 陷阱：訴願期限是 30 日，從「知悉處分」起算。超過期限就不能訴願，也就不能提撤銷訴訟。但可以考慮走其他訴訟類型。

## 混淆組 8：撤銷訴訟 vs 課予義務訴訟 vs 確認訴訟 vs 一般給付訴訟

### 一眼判斷法

```
def choose_lawsuit_type(你想要的結果):
    if 你想要 == "撤銷政府做的壞決定":
        return "撤銷訴訟"           # 行訴法§4
    elif 你想要 == "逼政府做一個決定（政府不做或拒絕）":
        return "課予義務訴訟"       # 行訴法§5
    elif 你想要 == "確認某個法律關係存在或不存在":
        return "確認訴訟"           # 行訴法§6
    elif 你想要 == "要求政府做某個事實行為或給付":
        return "一般給付訴訟"       # 行訴法§8
```

| 判斷基準 | 撤銷訴訟 | 課予義務訴訟 | 確認訴訟 | 一般給付訴訟 |
|----------|----------|-------------|----------|-------------|
| 目的 | 撤銷處分 | 命做處分 | 確認關係 | 要求給付 |
| 前提 | 有處分存在 | 申請被拒/怠為處分 | 有確認利益 | 有給付請求權 |
| 訴願前置 | 要 | 要 | 不要 | 不要 |
| 法條 | 行訴法§4 | 行訴法§5 | 行訴法§6 | 行訴法§8 |
| 比喻 | git revert | 催 reviewer merge PR | git status 查看狀態 | 要求公司付薪水 |

### 混淆原因

選擇取決於「你想要什麼結果」，但情境常可同時走好幾種。

### 深度解析

**確認訴訟的「補充性原則」（§6 III）：** 能提撤銷或課予義務訴訟就不能提確認訴訟，確認訴訟是備胎。

```
if (可以提撤銷訴訟):
    提撤銷訴訟          # 優先
elif (可以提課予義務訴訟):
    提課予義務訴訟      # 優先
else:
    提確認訴訟          # 補充性：其他都不行才用
```

**課予義務訴訟兩型態：** (1)**怠為處分**：申請了，機關超期不回（§5 I）(2)**駁回申請**：申請了，機關拒絕（§5 II）

### 案例對比：同一事件的不同角度

假設你申請建照被拒：

| 你想要的 | 訴訟類型 | 說明 |
|----------|----------|------|
| 撤銷拒絕的處分 | 撤銷訴訟 | 把「駁回」撤掉 |
| 命行政機關核發建照 | 課予義務訴訟 | 逼政府發照 |
| 確認你有申請建照的權利 | 確認訴訟 | 但有補充性問題 |

**正確選擇：課予義務訴訟。** 因為你的終極目的是「拿到建照」，不是只把駁回撤掉。如果只提撤銷訴訟，法院撤銷駁回處分後，行政機關可能再找別的理由拒絕你。

### 邊界案例

**已執行完畢的處分？** 例：罰款已繳，事後發現違法。撤銷已無意義 → 提「確認處分違法訴訟」（§6 I 後段），再據此請求國賠。

### 考試陷阱

- 陷阱：課予義務訴訟的前提是你「已經向行政機關申請」。如果你根本沒申請就直接告，法院會駁回。
- 陷阱：一般給付訴訟和課予義務訴訟的區別在於：課予義務訴訟是要求「做行政處分」，一般給付訴訟是要求「做事實行為或金錢給付」。
- 陷阱：撤銷訴訟有「起訴期限」（訴願決定後 2 個月），過了就不能提。但確認訴訟沒有這個限制。

## 混淆組 9：國家賠償 vs 損失補償

### 一眼判斷法

```
def classify(政府讓你受損害):
    if 政府的行為是「違法」的:
        return "國家賠償"        # 國賠法
    elif 政府的行為是「合法」的:
        return "損失補償"        # 各個別法律規定
```

| 判斷基準 | 國家賠償 | 損失補償 |
|----------|----------|----------|
| 政府行為 | 違法 | 合法 |
| 法理基礎 | 有錯要賠（侵權行為法理） | 合法但造成特別犧牲（公平負擔法理） |
| 賠償範圍 | 所受損害 + 所失利益 | 合理補償（通常低於完全賠償） |
| 法律依據 | 國家賠償法 | 各個別法律（如土地徵收條例） |
| 請求對象 | 賠償義務機關 | 做出合法行為的機關 |
| 比喻 | 外送員送錯餐 → 全額退款+補償 | 房東合法收回房子 → 給搬遷費 |

### 混淆原因

都是「政府造成損害要給錢」，但一個是「違法代價」，一個是「合法但公平分擔」。

### 深度解析

**國賠兩大類型：** (1)公務員違法侵害（§2）(2)公共設施欠缺（§3）。**損失補償核心 — 特別犧牲理論：** 合法行為讓「特定人」承受超越一般忍受程度的損失，應補償。

```
def needs_compensation(合法行為造成的損失):
    if 損失是所有人都承受的（社會義務）:
        return False       # 例：每個人都要繳稅
    elif 損失只有特定人承受（特別犧牲）:
        return True        # 例：只有你的土地被徵收
```

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 警察執法時違法開槍射傷路人 | 國家賠償 | 違法行為造成損害 |
| 政府合法徵收你的土地蓋高速公路 | 損失補償 | 合法行為但造成特別犧牲 |
| 消防隊為了救火破壞你家牆壁 | 損失補償 | 合法行為（緊急避難）但你承受特別犧牲 |
| 道路施工圍籬未設警示燈，你摔傷 | 國家賠償（§3） | 公共設施管理欠缺 |

### 邊界案例

**補償不合理？** 不是國賠（非違法），是補償金額爭議，依土地徵收條例救濟。**法律漏未規定補償？** 可援引憲法 §15 財產權 + 特別犧牲理論，類推適用損失補償（釋字 747 號）。

### 考試陷阱

- 陷阱：國賠要先協議，協議不成才能訴訟。直接告法院會被駁回。
- 陷阱：國賠的訴訟是向「普通法院」（民事庭）提起，不是行政法院。
- 陷阱：損失補償通常由各個別法律規定，沒有統一的「損失補償法」。如果個別法律沒規定，要回到憲法層次論述。

## 混淆組 10：國賠 §2（公務員責任）vs 國賠 §3（公共設施責任）

### 一眼判斷法

```
def classify(國賠案件):
    if 損害來自「公務員的行為」:
        return "§2 公務員責任"     # 人的問題
    elif 損害來自「公共設施的狀態」:
        return "§3 公共設施責任"   # 物的問題
```

| 判斷基準 | §2 公務員責任 | §3 公共設施責任 |
|----------|-------------|----------------|
| 損害來源 | 人（公務員的行為） | 物（公共設施的狀態） |
| 主觀要件 | 要（故意或過失） | 不要（客觀上有欠缺即可） |
| 責任性質 | 過失責任 | 無過失責任（通說） |
| 舉證責任 | 人民要證明公務員有故意過失 | 人民只要證明設施有欠缺 |
| 比喻 | 員工犯錯 → 要證明是員工的錯 | 機器故障 → 不管誰的錯，機器壞了就要賠 |

### 混淆原因

同一事件可能同時涉及兩者（如坑洞 + 公務員不修），可擇一或併用。

### 深度解析

**§2 四要件：** 公務員 + 執行職務行使公權力 + 故意或過失 + 不法侵害人民權利。**§3 三要件：** 公有公共設施 + 設置或管理有欠缺 + 人民受損害。**§3 優勢：不需要證明故意過失**，舉證較容易。

### 案例對比

| 場景 | 請求依據 | 理由 |
|------|----------|------|
| 警察執勤時違法毆打民眾 | §2 | 人的行為（公務員暴力）|
| 國小校園遊樂設施鏽蝕斷裂，學生受傷 | §3 | 物的問題（設施管理欠缺）|
| 路燈不亮導致行人跌倒 | §3 | 設施管理欠缺 |
| 公務員故意不修路燈，導致行人跌倒 | §2 + §3 | 人和物都有問題，可以併用 |

### 邊界案例

**「公有公共設施」範圍：** 道路/橋樑/學校 → 明確屬於。BOT 設施 → 通說仍屬 §3。天然河川 → 未經人工設置原則不屬。

### 考試陷阱

- 陷阱：§3 是「無過失責任」，但有免責事由。如果行政機關能證明設施的欠缺不是因為設置或管理的問題（如不可抗力的天災直接毀損），可以免責。
- 陷阱：國賠法 §2 第 2 項規定，公務員有故意或重大過失時，國家賠償後可以向公務員「求償」。§3 則是向設置或管理有過失的機關或人員求償。

# 第三部分：憲法混淆組

> 憲法的概念抽象度更高，混淆起來更致命。

## 混淆組 11：法律保留 vs 國會保留

### 一眼判斷法

```
def classify(某事項的規範方式):
    if 只要有「法律依據」即可（可以是法律授權的命令）:
        return "法律保留"
    elif 必須由「立法院」親自以法律規定，不得授權:
        return "國會保留"     # 法律保留的最高等級
```

| 判斷基準 | 法律保留 | 國會保留 |
|----------|----------|----------|
| 規範密度 | 有法律或授權命令即可 | 必須是法律本身 |
| 授權可能 | 可以授權行政機關訂命令 | 不能授權，立法院要親自來 |
| 適用範圍 | 一般侵害人民權利事項 | 重大基本權限制事項 |
| 法理基礎 | 依法行政原則 | 民主正當性要求 |
| 比喻 | CEO 可以授權部門主管做決策 | 董事會必須親自表決，不能授權 |

### 混淆原因

國會保留「是」法律保留的一種（最嚴格），但法律保留「不一定是」國會保留。

### 深度解析：層級化法律保留（釋字 443 號）

| 層級 | 內容 | 規範密度 | 例子 |
|------|------|----------|------|
| 第一層 | 憲法保留 | 只有修憲才能動 | 國家領土、基本權核心 |
| 第二層 | 國會保留（絕對法律保留） | 必須法律明文 | 人身自由限制、刑罰 |
| 第三層 | 相對法律保留 | 法律或授權明確之命令 | 一般行政管制 |
| 第四層 | 非法律保留（行政保留） | 行政機關可自行規定 | 給付行政的細節 |

```
// 判斷某事項適用哪一層
function getReservationLevel(事項) {
    if (涉及基本權核心/國體) return "憲法保留";
    if (涉及人身自由/刑罰/重大基本權限制) return "國會保留";
    if (涉及一般基本權限制) return "相對法律保留";
    if (純屬給付行政細節) return "非法律保留";
}
```

### 案例對比

| 場景 | 層級 | 理由 |
|------|------|------|
| 規定某行為是犯罪並處徒刑 | 國會保留 | 涉及人身自由的剝奪，必須法律明定 |
| 規定商店營業時間 | 相對法律保留 | 限制營業自由，法律授權命令即可 |
| 規定公立圖書館開放時間 | 非法律保留 | 給付行政的執行細節 |
| 廢除死刑 | 國會保留（甚至憲法保留有爭議） | 涉及最根本的生命權問題 |

### 考試陷阱

- 陷阱：「法律保留」是統稱，包含國會保留在內。不要把兩者當成完全平行的概念。
- 陷阱：釋字 443 號的「層級化法律保留」是超高頻考點，尤其是判斷某個事項該放在哪一層。
- 陷阱：「給付行政不需要法律保留」是過時見解。現代通說認為「重大的給付行政」仍需法律依據（如全民健保的強制納保）。

## 混淆組 12：比例原則 vs 平等原則

### 一眼判斷法

```
def challenge_government_action(你的訴求):
    if 你認為 == "手段太過分了（殺雞用牛刀）":
        return "主張違反比例原則"     # 憲法§23、行程法§7
    elif 你認為 == "為什麼對我跟對他不一樣":
        return "主張違反平等原則"     # 憲法§7
```

| 判斷基準 | 比例原則 | 平等原則 |
|----------|----------|----------|
| 審查角度 | 手段與目的的關係 | 不同對象之間的對待差異 |
| 核心問題 | 有沒有必要做到這麼嚴格？ | 為什麼差別對待？差別有沒有道理？ |
| 三階段 | 適當性→必要性→狹義比例 | 相同事物相同處理、不同事物不同處理 |
| 法條依據 | 憲法§23、行程法§7 | 憲法§7 |
| 比喻 | Code review 問「這個 fix 有必要重構整個模組嗎？」 | Code review 問「為什麼 A team 可以用這個 library，B team 不行？」 |

### 混淆原因

都用來審查政府行為是否合憲合法，常在同一案件中同時出現，但切入角度不同。

### 深度解析：比例原則三階段（行程法 §7）

```
function proportionalityTest(手段, 目的) {
    // 第一階：適當性
    if (!手段能達成目的) return "違反比例原則";

    // 第二階：必要性（最小侵害）
    if (存在其他同樣有效但侵害更小的手段) return "違反比例原則";

    // 第三階：狹義比例性（衡量性）
    if (手段造成的損害 > 目的達成的利益) return "違反比例原則";

    return "合比例";
}
```

**平等原則的審查密度（三重審查基準，源自美國法）：**

| 審查密度 | 適用情形 | 標準 |
|----------|----------|------|
| 寬鬆審查 | 一般經濟立法 | 有合理基礎即可 |
| 中度審查 | 性別等分類 | 須有重要目的，手段須實質關聯 |
| 嚴格審查 | 種族等可疑分類 | 須有迫切目的，手段須嚴密量身 |

### 案例對比

| 場景 | 用比例原則分析 | 用平等原則分析 |
|------|---------------|---------------|
| 政府禁止在學校 500 公尺內開酒吧 | 目的正當（保護學童），但 500 公尺是否必要？100 公尺是否就夠？ | 為什麼只禁酒吧不禁檳榔攤？是否差別對待？ |
| 規定外國人不得擔任公務員 | 限制外國人的工作權，目的（國家忠誠）是否正當？手段是否必要？ | 以國籍為分類標準，差別對待是否有正當理由？ |

### 考試陷阱

- 陷阱：比例原則是「三階段」不是「三步驟」。三階段有嚴格的邏輯順序，前一階不過就不用審後一階。
- 陷阱：平等原則不是「完全相同對待」。合理的差別對待是允許的（實質平等）。考試常考的是：這個差別對待「合不合理」。
- 陷阱：申論題最好兩個原則都討論。例如政府的某項管制措施，先用比例原則審查手段，再用平等原則審查是否差別對待。

## 混淆組 13：違憲審查的各種類型

### 一眼判斷法

```
def classify(違憲審查):
    if 審查對象是「法律/命令」本身:
        return "抽象審查"      # 不需要具體個案
    elif 審查是在「具體訴訟案件」中附帶進行:
        return "具體審查"      # 法官認為法律違憲，聲請釋憲
    elif 審查對象是「法院裁判」本身:
        return "裁判憲法審查"  # 2022憲法訴訟法新制
```

| 判斷基準 | 抽象審查 | 具體審查 | 裁判憲法審查 |
|----------|----------|----------|-------------|
| 審查對象 | 法律/命令 | 法律/命令（透過個案） | 法院裁判本身 |
| 聲請人 | 機關、立委（1/4以上） | 各級法院法官 | 用盡審級的當事人 |
| 需要個案？ | 不需要 | 需要 | 需要（且已確定） |
| 法律依據 | 憲訴法§47等 | 憲訴法§55 | 憲訴法§59 |
| 比喻 | 直接做 code review 找 bug | 跑 test case 時發現 bug 才回報 | 對法官的 commit 做 review |

### 混淆原因

2022 年憲法訴訟法新增「裁判憲法審查」，使類型更複雜。

### 深度解析

**裁判憲法審查（2022 新制）：** 以往只能審查「法律」違憲，不能審查「裁判」違憲。新制（憲訴法 §59）開放：

```
if (你已用盡所有審級的救濟):
    if (你認為確定終局裁判「適用法律的方式」侵害基本權):
        可以向憲法法庭聲請「裁判憲法審查」
        期限：裁判送達後6個月內
```

### 案例對比

| 場景 | 審查類型 | 理由 |
|------|----------|------|
| 1/4 立委認為某法律違憲 | 抽象審查 | 機關聲請，不需要個案 |
| 法官審理案件時認為要適用的法律違憲 | 具體審查 | 法官停止審判，聲請釋憲 |
| 當事人認為最高法院的判決適用法律侵害基本權 | 裁判憲法審查 | 審查的是裁判本身，非法律本身 |

### 考試陷阱

- 陷阱：裁判憲法審查不是「第四審」。憲法法庭只審查「有無侵害基本權」，不審查事實認定和一般法律適用。
- 陷阱：聲請裁判憲法審查必須「用盡審級救濟」。如果你還能上訴卻沒上訴，不能聲請。
- 陷阱：2022 年之前的舊制（大法官會議解釋）和新制（憲法法庭裁判）的程序差異是考試重點。

## 混淆組 14：基本權的防禦功能 vs 保護義務 vs 給付功能

### 一眼判斷法

```
def classify(你對國家的要求):
    if 你要國家 == "不要侵害我":
        return "防禦功能"       # 基本權的傳統核心
    elif 你要國家 == "保護我不被第三人侵害":
        return "保護義務"       # 國家的積極保護責任
    elif 你要國家 == "給我某些東西/提供某些服務":
        return "給付功能"       # 社會權面向
```

| 判斷基準 | 防禦功能 | 保護義務 | 給付功能 |
|----------|----------|----------|----------|
| 方向 | 國家 → 人民（不要侵害） | 第三人 → 人民（國家要擋住） | 國家 → 人民（要提供） |
| 國家角色 | 被動不作為 | 主動保護 | 主動給付 |
| 拘束力 | 最強 | 中等（有立法裁量） | 最弱（資源限制） |
| 比喻 | 伺服器不要 DDoS 我 | 幫我設防火牆擋外部攻擊 | 提供免費 CDN 服務 |

### 混淆原因

三者都是基本權的「功能」，差別在你對國家的要求方向。

### 深度解析

**防禦功能：** 最傳統。人民直接援引基本權要求國家停止侵害，法院完全審查。**保護義務：** 國家有義務保護人民不受第三人侵害，但有裁量空間（保護不足禁止原則）。

```
if (國家的保護措施 < 最低保護標準):
    違憲（保護不足）
elif (國家的保護措施 >= 最低保護標準):
    合憲（即使不是最佳方案）
```

**給付功能：** 人民能否要求國家提供教育、醫療等？原則有義務，但受財政限制，立法者有廣泛形成空間。

### 案例對比

| 場景 | 基本權功能 | 說明 |
|------|-----------|------|
| 政府禁止某報紙出版 | 防禦功能 | 國家侵害了新聞自由，報紙可以要求撤銷 |
| 某企業壟斷媒體市場，妨害新聞多元 | 保護義務 | 國家應立法防止媒體壟斷，保護新聞自由 |
| 偏鄉居民要求國家補助設立社區媒體 | 給付功能 | 要求國家積極提供資源 |

### 考試陷阱

- 陷阱：保護義務的違憲審查用「保護不足禁止」，不是比例原則。比例原則是審查「國家侵害是否過度」（禁止過度），保護義務是審查「國家保護是否不足」（禁止不足）。
- 陷阱：給付功能不等於「國家什麼都要給」。受限於資源，立法者有廣泛裁量空間。考試如果問「人民可否請求國家提供免費大學教育」，答案通常是「有給付功能但非具體請求權」。

## 混淆組 15：特別權力關係的各種突破

### 一眼判斷法

```
def can_sue_government(身分, 爭議事項):
    if 爭議涉及「基本權的重大侵害」:
        return True    # 現代法：都可以救濟
    elif 爭議只是「內部管理措施」:
        return False   # 仍然不能告（但界線越來越模糊）
```

| 身分 | 傳統見解（不能告） | 現代突破（可以告） | 關鍵釋字 |
|------|------------------|-------------------|---------|
| 公務員 | 所有人事措施 | 免職、記大過等影響身分的處分 | 釋字 243、298、785 |
| 軍人 | 一切軍事管理 | 影響基本權的懲罰措施 | 釋字 430 |
| 學生 | 學校管理一切 | 退學、不准畢業等改變身分的處分 | 釋字 382、684 |
| 受刑人 | 監獄管理一切 | 影響基本權的措施（通信、健康） | 釋字 653、755 |

### 混淆原因

傳統「特別權力關係」排除司法救濟，大法官逐步打開缺口，問題是開了多大。

### 深度解析

**釋字 785（2019）公務員全面開放：** 公務員服公職權受侵害，得循司法救濟請求，「不再以是否改變身分為限」。

```
// 釋字785之前
if (公務員爭議涉及身分改變（免職/停職）):
    可以告
else:
    不能告（純屬內部管理）

// 釋字785之後
if (公務員認為服公職權受侵害):
    可以告  // 範圍大幅擴張
```

**釋字 684（2011）學生突破：** 釋字 382 只允許「退學或類此」時救濟，684 開放大學生權利受侵害即可救濟。

### 案例對比

| 場景 | 能不能告？ | 理由 |
|------|-----------|------|
| 公務員被免職 | 能（一直都能） | 改變身分，影響重大 |
| 公務員被記申誡 | 能（釋字785後） | 侵害服公職權 |
| 大學生被退學 | 能 | 釋字382即已開放 |
| 大學生被扣操行成績 | 能（釋字684後） | 權利受侵害即可救濟 |
| 受刑人被禁止與律師通信 | 能 | 釋字755，侵害基本權 |
| 受刑人被調整房間 | 不能（通說） | 純屬內部管理 |

### 考試陷阱

- 陷阱：釋字 785 號後，「特別權力關係」在公務員領域已經幾乎被完全突破。考試如果問「公務員可否對考績丙等提起行政訴訟」，答案是「可以」。
- 陷阱：學生領域的開放程度低於公務員。大學生已全面開放（釋字 684），但中小學生仍有爭議。
- 陷阱：受刑人的突破最保守。釋字 755 號開放「影響基本權」的監獄處分可以救濟，但「日常管理措施」仍然不行。

# 第四部分：跨公私法混淆組

> 公法和私法的界線不是涇渭分明的。政府也會簽民事契約、也會構成不當得利。

---

## 混淆組 16：行政契約 vs 民事契約（政府也會簽民事契約！）

### 一眼判斷法

```
def classify(政府簽的契約):
    if 契約的標的是「公法上的權利義務」:
        return "行政契約"       # 行程法§135以下
    elif 契約的標的是「私法上的權利義務」:
        return "民事契約"       # 民法
```

| 判斷基準 | 行政契約 | 民事契約（政府簽的） |
|----------|----------|---------------------|
| 契約標的 | 公法事項（行使公權力） | 私法事項（市場交易） |
| 法律依據 | 行程法 §135-149 | 民法 |
| 管轄法院 | 行政法院 | 普通法院（民事庭） |
| 判斷標準 | 契約內容涉及公權力行使 | 政府以「私人」身分參與市場 |
| 比喻 | 政府和你簽 SLA（公共服務條款） | 政府在電商平台買辦公用品 |

### 混淆原因

政府也會「買東西」「租房子」，這些市場交易是民事契約，不是行政契約。

### 深度解析（契約標的理論）

```
function classifyContract(契約) {
    // 第一步：看契約的「標的」（內容）
    if (契約內容涉及公法上的權利義務) {
        return "行政契約";
        // 例：政府與廠商簽BOT契約 → 涉及公共建設的特許經營
    }
    if (契約內容是一般商業交易) {
        return "民事契約";
        // 例：政府採購辦公桌椅 → 一般買賣
    }

    // 第二步：輔助標準
    if (契約一方可以行使公權力) return "行政契約";
    if (契約替代了行政處分) return "行政契約";
}
```

### 案例對比

| 場景 | 分類 | 理由 |
|------|------|------|
| 政府和廠商簽 BOT 契約經營高速公路 | 行政契約 | 涉及公共建設特許經營（公法事項） |
| 政府向文具行採購辦公用品 | 民事契約 | 一般商業買賣（私法事項） |
| 政府出租國有土地供民間使用 | 看情形 | 如果是公用目的（如社會住宅） → 行政契約；如果是單純收租 → 民事契約 |
| 公立醫院聘僱清潔人員 | 民事契約 | 勞動關係，私法事項 |

### 邊界案例

**政府採購契約？** 通說：原則上是民事契約。採購「程序」受公法規範，但「契約本身」是私法買賣/承攬，爭議去普通法院。

### 考試陷阱

- 陷阱：「政府簽的契約」不等於「行政契約」。要看契約內容是公法事項還是私法事項。
- 陷阱：管轄法院不同。行政契約的爭議去行政法院（一般給付訴訟），民事契約的爭議去普通法院。選錯法院就被裁定移送或駁回。

## 混淆組 17：公法上不當得利 vs 民法不當得利

### 一眼判斷法

```
def classify(不當得利):
    if 得利的原因是「公法關係」:
        return "公法上不當得利"    # 行程法§127
    elif 得利的原因是「私法關係」:
        return "民法不當得利"     # 民法§179
```

| 判斷基準 | 公法上不當得利 | 民法不當得利 |
|----------|--------------|-------------|
| 發生原因 | 公法關係（如行政處分撤銷後） | 私法關係（如契約無效後） |
| 法律依據 | 行程法 §127 | 民法 §179 |
| 消滅時效 | 5 年（行程法 §131） | 15 年（民法 §125） |
| 管轄法院 | 行政法院 | 普通法院 |
| 比喻 | 政府多給你的退稅 → 公法關係 | 賣家多收你的貨款 → 私法關係 |

### 混淆原因

法理結構類似（無法律上原因受利益致他人損害），但底層關係不同 → 法院和時效不同。

### 深度解析

**常見情境：** (1)處分撤銷後返還補助款 (2)溢繳稅款退還 (3)誤發款項返還。**行程法 §127：** 授益處分撤銷/廢止後，受益人應返還所受利益。

```
// 公法上不當得利的典型流程
const 授益處分 = 核定補助款50萬;
const 人民已領取 = 50萬;
// 事後發現處分違法
撤銷(授益處分);
// 處分基礎消滅 → 人民持有的50萬變成「無法律上原因」
行政機關.請求返還(50萬);  // 公法上不當得利
```

### 案例對比

| 場景 | 分類 | 管轄法院 |
|------|------|----------|
| 政府撤銷補助處分後，要求返還已發的補助款 | 公法上不當得利 | 行政法院 |
| 你多繳了所得稅，申請退稅 | 公法上不當得利 | 行政法院 |
| 你網購多付了錢，要求商家退款 | 民法不當得利 | 普通法院 |
| 政府買辦公用品多付了錢，要求廠商退款 | 民法不當得利 | 普通法院（因為底層是民事採購契約） |

### 考試陷阱

- 陷阱：時效差異是考試最愛考的。公法上不當得利的時效只有 5 年，民法是 15 年。同樣是不當得利，定性錯誤會導致時效判斷錯誤。
- 陷阱：政府要求人民返還公法上不當得利，需要作成行政處分（下命處分），不能直接起訴。人民不服可以訴願。但人民要求政府返還溢繳稅款，是提課予義務訴訟或一般給付訴訟。

## 混淆組 18：行政罰 vs 刑罰（一行為不二罰原則）

### 一眼判斷法

```
def classify(處罰類型):
    if 處罰機關是「行政機關」:
        return "行政罰"        # 行政罰法
    elif 處罰機關是「法院」（刑事庭）:
        return "刑罰"          # 刑法
```

| 判斷基準 | 行政罰 | 刑罰 |
|----------|--------|------|
| 處罰機關 | 行政機關 | 法院 |
| 法律依據 | 行政罰法、各行政法規 | 刑法、各刑事特別法 |
| 處罰類型 | 罰鍰、沒入、停業、吊照 | 死刑、自由刑、罰金、沒收 |
| 程序 | 行政程序 | 刑事訴訟程序 |
| 構成要件 | 違反行政法上義務 | 構成犯罪 |
| 比喻 | 違反公司規定被罰款 | 違反法律被送法院 |

### 混淆原因

同一行為可能同時違反行政法和刑法（如酒駕 = 行政違規 + 公共危險罪）。

### 深度解析（行政罰法 §26）

```
def handle_concurrent_violation(行為):
    if 同時構成犯罪和行政違規:
        // 原則：刑事優先
        先進行刑事程序
        if 被科處刑罰:
            「罰鍰」部分 → 不再處行政罰鍰（吸收）
            「其他種類行政罰」（如吊照）→ 仍可裁處（併罰）
        elif 不起訴/無罪:
            可以再處行政罰
```

**核心規則：**

1. **刑事優先原則**：先走刑事程序
2. **罰鍰部分被刑罰吸收**：同一行為不能既罰刑事罰金又罰行政罰鍰
3. **非罰鍰的行政罰可以併罰**：例如吊銷駕照、停業處分等不是「罰鍰」性質的行政罰，可以和刑罰並行

### 案例對比

| 場景 | 處罰方式 | 理由 |
|------|----------|------|
| 酒駕，刑事判處罰金 5 萬 | 不再處行政罰鍰（被吸收），但可以吊銷駕照 | 罰鍰被罰金吸收，吊照是「其他種類」可併罰 |
| 酒駕，刑事判處有期徒刑（沒罰金） | 可以處行政罰鍰 + 吊銷駕照 | 刑罰不含罰金，沒有吸收效果 |
| 酒駕，刑事不起訴 | 可以處行政罰鍰 + 吊銷駕照 | 沒有刑罰，行政罰自由處罰 |
| 違規排放廢水，刑事判處罰金 + 罰鍰 | 不得再處行政罰鍰，但可以命停工 | 罰鍰被吸收，停工是其他種類 |

### 邊界案例

**「一行為」的認定：**

如果是「數行為」分別違反刑法和行政法，就不適用一行為不二罰原則，而是分別處罰。關鍵在於「一行為」的認定。

例：A 長期違法排放廢水（持續犯）。每次排放是「一行為」還是整體是「一行為」？行政罰法上的一行為認定和刑法不完全相同，這是考試的高級考點。

### 考試陷阱

- 陷阱：「一行為不二罰」只禁止重複科處「同種類」的罰。罰鍰和罰金是同種類（都是金錢罰），所以不能並罰。但罰鍰和吊照是不同種類，可以並罰。
- 陷阱：刑事優先不是「行政罰消滅」。如果刑事程序最終結果是無罪或不起訴，行政機關仍可以處行政罰。
- 陷阱：行政罰法 §26 第 3 項規定，刑事程序進行中，行政機關可以「暫時」停止行政罰程序，等刑事結果出來再決定。

# 附錄二：綜合實戰 — 一個案例走遍多組混淆

> 以下用一個完整案例串聯多組混淆概念，示範如何在申論題中精確使用本文件的判斷框架。

### 案例事實

甲在 A 市經營一家小型工廠。A 市環保局以甲違反空氣污染防制法為由，做出以下行為：

1. 命甲限期改善排放設備（行為一）
2. 裁處罰鍰 30 萬元（行為二）
3. 派員進入甲的工廠採集空氣樣本（行為三）
4. 公告 A 市工業區全面禁止使用 X 類燃料（行為四）

事後，甲因同一排放行為被檢察官依刑法公共危險罪起訴，法院判處罰金 10 萬元。

甲不服上述所有處置，請問應如何救濟？

### 分析架構

**第一步：行為分類（混淆組 1、4）**

```
行為一：命限期改善 → 行政處分（下命處分，對甲產生作為義務）
行為二：裁處罰鍰   → 行政處分（裁罰處分）
行為三：派員採樣   → 事實行為（沒有改變甲的法律關係）
行為四：公告禁用燃料 → 一般處分（對工業區內可得確定之工廠）
```

**第二步：救濟管道（混淆組 7、8）**

```
行為一、二、四：
  → 訴願（30日內）→ 不服 → 撤銷訴訟（行訴法§4）

行為三（事實行為）：
  → 如有損害 → 國家賠償（國賠法§2）
  → 如要求停止 → 一般給付訴訟（行訴法§8）
```

**第三步：一行為不二罰（混淆組 18）**

```
甲的同一排放行為：
  刑事罰金 10 萬（已判處）
  行政罰鍰 30 萬
  → 行政罰法§26：罰鍰被罰金吸收
  → 30 萬罰鍰不得再裁處
  → 但「命限期改善」不是罰鍰，是「其他種類處分」，不受影響
```

**第四步：如果甲主張空污法的處罰規定違憲（混淆組 11、12）**

```
比例原則審查：
  目的（保護環境）正當？→ 是
  手段（罰鍰+停工）適當？→ 是
  手段必要？→ 是否有侵害更小的替代手段？
  狹義比例？→ 罰鍰金額和環境損害是否均衡？

法律保留審查：
  罰鍰有法律依據（空污法明定）→ 符合法律保留
  是否需要國會保留？→ 罰鍰非人身自由限制，相對法律保留即可
```
