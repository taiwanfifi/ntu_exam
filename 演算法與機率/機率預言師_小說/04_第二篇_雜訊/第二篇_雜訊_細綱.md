## 第 13 章：永和的牆

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：深藍灰。像走進一間沒開燈的房間——你知道裡面有東西。
POV：第一人稱（貝葉）
機率概念：常態分布 vs 其他分布的辨識
本章目標：蘇立恆的調查基地揭曉。貝葉看到「全貌」。從七個區域升級到二十三個。
```

### 場景分鏡

```
【開場】
紙條上的線索：「先驗端口。1,247 次。查 B2。」
貝葉不知道怎麼「查 B2」。她沒有地下二層的權限。
但她知道一個人有——就是給她紙條的人。

問題是：她不知道他是誰、在哪裡、怎麼聯絡。

然後她想起白奶奶的話。
「穿黑衣的少年仔。每天下午都來。」
每天下午。

她去了蔥油餅攤。週六下午。
石椅上沒有人。但有一張蔥油餅。紙包的。還溫的。

她坐在白奶奶攤位的小板凳上。等。
白奶奶看了她一眼。什麼都沒問。只是多做了一張餅。
「今天的蔥特別香。」

四十七分鐘後。
他來了。

蘇立恆。站在石椅旁邊。看了一眼那張沒被拿走的蔥油餅。
然後看到了貝葉。

兩個人對視了三秒。
第二次。但感覺跟第一次完全不同。
第一次是「你是誰？」
這一次是「你準備好了嗎？」

「跟我來。」他說。

白奶奶在後面喊：「蔥油餅不帶嗎？」
蘇立恆回頭。猶豫了半秒。走回去。拿了那張餅。
白奶奶笑了：「少年仔，終於願意自己拿了。」

→ 微小的溫暖。蘇立恆的第一個「非理性」行為——不是因為餓。

【場景一】永和。巷弄。
一棟五十年的舊公寓。四樓。沒有電梯。
樓梯間有人曬的衣服。有小孩騎三輪車留下的輪胎痕。
生活的味道。但蘇立恆不屬於這裡。

打開門。一間十坪的套房。

牆上。
滿了。

紅色的點。藍色的線。打印出來的圖表。手寫的筆記。
地圖。時間線。公式。
全台灣的行政區地圖被放大了五倍貼在牆上——
上面有 23 個紅點。每個紅點旁邊寫著數字。

「23 個區域。」蘇立恆站在地圖前面。
「先驗被修改的幅度從 12% 到 61%。
 全部往同一個方向——壓低風險認知。
 讓人覺得更安全。」

貝葉走到地圖前面。
她的七個區域全部在上面——但還有十六個她沒查到的。

「你查了三年？」
「三年。」
「一個人？」
「一個人。」

停頓。

「為什麼是現在才找我？」

蘇立恆看著牆。
「因為你是第一個……用對方法的人。」
「什麼意思？」
「其他人查的是數據。看有沒有被修改。
 答案是沒有。數據是真的。
 所以他們放棄了。」

他轉頭看她。

「你是第一個去查先驗的。」

→ 貝葉的特殊性：不是比別人聰明。是高思遠教她的那句話——
  「質疑先驗。永遠質疑先驗。」
  大部分人質疑數據。她質疑起點。

【場景二】分布辨識——不同區域用了不同的手法
蘇立恆拉出一張圖表。23 個區域的先驗分布類型。

  ┌─ 先驗操控類型分布 ──────────────────────┐
  │                                          │
  │ 使用 Gamma 先驗：9 個區域（犯罪率）     │
  │ 使用 Normal 先驗：7 個區域（經濟指標）   │
  │ 使用 Exponential 先驗：4 個區域（等待時間）│
  │ 使用 Beta 先驗：3 個區域（??? 未知）     │
  │                                          │
  │ 總計：23 個                               │
  └──────────────────────────────────────────┘

「不同類型的數據用了不同的分布族。」蘇立恆指著圖表。
「犯罪率用 Gamma——因為犯罪事件數是正數、連續的。
 經濟指標用 Normal——因為經濟波動近似常態。
 等待時間用 Exponential——醫院等候、行政審批。」

「那三個 Beta 呢？」貝葉指著最後一行。

蘇立恆沉默了。

「那三個……我還沒搞清楚。
 Beta 分布的值域是 [0, 1]——它通常用來表示『機率的機率』。
 或者——信念的強度。
 有人用 Beta 分布做了什麼，但我不知道是什麼。」

→ Beta 分布的懸念。在第 16 章揭曉——共軛先驗的完美犯罪。
→ 在第 22 章再次出現——林岳的遺言 Beta(72, 28)。

「你說你離開的時候沒帶走證據。那這些——」
「都是我從頭開始、從外部蒐集的。公開數據。側面管道。」
「三年。」
「三年。」

貝葉環顧四周。
小桌上有兩台筆電（都很舊）、一疊文件、一杯喝到一半的黑咖啡。
沒有床。只有一張瑜伽墊。
「你睡這裡？」
「有時候。」
「瑜伽墊？」
「便宜。而且不佔空間。」

→ 蘇立恆的生活方式：極簡。孤獨。三年獨自戰鬥的痕跡。
```

## 第 14 章：分布的味道

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：電子綠。數據分析的光。兩個人面對面破解謎題。
機率概念：Normal / Exponential / Gamma 分布的辨識方法
本章目標：貝葉學會「看分布」——每種分布都有自己的味道、形狀、個性。
```

### 核心場景

```
蘇立恆教貝葉辨識被操控的分布。
不是教科書式的——是像偵探辨識筆跡一樣。

「每種分布都有自己的『簽名』。」蘇立恆說。
他在白板上畫了三條曲線。

「常態分布——對稱。鐘形。像一座山。
 如果你看到一組數據的直方圖長這樣——
 中間高、兩邊低、左右對稱——
 那它可能是常態的。
 經濟指標、考試成績、身高——這些天生就接近常態。」

「指數分布——L 形。一邊高然後急速下降。
 像排隊等候時間——大部分人等很短，少數人等很久。
 它只有正值。沒有負的等待時間。
 如果你看到一組數據全部大於零，右邊有一條長尾巴——
 可能是指數。」

「Gamma 分布——是指數分布的『升級版』。
 形狀更靈活。可以從 L 形變到接近鐘形。
 看 α 和 β 兩個參數。
 α 大 → 越像常態。α 小 → 越像指數。
 犯罪率、保險理賠——這些用 Gamma 建模。」

貝葉看著三條曲線。

「所以……辨識分布就像辨認人的臉？」
「差不多。看多了就有感覺。」

她看了他一眼。
「你剛才說的話比我們認識以來所有對話加起來都多。」

蘇立恆停頓了。
「……是嗎。」
「是。」

他轉回去看白板。
「繼續。」

→ 蘇立恆在教學的時候會忘記「節省字數」。
  這是他唯一話多的場合。
  貝葉注意到了。小拉也注意到了（但被靜音了）。

【實作場景】
兩人坐在小桌前。面對面。兩台筆電。
蘇立恆把 23 個區域的原始數據傳給貝葉。

「你來判斷。每個區域的分布類型。」
「現在？」
「現在。」

貝葉打開數據。
第一個區域——大安區。
她畫了直方圖。看了三秒。

「Gamma。偏態明顯。右尾巴。α 大概 2-3。」
「對。」

第二個——信義區。
直方圖。不太對稱。但不像 Gamma。

「這個……Normal？但偏態有一點。」
「再看一次。」
她放大了。「等一下——這個峰太尖了。
 Normal 的峰度是 3。這個超過了。」
「所以？」
「不是 Normal。是……Gamma 裝成 Normal。
 有人把 α 調很大，讓 Gamma 看起來像 Normal。」

蘇立恆第一次——微微笑了。
嘴角動了不到一毫米。但貝葉看到了。

「你的直覺很好。」
「不是直覺。是老高教的。」
「高教授？」
「嗯。他說：『分布不會說謊。但會化妝。你要學會卸妝。』」

蘇立恆沒有回答。
但他在筆記本的空白處寫了一行字：
「她的先驗比我預期的好。」
然後劃掉了。因為那看起來太像——某種東西。
```

## 第 15 章：馬可瑜的數據

### 基本資訊
```
字數目標：4,500-5,000 字
情緒色調：暖色系暫時回歸。馬可瑜帶來了陽光——但也帶來了更大的陰影。
機率概念：不同分布族的應用場景（保險數據視角）
本章目標：馬可瑜正式加入。保險數據也出問題了。戰場從「犯罪率」擴大到「全社會」。
```

### 核心場景

```
貝葉跟馬可瑜約在她們大學時常去的咖啡店。

馬可瑜遲到了七分鐘。衝進來的時候手裡抱著柯基 theta——
「抱歉抱歉！theta 在公園追松鼠追了太久——
 松鼠跑上樹牠還在下面叫——
 我跟牠說你抓到松鼠的機率是 0.003 牠不聽——」

她把 theta 放在椅子上。theta 趴下來。看起來很累但很滿足。

「所以你的精算模型考慮了柯基追松鼠的條件機率嗎？」貝葉問。
「哈！如果我把 theta 的行為建模，variance 會爆炸。」

笑了。
貝葉已經好幾天沒有笑了。

然後——她把事情告訴了馬可瑜。
不是全部。只是「我發現某些區域的犯罪率數據不太對」。

馬可瑜聽完之後。
沒有驚訝。
她用一根吸管攪拌她的拿鐵。攪了很久。

「……貝葉。」
「嗯？」
「我也發現了。」

「什麼？」

「保險數據。不是犯罪率。是理賠率。」
她拿出平板。打開一個表格。

「過去一年。大安區、信義區、中正區——
 車險理賠率突然下降。
 不是慢慢降的。是……跳崖式下降。」

「多少？」

「平均下降 38%。
 但實際事故數沒有減少。」

貝葉的手停在咖啡杯上。

「理賠率降了，但事故沒降？」

「對。知道為什麼嗎？
 因為透鏡告訴居民那些區域很安全。
 居民就覺得安全。就不買那麼多保險了。
 保險購買率下降 → 理賠率（理賠次數/保單數）的分母縮小——
 但分子（實際事故）沒變——
 所以理賠率反而——」

「等等。」貝葉打斷她。「理賠率不是應該上升嗎？」

馬可瑜搖頭。

「正常情況是。但保險公司也看透鏡的數據。
 透鏡說安全 → 保險公司降低了那些區域的風險評級 →
 降低了保費 → 更多人覺得『反正便宜就保一下』→
 保單數暴增 → 分母更大 → 理賠率被稀釋——」

「所以表面上看起來更安全了。」
「但實際上——事故數完全沒變。
 只是數字上被稀釋了。」

馬可瑜把吸管放下。
她的臉上沒有笑了。
theta 在椅子上打了一個小哈欠。

「貝葉。這不是在改幾個犯罪率。
 他們在改整個社會的風險認知。」

→ 「風險認知」——從個人到社會。
  先驗操控影響的不只是透鏡上的數字。
  是人的行為。是市場。是整個經濟。
```

## 第 16 章：共軛先驗

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：冷。數學的冷。像解剖台上的光——精確、無情。
機率概念：Beta 分布、共軛先驗（核心）
本章目標：揭露操控的最核心手法。數學上完美的犯罪。
```

### 核心場景

```
回到永和套房。深夜。
蘇立恆和貝葉面對面。兩台筆電。兩杯黑咖啡。
牆上的地圖在燈光下投下陰影。

「三個 Beta 區域。」蘇立恆指著地圖上的三個點。
「我一直搞不清楚這三個區域被改了什麼。
 犯罪率用 Gamma。經濟用 Normal。等待時間用 Exponential。
 但 Beta——Beta 的值域是 [0, 1]。」

「[0, 1] 意味著它代表的是一個比例。或者——一個機率。」貝葉說。

「對。一個機率的機率。」

她拿過數據。三個區域。
不是犯罪率、不是經濟指標、不是等待時間——
是「民意調查」。
政策支持度。候選人好感度。公共議題認同度。

「他們用 Beta 分布來操控民意。」

蘇立恆點頭。

「Beta(α, β) 的期望值是 α/(α+β)。
 如果你想讓某個政策的支持度從 50% 變成 70%——
 你只需要把先驗從 Beta(5, 5) 改成 Beta(7, 3)。」

「但——這不是很容易被發現嗎？」
「正常情況下，是。但 Beta 有一個特性——」

他在白板上寫：

  Beta(α, β) 是 Bernoulli/Binomial 的共軛先驗。

  這意味著：
  先驗 Beta(α, β)
  + 觀測 n 次實驗，其中 k 次成功
  → 後驗 Beta(α+k, β+n-k)

  後驗還是 Beta。參數只是簡單地相加。

「看到了嗎？」蘇立恆說。

「如果你修改了先驗的 α 和 β——
 新數據進來之後，貝氏更新只是把新的觀測數加上去。
 後驗還是 Beta。形式完全不變。
 你分不出來——到底是先驗本來就是這樣，
 還是被人改過了。」

「因為共軛先驗的更新是……無縫的。」

「對。無縫。沒有痕跡。
 改了 Gamma 的先驗，你可以從後驗的形狀裡看到『接縫』。
 改了 Normal 的先驗，殘差裡會有『回聲』。
 但改了 Beta——後驗完美地吸收了修改。
 就像一條河吞掉了一滴墨水。你知道墨水在裡面。但你看不到了。」

貝葉盯著白板上的公式。

「數學上完美的犯罪。」她說。

聲音很輕。
不是佩服。是恐懼。
一種很特殊的恐懼——
不是因為敵人很壞。是因為敵人的數學很好。

小拉在透鏡裡（離線模式下有限功能）閃了一下：
「貝姊……我有一個問題。」
「什麼？」
「如果共軛先驗的操控是無痕的……
 那我們怎麼知道——我自己的先驗有沒有被改過？」

沉默。

小拉的焦慮不再是碎念了。是真的害怕。

「我不知道。」貝葉說。
「但我們會找到辦法。」

她看了蘇立恆一眼。
他在看她。
不是平常那種不看人的眼神。是直視。

「你怕了嗎？」他問。

「怕。」她說。很坦白。

「嗯。」他轉回去看白板。
「怕是正常的。不怕才有問題。」

停頓。

「但你還要繼續嗎？」

「你呢？」

「我已經三年了。我不會停。」

「那我也不會。」

蘇立恆的嘴角動了一下。不到一毫米。
但在永和套房的冷白燈光下，那一下像一絲暖意。
```

## 第 17-18 章：被標記了

### 基本資訊
```
字數目標：各 4,500 字
情緒色調：暗紅。危險逼近。像水面下有什麼東西在游。
機率概念：Beta 分布（深化）、先驗操控的社會影響
本章目標：貝葉的調查行為被系統偵測。費雪開始注意她。
```

### 核心場景

```
第 17 章：
  貝葉白天在公司正常上班。晚上去永和套房分析數據。
  她以為自己很小心。離線模式。加密文件。

  但她忽略了一件事——
  她的透鏡即使在離線模式下，開機紀錄還是會被同步到雲端。
  「離線」只是不連線。但開關機的時間戳——會記錄。

  費雪的安全部門有一個監控系統：
  「異常使用模式偵測器」。
  任何校準師如果在非工作時間頻繁開關透鏡——
  系統會自動標記為「可疑行為」。

  貝葉不知道。但小拉知道。
  小拉在離線模式下沒有辦法告訴她——
  因為離線模式下，小拉的監控警告功能被關閉了。

  回到公司。週一。
  螢幕上有一條系統通知：
  「您的透鏡使用模式已更新。請確認您的校準排程。」

  正常的系統通知。每個人都收過。
  但小拉偷偷看了一眼通知的後設資料：
  「貝姊……這條通知的來源不是 IT 部門。是安全部門。」

  貝葉的手停在鍵盤上。

  「他們在看我。」

─────────────────────────

第 18 章：
  恐懼開始蔓延。
  貝葉在辦公室裡感覺到了——有人在看她。
  不是同事的目光。是系統的目光。

## 第 21-23 章：破解 Beta(72, 28)

### 基本資訊

### 核心場景（三章合併概述）

```
第 21 章：
  Beta(72, 28)。什麼意思？
  字面上：一個 Beta 分布，α=72, β=28。期望值 = 72/(72+28) = 0.72。
  72% 的信心度。關於什麼？

  蘇立恆和貝葉在永和套房破解。
  「林岳不會隨便寫一組參數。這裡面有資訊。」

  蘇立恆提出假設：
  「如果林岳發現了一個聯合分布——兩個變數的關係——
   然後他用邊際化把不需要的變數積掉，
   留下的就是一個一維的分布。
   Beta(72, 28) 可能是某個聯合分布的邊際。」

  問題：哪兩個變數？

第 22 章：
  貝葉用林岳之前跟她提到的資訊——
  「某些區域的先驗跟兩年前不一樣」——
  假設兩個變數是：
  X = 先驗修改的幅度
  Y = 修改的時間點

  如果 (X, Y) 有一個聯合分布，
  那 Beta(72, 28) 是 X 的邊際分布。
  這意味著：先驗修改的幅度有 72% 的機率落在某個特定範圍內。

  但更重要的是——
  從聯合分布中，你可以算出條件分布：
  P(Y|X) = 在已知修改幅度的情況下，修改時間的分布。
  = 你可以推斷出「什麼時候」先驗被修改了。

  「林岳不只是告訴我們有操控。
   他告訴我們——操控的時間模式。」

第 23 章：
  變數轉換。
  貝葉用條件分布推斷出：
  先驗修改集中在三個時間窗口——
  全部是深夜。全部在「系統自動校準」的排程之後。
  有人在自動校準完成之後、手動覆蓋了先驗。

  「自動校準是幌子。真正的修改在它之後。」

  更關鍵的發現：
  三個時間窗口跟 Oracular B2 層的門禁紀錄有相關性。
  「有人在這些時間進出了 B2。」

  紀錄被清除了。

  「清除紀錄的人——比操控先驗的人更危險。
   因為他不只在改數據。他在改歷史。」

  蘇立恆看著螢幕。
  「費雪。」他說了一個名字。
  「安全部門有權限清除門禁紀錄。」

  → 費雪的角色從「暗示」升級為「確認」。
