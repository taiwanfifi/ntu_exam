### 演算法 AB 線分析

### 演算法 AB 線分析

```
A 線（Heap / Priority Queue）：
  核心概念：動態維護最大值。插入 O(log n)，取出 O(1)
  場景實現：
    - 米亞把所有未覆蓋的記憶放入 Heap
    - 排序標準 = 緊急程度 × 成功率 / 耗時
    - 頂端永遠是當前最高分
    - 維倫取走頂端 → 下一個最高分自動浮上來
    - 新記憶進入前沿 → 插入 Heap 正確位置
    - 記憶被覆蓋 → 從 Heap 移除

  Heap 的核心教學完成度：
    ✓ 結構（倒三角 / 塔）
    ✓ 插入（新光點放入正確位置）
    ✓ 取出（頂端被拿走，下一個浮上來）
    ✓ 動態維護（米亞持續更新排序）
    ✓ O(log n) 暗示（「塔會自己重新排列」= 自動調整）

  與 Greedy 的配合：
    ch163-164 = Greedy 決定「每步選最好的」
    ch165 = Heap 決定「什麼是最好的」
    Greedy 是策略。Heap 是工具。
    兩者是搶救的左右手

B 線（無力感 + 信任）：
  ch164 的維倫在親自掃描三萬個 → 燃燒自己
  ch165 的維倫只看頂端一個 → 信任系統
  轉變的意義：
    不是「他變輕鬆了」——是「他接受了自己看不完所有人」
    Heap 不是解藥。Heap 是「在無力中唯一能做的事」
    讓系統排序 = 承認人的有限性

AB 線同步點：
  「排在最上面的不是最痛苦的——是最有可能被救回的」
    A 線 = Heap 的排序邏輯（成功率 > 情感權重）
    B 線 = 末日中的倫理：你不能選「最值得救」的，只能選「最可能被救」的
  → 讀者理解的不是數據結構——是「在末日中排隊的規則」
```
