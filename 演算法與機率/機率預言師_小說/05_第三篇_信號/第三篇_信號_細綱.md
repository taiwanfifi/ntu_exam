## 第 26 章：期望值與代價

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：暗金轉為鐵灰。團隊的第一次認知到——代價比想像的大。
機率概念：期望值（正式）、LOTUS（Law of The Unconscious Statistician）
本章目標：大規模數據分析。期望損害超乎想像。
```

### 核心場景

```
永和套房。但空間不夠了——四個人 + theta 塞不下。
他們搬到了高思遠在台大的研究室。301。
門上的名牌還是「高思遠」。白板上還留著最後一堂課的板書。
現在白板被擦乾淨了。上面寫著新的東西。

三天的數據清理之後。結果出來了。

  ┌─ 先驗操控全景估計 ──────────────────────────┐
  │                                               │
  │ 總分析區域：152 個（有足夠數據品質的區域）   │
  │ 偵測到先驗偏移的區域：109 個                 │
  │ 偵測率：109/152 = 71.7%                      │
  │ 與 Beta(72, 28) 的 72% 高度吻合 ✓           │
  │                                               │
  │ 先驗偏移幅度（109 個區域）：                  │
  │   平均值 E[ΔP] = 0.38（38%）                │
  │   標準差 σ = 0.15                            │
  │   最大值 = 0.72（某選區民意）                │
  │   最小值 = 0.08（某偏遠區域）                │
  └───────────────────────────────────────────────┘

「109 個區域。平均偏移 38%。」馬可瑜的聲音不太穩。

貝葉需要算出「經濟損失的期望值」。
但先驗偏移不是直接的金額。
她需要一個轉換——

  ┌─ LOTUS（無意識統計學家定理）───────────────┐
  │                                              │
  │ E[g(X)] = Σ g(xᵢ) · P(xᵢ)                 │
  │                                              │
  │ 直覺：                                      │
  │   你知道 X 的分布（先驗偏移幅度）。          │
  │   你想算 g(X) 的期望值（損失金額）。         │
  │   你不需要先求出 g(X) 的分布——              │
  │   直接用 X 的分布來算就好。                  │
  │                                              │
  │ 就像：你知道每個員工的遲到分鐘數的分布。     │
  │ 你想算「因遲到造成的薪資損失」的期望值。     │
  │ 你不需要先算「薪資損失」的分布。             │
  │ 直接把「遲到→損失」的函數套上去就好。        │
  └──────────────────────────────────────────────┘

馬可瑜用精算模型定義了 g(x)——
把先驗偏移幅度轉換成經濟損失。

結果。

「全台灣。一年。
 先驗操控造成的保險錯誤定價：約 480 億台幣。
 社會風險認知偏差造成的間接損失：無法估計。」

高思遠摘下老花眼鏡。

「這不是學術問題了。」

→ LOTUS 的出現不是為了教學。
  是因為「他們真的需要這個工具」。
  讀者學到 LOTUS 是因為馬可瑜需要算錢。
```

## 第 27 章：變異數與分歧

### 基本資訊
```
字數目標：4,500-5,000 字
情緒色調：灰綠。團隊的第一次內部分歧。
機率概念：變異數、標準差
本章目標：團隊意見分歧。貝葉和蘇立恆的第一次衝突。
```

### 核心場景

```
「我們有了期望損害。現在需要衡量不確定性。」
蘇立恆說。

「什麼意思？」

「期望值只是平均。
 但平均不代表每個區域都是 38%。
 有的 8%。有的 72%。
 這個差異——」

  ┌─ 變異數與標準差 ─────────────────────────────┐
  │                                                │
  │ Var(X) = E[(X - μ)²] = E[X²] - (E[X])²      │
  │ σ = √Var(X)                                   │
  │                                                │
  │ 直覺：                                        │
  │   期望值 = 射擊的瞄準點（平均打在哪裡）       │
  │   變異數 = 子彈的散布範圍（打得多不準）       │
  │                                                │
  │ 如果 σ 很小 → 操控很均勻 → 有統一的策略      │
  │ 如果 σ 很大 → 操控很不均勻 → 可能多人執行    │
  └────────────────────────────────────────────────┘

σ = 0.15。
「不小。也不大。
 這意味著操控有統一策略，
 但在執行層面有一定的彈性。」

然後——衝突。

貝葉問蘇立恆：
「你在 Oracular 的時候就知道了。為什麼不公開？」

蘇立恆看著白板。
「因為我算過。公開的後果比隱瞞更糟。」

「什麼意思？」

「如果一夜之間全台灣的人都知道透鏡的數字可能是假的——
 你知道會發生什麼嗎？
 保險業崩盤。房價暴跌。犯罪率飆升。
 因為所有人的決策基礎——瞬間消失。
 那個期望損害——不是 480 億。
 是整個社會秩序。」

貝葉盯著他。

「所以你選擇讓所有人繼續被騙？」

「我選擇——找到一個不會造成更大損害的方式。
 但我沒找到。三年。沒找到。」

「那你現在為什麼要幫我？」

蘇立恆沉默了五秒。

「因為……你算了不一樣的東西。」
「什麼？」
「我只算了後果。你算了……值不值得。」

停頓。

→ 先驗（貝葉的信念：真相最重要）
  × 似然（蘇立恆的證據：後果很嚴重）
  → 後驗 = ？
  他們的分歧不是對錯。是兩種先驗的碰撞。
  這是全書的核心哲學問題。

高思遠在旁邊聽。
他沒有說話。
他只是把白板上的一行字指給兩個人看。
那行字是他最後一堂課留下的：
「質疑先驗。永遠質疑先驗。」

「包括你們自己的。」他補了一句。
```

## 第 28 章：大數法則的幽靈

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：深藍。像深海。越潛越深。
機率概念：大數法則（LLN）
本章目標：發現操控為什麼「不可能」被偵測——LLN 的保護傘。
```

### 核心場景

```
「我們知道了操控的全貌。但有一個問題——」
蘇立恆站在白板前。
「為什麼三年來沒有人發現？
 全台灣那麼多統計學家、數據科學家、精算師——
 沒有人看到嗎？」

「也許有人看到了。但——」貝葉想起林岳。

「不。不只是被消滅。
 是——大部分人根本不會覺得有問題。
 因為柏朗利用了一個東西。」

他在白板上寫：

  ┌─ 大數法則（LLN）──────────────────────────┐
  │                                              │
  │ 弱大數法則：                                │
  │ 隨著樣本數 n → ∞，                         │
  │ 樣本平均值 X̄ₙ 以機率收斂到真實期望值 μ。   │
  │                                              │
  │ P(|X̄ₙ - μ| > ε) → 0  當 n → ∞            │
  │                                              │
  │ 直覺：                                      │
  │ 如果你投硬幣投夠多次，                      │
  │ 正面的比例一定會接近 50%。                   │
  │ 樣本夠大，雜訊就會被平均掉。               │
  │                                              │
  │ 所以——如果你是統計局的人，                  │
  │ 你看到全台灣的犯罪率平均值「接近歷史值」—— │
  │ 你不會懷疑。因為 LLN 告訴你：這是正常的。  │
  └──────────────────────────────────────────────┘

「柏朗把操控的幅度控制在 LLN 允許的波動範圍內。
 單看任何一個區域——可能有點奇怪。
 但全部加起來——平均值跟歷史值一樣。
 因為他讓高偏移和低偏移的區域互相抵消。」

「所以整體平均看起來正常。」

「對。大數法則保護了他。
 因為任何人看大的趨勢——都會說『沒問題』。
 只有看個別區域的人才會發現。」

「只有校準師。」貝葉說。
「因為我們是唯一逐一看區域數據的人。」

「對。這就是為什麼林岳發現了。為什麼你發現了。
 而統計局、媒體、學者——都沒有。
 因為他們看的是 X̄。你看的是 Xᵢ。」

→ LLN 在課本上 = 「樣本夠大就準」。
  在這裡 = 「操控者把謊言藏在大數法則的庇護下」。
  讀者會發現：數學定理本身沒有善惡。
  它是一面盾。看誰拿著它。
```

## 第 29 章：中央極限定理的偽裝

### 基本資訊
```
字數目標：5,000-6,000 字
情緒色調：電藍。像看到了敵人的藍圖——震撼但清晰。
機率概念：中央極限定理（CLT）
本章目標：理解操控為什麼「統計上完美」——CLT 的偽裝。
```

### 核心場景

```
「LLN 解釋了為什麼平均值沒問題。
 但還有第二道防線。」蘇立恆說。

「第二道？」

  ┌─ 中央極限定理（CLT）─────────────────────────┐
  │                                                 │
  │ 不管 X 的原始分布是什麼——                     │
  │ 只要樣本夠大，                                 │
  │ 樣本平均值 X̄ₙ 近似常態分布：                   │
  │                                                 │
  │ X̄ₙ ~ N(μ, σ²/n)  大約成立（n 夠大時）        │
  │                                                 │
  │ 直覺：                                         │
  │ 不管你丟什麼形狀的數據進去——                   │
  │ 只要夠多——出來的平均值分布都長得像鐘形。      │
  │ 常態分布。那個永遠對稱的鐘。                   │
  │                                                 │
  │ 這意味著——                                     │
  │ 如果你的操控讓偏移落在 ±2σ/√n 以內——          │
  │ 任何人做統計檢定都會判定：「正常波動」。       │
  └─────────────────────────────────────────────────┘

「他把先驗偏移量精確控制在 CLT 的標準差以內。
 也就是說——不管你用什麼統計檢定——
 z-test、t-test、什麼都好——
 你得到的 p-value 都不會低於 0.05。
 因為偏移量在『正常隨機波動』的範圍裡。」

貝葉的手指在桌面上停了。

「所以他不只是修改了先驗。
 他精確計算了——修改多少才不會被偵測到。」

「對。他把操控偽裝成雜訊。
 而且偽裝得完美。
 因為他比任何人都懂 CLT。」

蘇立恆在白板上寫了一行字。字很大。紅色。

  「他把謊言穿上了常態分布的外衣。」

馬可瑜看著白板。
「所以——我們永遠抓不到他？」

「不。」蘇立恆說。
「CLT 有一個前提。」

他指著公式裡的一行：

  「X̄ₙ ~ N(μ, σ²/n)」

「σ²/n。分母是 n。
 n 越大，容許的波動越小。
 他的操控必須隨著樣本量增加而縮小——
 否則就會暴露。」

「所以——如果我們增加 n——」

「如果我們能在特定區域大量增加觀測數據——
 CLT 的允許範圍會縮小到連他的操控都藏不住。」

「但——我們怎麼增加觀測？」

「我們不能。但——
 我們可以在系統裡放一個東西。
 一組我們自己設計的數據。
 一個陷阱。」

→ CLT 從教學工具變成劇情引擎。
  它保護了壞人三年。
  但它的弱點——也是打敗壞人的突破口。
```

## 第 30-31 章：馬可夫不等式與切比雪夫陷阱

### 基本資訊
```
字數目標：各 5,000-5,500 字
情緒色調：金色。像鍛造武器——火光與鐵的碰撞。
機率概念：Markov 不等式、Chebyshev 不等式
本章目標：貝葉用不等式設計「統計蜜罐」——一個數學陷阱。
```

### 核心場景

```
第 30 章：
  高思遠站在白板前。
  「在設計陷阱之前，你需要知道兩個工具。」

  ┌─ Markov 不等式 ──────────────────────────────┐
  │                                                │
  │ 如果 X ≥ 0，對任何 a > 0：                   │
  │                                                │
  │ P(X ≥ a) ≤ E[X] / a                          │
  │                                                │
  │ 直覺：                                        │
  │ 一個非負隨機變數超過某個門檻的機率——          │
  │ 不會比「平均值/門檻」更大。                    │
  │                                                │
  │ 例子：                                        │
  │ 如果平均月薪 5 萬——                           │
  │ 月薪超過 50 萬的人比例 ≤ 5/50 = 10%。         │
  │ 很粗糙。但絕對成立。                          │
  └────────────────────────────────────────────────┘

  ┌─ Chebyshev 不等式 ──────────────────────────┐
  │                                               │
  │ P(|X - μ| ≥ kσ) ≤ 1/k²                     │
  │                                               │
  │ 直覺：                                       │
  │ 偏離平均值超過 k 個標準差的機率 ≤ 1/k²。    │
  │                                               │
  │ k=2: P(|X-μ| ≥ 2σ) ≤ 25%                    │
  │ k=3: P(|X-μ| ≥ 3σ) ≤ 11.1%                  │
  │ k=5: P(|X-μ| ≥ 5σ) ≤ 4%                     │
  │                                               │
  │ 不需要知道 X 的分布是什麼。                   │
  │ 只需要知道 μ 和 σ。                          │
  │ 這是一個「萬用上界」——對任何分布都成立。     │
  └───────────────────────────────────────────────┘

  「Markov 很粗糙。Chebyshev 好一些。
   但它們的共同優點是——對任何分布都成立。
   不管數據是 Normal、Gamma、Beta、什麼都好。
   只要偏移超過這些上界——就一定是異常的。」

  「一定？」

  「一定。數學上的一定。
   不是『很可能』。不是『在某些假設下』。
   是——數學證明的一定。」
```
