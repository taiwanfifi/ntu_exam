### v0.2 自評

#### 強項

```
1. 角色深度——超越預期
   - 柏朗的「灰色反派」完全實現：不是壞人，是痛苦的人
   - 漁夫出海的比喻把反派的動機和主角的反駁同時推到哲學高度
   - 白奶奶的「蔥花宇宙論」成為全書最強的情感錨
   - 蘇立恆的三層揭露（前員工→系統設計者→三年前就準備回來的人）節奏完美
   - 費雪從「冷血執行者」到「放行電話」的轉變有說服力
   - 小拉的成長線（從碎念機器→學會沉默→「先驗」資料夾→改名「後驗」）是隱藏的第二條感情線

2. 機率概念整合——故事驅動而非教學驅動
   - 每個概念都有明確的劇情功能
   - 先驗操控 → 核心陰謀（不是裝飾）
   - 假設檢定 → 法庭攻防（不是課堂）
   - 馬可夫鏈 → 終極 Boss 的結構（不是名詞）
   - Random Walk → 逃跑策略（不是公式）
   - Poisson Process → 意外的數學（不是定義）
   - v5.0 先驗選擇 → 故事的哲學結論（不是工具）

3. 名場面密度——30 個名場面全部落地
   - ★★★★ 天台漁夫對話（Ch55）
   - ★★★ 白奶奶的蔥花（Ch51「萬一你終於點頭」）
   - ★★★ 蘇立恆的「習慣」（Ch54）
   - ★★★ S.L.H. // She did.（Ch56）
   - ★★★ 白奶奶的石椅（Ch58「我擦了三年了」）
   - ★★★ 蘇立恆的 Type II Error 紙條（Ch23）
   - 所有名場面都有機率概念的底層支撐

4. 母題系統——五線交織
   - 先驗問題：從 Ch1 到 Ch60，每次問都有新意
   - 蔥油餅：白奶奶→貝葉的爸→蘇立恆買兩張→漁村的餅
   - 心率同步：小拉的追蹤→849 筆數據→「先驗」→「後驗」
   - 11 秒：關鍵停頓的計量
   - Beta 分布：高思遠的老 bug→夕陽的形狀

5. 結構完整性
   - 60 章，5 篇，每篇 12 章，每篇有獨立的情緒弧線
   - 起承轉合明確：校準→雜訊→信號→檢定→隨機
   - 機率概念由基礎到進階，學習曲線合理
   - 每一篇的結尾都鉤住下一篇
```

#### 待改進

```
1. 法庭戲的法律細節需要專業校對
   - 台灣的公民訴訟程序具體流程
   - 統計證據的法律可採性
   - 嚴重性：中。故事邏輯通順，但細節可能經不起法律人的檢驗
   → 留待 v0.4

2. 機率公式的數學準確性需逐一校對
   - 所有 ASCII box 裡的公式
   - 特徵值計算、卡方自由度、CI 公式
   - 嚴重性：高。教育小說的數學不能有錯
   → 留待 v0.4

3. 第二篇「雜訊」的情節密度偏高
   - 12 章塞了太多概念（S/N ratio, 暗數據, 倖存者偏差, 迴歸均值,
     相關≠因果, 辛普森悖論, 大數法則, 邊際分布）
   - 讀者可能消化不完
   → 留待 v0.5：考慮合併或簡化部分概念

   - 周副理（校準組主管）只出現過幾次
   - 林岳被校準壞之前的心理描寫可以更多
   - 林安（弟弟）只出現一次——但效果其實很好（Poisson event）
   → 留待 v0.5

5. 蘇立恆的「前 Oracular 員工」身份揭露節奏
   - 他的秘密分三次揭露（第二篇→第四篇→第五篇）
   - 但第一次揭露可能太突然
   → 留待 v0.5：在第一篇增加一兩個伏筆

6. 全書的「第一章」文學品質
   - 第一章是讀者決定要不要繼續讀的關鍵
   - 目前的精修稿格式（帶寫作指令的場景稿）需要轉成純小說文字
   → 留待 v0.5：第一章完整小說試寫
```
