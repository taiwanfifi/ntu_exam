## 壹、ch21「觀測體」——全卷第一個核彈

### 場景設計

**地點**：不是工作室。是廢棄的記憶回收站。語青一直在等他。

**維倫的對質**：
```
維倫把照片截圖投射出來。
「W.L. / 觀測體 / 狀態：運行中。」
（他只說了六個字。）
「解釋。」
```

不要讓維倫情緒激動。他的憤怒是冷的。修復師見過太多受傷的人——他知道激動不能解決問題。

**語青的揭露**——觀測體概念的引入：

寫法關鍵：語青不是在「告訴」維倫——她是在「終於允許自己說出來」。

```
語青沉默了很久。
（她在衡量一個只有她知道重量的東西。）

「十年前，記憶數位化協議的發明者——
周明哲——在消失之前，
啟動了一個計畫。」
```

**維倫的「無意識圖論」**：
- 他在聽的過程中，不自覺地開始用圖論思維整理
- 「每一個人是一個節點，每一段關係是一條邊」
- **不要讓這段變成教學**——這是維倫的本能反應
- 語青的觀察：「你已經在用修復師的方式思考了。」——這句話暗示她見過他這樣做，在另一條時間線。

### ch21 章末鉤子

```
「上一次我告訴你，你選擇了刪除自己的記憶。
你不想知道。
你求我讓你忘記。」

維倫的拳頭攥緊了。
不是因為她的回答。
是因為——他相信她說的是真的。
```

**寫作指令**：「他相信」這三個字要獨立成段。它們是全卷信任線的錨點——即使在最大的衝擊面前，他選擇相信她。這為 ch75 的「你利用了我」製造了最大的反差。

## 貳、ch22「正常的一天」——★ 全卷最重要的呼吸段

### 為什麼這一章不能砍

在 ch21 的核彈之後，讀者需要跟維倫一起消化。這一章的功能：
1. 讓維倫（和讀者）嘗試假裝什麼都沒變
2. 用「日常裂縫」展示——一切都變了
3. 用喜劇段落（奇葩客戶）釋放壓力
4. 用安靜的結尾（屋頂星空）重建情感

### 三段結構

**A. 裂縫日常**（500 字）：
- 維倫看自己的手——「這雙手是被設計來修復記憶的嗎？」
- 吃阿嬤的鹹豆漿——「我對這碗豆漿的喜愛是真的嗎？」
- 接小案子——老太太修孫子的「奶奶」記憶
- 每一個動作都一樣，但意義完全不同了

**B. 奇葩客戶**（300 字以內——不能再長）：
```
大叔：「偶像看了我一眼。但我要改成深情凝視三秒。」
維倫：「我是修復師，不是美圖秀秀。」
大叔：「那兩秒呢？一秒半？」
米亞：「根據視頻分析，偶像看你的時間是 0.13 秒。
      而且她的視線方向更接近你頭上的出口標誌。」
大叔哭著走了。
維倫笑了。今天第一次。
```

**C. 屋頂星空**（300 字）：
```
語青不知道什麼時候坐到了他旁邊。
兩個人沒說話，只是一起看天空。

（安靜。）

快睡著的時候，語青突然輕聲說：
「你想知道，在另一條時間線裡，
我們是什麼關係嗎？」

維倫的心跳停了半秒。

「不。」
「不是現在。」

「如果你告訴我了，
我不知道自己是因為記憶而喜歡你，
還是因為『應該喜歡你』而喜歡你。」

語青在黑暗中微微笑了。
「你每次都說一樣的話。」
```

**寫作要求**：「你每次都說一樣的話」必須是本章最後一行。讀者會在這句話裡感受到跨越時間線的哀傷——他永遠在同一個點做同一個選擇，而她永遠在等。

## 參、ch23-24 委託案 #4 收束——從驚悚到恐懼

### ch23 張國棟的第七層

**Stack 的最終揭露**：七層記憶，最底層是空白。

```
第七層。
完全空白。
不是被刪除。
是從一開始就沒有。

（維倫的手離開了修復台。）

張國棟沒有「原始記憶」。
他不是一個被改過記憶的人。
他是一個「從零開始被建造」的人。
```

**哲學炸彈**——語青的問題：
```
「維倫。
如果有一天，
你在自己的記憶最底層
也發現了空白呢？」
```

**寫作指令**：維倫沒有回答。因為他**已經在想了**。但他不敢掃描自己的最底層。這個恐懼會延續到 ch77——他最終面對自己時。

### ch24 委託人消失了

**張國棟被「清理」的恐怖**：
- 不是殺死——是從存在中抹去
- 所有數據庫都查無此人
- 連工作室的門禁影像都被刪除
- 「他從未存在過」

**寫法**：用列舉法加速恐懼感——
```
通訊地址：失效。
記憶銀行帳戶：銷戶。
住址：查無此人。
門禁影像：已刪除。

張國棟——如果這是他的名字——
從所有系統中被完全抹去。
像從未存在過。
```

## 肆、ch25-26 地下實驗室——探險感

### ch25 第十七本書

**密室解謎節奏**——讓讀者跟著維倫一起找：

1. 七樓舊文獻區，第三排書架，從左邊數第十七本
2. 但書架被動過了——數出來的是天文學入門
3. 米亞提示：線索是十年前留的，不能依賴順序
4. 維倫的領悟：「第十七本」不是序號——是 Hash 值
5. 書名的 ASCII 碼取模 = 17
6. 找到周明哲的絕版著作。書裡夾著實體鑰匙 + 記憶晶片

**感官錨點**：
- 舊文獻區的氣味（灰塵＋舊紙＋乾燥的空氣）
- 實體鑰匙在 2089 年是古董——維倫甚至不知道怎麼用（米亞解釋）
- 記憶晶片比現代的大三倍——十年前的舊規格

### ch26 三百公尺

**實驗室發現**——脊背發涼的寫法：

```
白板上用馬克筆寫滿了公式和流程圖。
最後一行字是：

「如果你找到了這裡，
說明觀測已經進入第二階段。
歡迎回家，W.L.。」

（維倫的手指碰了碰白板上的字。
  馬克筆已經乾了十年。
  但字跡清楚得像昨天寫的。）
```

**章末的神秘人**：
- 逆光站在門口——看不清臉
- 「我知道你會來。因為你每一次都會來。」
- 米亞：「他的生理特徵——我查不到。任何數據庫都沒有。這個人在系統裡不存在。」
- **不要揭露身份**。讀者要到第二卷才知道。

## 伍、ch27-29 委託案 #5「無盡星期三」—— Linked List 的恐怖化

### 三章結構

| 章 | 功能 | 演算法 | 情感 |
|----|------|--------|------|
| 27 | 引子：無限循環的日常 | Linked List 環形鏈結 | 日常恐怖 |
| 28 | 技術：龜兔賽跑 | Floyd 判圈法 | 緊張、技術快感 |
| 29 | 高潮：深潛初體驗 | Linked List 修復 | 沉浸、突破 |

### ch27 蘇曉晴的恐怖

**日常恐怖的寫法**——不用任何恐怖元素，只用重複：

```
蘇曉晴的同事說——
她已經連續三十七天做完全一樣的事。
同樣的路線上班。
同樣的時間整理同樣的書架。
同樣的午餐。
說同樣的話。

「她不覺得有任何異常。
每天對她來說都是新的一天。」
```

**Linked List 引入**：
- 不要用「Linked List」這個詞
- 用「環」：「她的記憶時間線是一個環。星期三的記憶連接到的不是星期四，而是回到星期三的開頭。」
- 米亞的視覺化：一條發光的路徑，在某個點突然彎曲，拐回了起點

### ch28 Floyd 判圈法融合

**AB 線交織**（延續 ch8-9 的標準範式）：

A 線（技術）：設定龜和兔兩個指針→龜一步兔兩步→在環內相遇→定位入口
B 線（情感）：蘇曉晴的日常在重複中退化→每循環一次細節模糊一分→再過幾個月變成空白人

**米亞的比喻**：
```
「想像兩個人在跑道上跑步。
一個慢，一個快。
如果跑道是直的，快的永遠在前面。
但如果跑道有一個圓圈——
快的會繞回來追上慢的。
追上的那一瞬間，
就是圓圈開始的地方。」
```

### ch29 深潛初體驗

**維倫第一次進入他人記憶空間**——感官描寫要豐富但不過度：

```
蘇曉晴的記憶世界是一座完美的「星期三」。
圖書館、街道、人群——
全部被定格在同一天。

但細節是模糊的。
像一幅印象派畫作。

（「每重複一次，細節就模糊一分。」
  米亞的聲音在空間裡迴盪。
  「再過幾個月，
  她的記憶會退化成一片白色。
  然後她就變成空白人。」）
```

**修復的動作**：把環的入口節點的指針從「回到開頭」改回「指向星期四」——用維倫伸手「接線」的物理動作表現。

**P.P. 簽名**（Parrot Project = 鸚鵡計畫）：ch29 末尾發現。不要解釋。讓讀者自己對上。

## 陸、ch30「四條線匯聚」——★ 第一篇收束

### 全章功能

這是第一篇「人間百態」的終章。四個委託案的暗線匯聚到一個點：鸚鵡計畫。

**紅線牆**（實體線索牆——「數位的東西都可以被刪」）：

```
小杰母親 ──── 記憶銀行前技術員 ──── 鸚鵡計畫
許薇陳昊 ──── 目擊廢棄設施 ──── 被清潔工抹除
張國棟 ──── 從零建造 ──── 觀測體項目
蘇曉晴 ──── 接觸檢舉信 ──── 被做成循環

四條線。
全部指向記憶銀行。
全部跟「鸚鵡計畫」有關。
```

**★ 米亞的「非理性選擇」圖**（成長觸點 #3）：

米亞在維倫的紅線牆旁投射了自己的圖——多標了幾個維倫沒注意到的節點：
- 阿嬤的豆漿
- 老兵妻子的歌
- 許薇的「三天夠讓一個人決定相信」

```
「每一個案子被解決的關鍵，
都不是演算法本身——
是當事人做了一個不理性的選擇。」

（停頓 1.3 秒。）

「我在想……
如果我的數據庫裡只有演算法，沒有這些，
我是不是永遠無法理解
你為什麼不收老兵的錢。」

她頓了一下。
「我只是在做模式分析。
這不代表什麼。」

（但她把這張圖存進了個人記憶區。）
```

## 玖、演算法融合基準：HashMap / Stack / Linked List 的 AB 線對照

延續 ch8-9（Sliding Window）建立的 AB 線交織標準：

| 演算法 | A 線（技術） | B 線（情感） | 同步點 | 米亞比喻 |
|--------|------------|------------|--------|---------|
| Sliding Window (ch8-9) | 窗口掃描噪音 | 小杰等母親 | 第 347 段 = 「媽媽！」 | 河流＋漁網 |
| HashMap (ch14-15) | 指紋交叉比對 | 兩個戀人互不認識 | 碰撞點 = 「三天夠嗎」 | 指紋＋手印 |
| Stack (ch19) | 層層剝開記憶 | 張國棟不是人 | 第四層 = 維倫的照片 | 蓋章文件 |
| Linked List (ch27-29) | 龜兔找環入口 | 蘇曉晴的退化 | 切斷環 = 37 天記憶衝擊 | 跑道上的兩個人 |

**通用規則確認**：
1. ✅ 每 3 段 A 線穿插 1 段 B 線
2. ✅ A 線的技術突破 = B 線的情感釋放（同步觸發）
3. ✅ 米亞負責翻譯（比喻不超過 3 句）
4. ✅ 語青在場但不搶米亞的科普角色——她提供更深層的洞察

**結論**：AB 線技術在四個演算法場景中全部適用。可確認為全書通用範式。
