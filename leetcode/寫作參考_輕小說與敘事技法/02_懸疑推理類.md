# 懸疑推理類 — 推理結構、佈線、伏筆技法

> 推理小說的核心能力：佈線、伏筆、節奏控制、讀者引導。
> 這些技法直接適用於演算法教學——每個演算法都是一個「推理過程」。

---

## 1. 《嫌疑犯X的獻身》(The Devotion of Suspect X) ⭐

| 項目 | 內容 |
|------|------|
| **作者** | 東野圭吾 |
| **類型** | 推理 / 數學驚悚（伽利略系列） |
| **規模** | ~20 章；伽利略系列跨 10+ 本 |
| **獎項** | 第 134 回直木獎；日本三大推理榜全部第一 |
| **維基** | https://zh.wikipedia.org/zh-tw/嫌疑犯X的獻身 |

### 寫作技法分析
- **倒敘推理的革新**：讀者知道兇手是誰，但不知道「詭計」是什麼
- **數學天才做主角**：石神哲哉的邏輯思維直接對應演算法的問題解決方式
- **核心詭計本質上是一個「誤導算法」**：石神建構了一個看似正確但指向錯誤答案的推理路徑
- **物理學家 vs 數學家**：兩種不同的思維方式對決

### 可借鑑的技法
> **「誤導讀者的推理鏈」**：讓讀者以為自己懂了 O(n²) 的暴力解就夠了，
> 然後揭露一個 O(n log n) 的優雅解——就像石神的詭計被湯川拆穿。

## 2. 《白夜行》(Journey Under the Midnight Sun)

| 項目 | 內容 |
|------|------|
| **作者** | 東野圭吾 |
| **類型** | 推理 / 心理懸疑 |
| **規模** | 13 章，~850 頁 |
| **參考** | https://www.hk01.com/藝文/809811/ |

### 寫作技法分析
- **「負空間」敘事法**：兩個主角的關係從頭到尾沒有直接描寫，只能從累積的證據中推論
- **跨越 19 年的長線佈局**：伏筆的回收跨度極長
- **pattern recognition**：讀者必須像演算法一樣，從分散的數據中識別出隱藏的模式

### 可借鑑的技法
> **「負空間」= 讓讀者自己推導結論**：
> 不要說「Binary Search 是 O(log n)」，而是給足線索讓讀者自己發現。
> 就像白夜行不直接寫兩個人在一起，但讀者自己會推理出來。

## 3. 《冰菓》/ 古典部系列 (Hyouka) ⭐

| 項目 | 內容 |
|------|------|
| **作者** | 米澤穗信 |
| **類型** | 日常之謎 / 青春推理 |
| **規模** | 7 卷（2001-至今）；累計銷量 220 萬+ |
| **平台** | 角川文庫；京都動畫改編動畫（2012） |
| **維基** | https://zh.wikipedia.org/wiki/古籍研究社系列 |

### 寫作技法分析
- **「日常之謎」開創者**：沒有殺人、沒有竊盜、沒有犯罪——只有「為什麼學校的窗戶每天都是開的？」
- **邏輯嚴謹度不因題材而降低**：即使是校園小謎團，推理過程也極為細密
- **「節能主義」主角**：折木奉太郎的「不做不必要的事」本身就是演算法優化的隱喻
- **角色驅動的推理**：千反田的好奇心（「我很好奇！」）是推動推理的引擎

### 可借鑑的技法
> **「日常之謎」= 演算法的日常應用**：
> 不需要「駭客入侵」這種大場面。
> 「食堂的排隊為什麼今天特別慢？」→ Queue 的概念。
> 「為什麼 WiFi 在某些位置特別差？」→ Graph 的概念。
> 小問題，大道理。這就是冰菓的精髓。

## 4. 《藥師少女的獨語》(The Apothecary Diaries)

| 項目 | 內容 |
|------|------|
| **作者** | 日向夏 |
| **類型** | 歷史推理 / 宮廷懸疑 |
| **規模** | 14+ 卷；銷量超 1200 萬 |
| **平台** | 小說家になろう（原始連載）；中文翻譯由角川台灣出版 |
| **維基** | https://zh.wikipedia.org/zh-hant/藥師少女的獨語 |

### 寫作技法分析
- **專業知識驅動推理**：貓貓用藥理學知識解決後宮謎團——每個案件本質上是「用 domain knowledge 做 debug」
- **受限環境設定**：宮廷 = 封閉問題空間（就像 LeetCode 的限制條件）
- **好奇心驅動型主角**：貓貓主動找問題解決，而不是被動等待

### 可借鑑的技法
> **「domain expertise 做 debug」模式**：
> 貓貓用藥理學 = 我們用演算法。
> 場景不同，但結構相同：面對異常現象 → 用專業知識分析 → 找出根因。

## 5. 《偵探已經，死了。》

| 項目 | 內容 |
|------|------|
| **作者** | 二語十 |
| **類型** | 推理 / 動作 / 戀愛喜劇 |
| **規模** | 12 卷 |
| **獎項** | 第 15 回 MF 文庫J 新人獎大獎；輕小說書店員推薦大獎 2020 第 1 名 |
| **維基** | https://zh.wikipedia.org/zh-tw/偵探已經，死了。 |

### 寫作技法分析
- **「倒敘結構」**：從偵探的死亡開始，回溯她的傳奇
- **多重語調混合**：喜劇、懸疑、動作、感傷——在同一部作品中無縫切換
- **遺產繼承主題**：主角繼承偵探的意志 = 初學者繼承前輩的知識

### 可借鑑的技法
> **語調切換的能力**：演算法教學小說不能一直緊繃，
> 需要在「燒腦推理」和「幽默日常」之間來回切換。
> 這部作品示範了怎麼做到自然。

## 6. 《Death Note》死亡筆記

| 項目 | 內容 |
|------|------|
| **作者** | 大場鶇（原作）/ 小畑健（作畫） |
| **類型** | 心理驚悚 / 超自然 / 貓鼠遊戲 |
| **規模** | 12 卷，108 章；動畫 37 集 |
| **平台** | 週刊少年Jump |

### 寫作技法分析
- **遞升陷阱結構**：每一章都是一步棋，而不是線性敘事
- **規則基礎世界觀**：死亡筆記的規則被嚴格定義，創造了可信的邏輯框架
- **貓鼠遊戲的智力對決**：Light vs L 的策略和反策略驅動劇情
- **角色會「說出推理過程」**：讓內部策略對讀者可見

### 可借鑑的技法
> **「規則基礎」+ 「策略對決」**：
> 演算法的「時間複雜度限制」就是規則。
> 「暴力解 vs 優化解」就是策略對決。
> 讓角色像 L 一樣把推理過程說出來。

## 7. 《Another》

| 項目 | 內容 |
|------|------|
| **作者** | 綾辻行人 |
| **類型** | 恐怖 / 推理 / 超自然 |
| **規模** | 35 章，4 卷 |
| **平台** | 角川書店；有續作《Another 2001》 |
| **維基** | https://en.wikipedia.org/wiki/Another_(novel) |

### 寫作技法分析
- **封閉空間恐怖**：只用一間教室/學校就建構出完整的恐怖推理
- **詛咒＝謎題引擎**：「班級詛咒」機制同時創造恐怖氛圍和推理框架
- **新本格推理先驅**：在超自然設定中維持公平的解謎邏輯

### 可借鑑的技法
> **「封閉空間」= 限制條件**：
> LeetCode 的每道題都有限制（時間/空間/輸入範圍）。
> 封閉空間就是限制條件的具象化。在限制中解題，才需要演算法。

| 項目 | 內容 |
|------|------|
| **作者** | 城平京（原作）/ 水野英多（作畫） |
| **類型** | 推理 / 懸疑 / 少年 |
| **規模** | 15 卷（1999-2005） |
| **平台** | 月刊少年GANGAN |
| **維基** | https://zh.wikipedia.org/zh-tw/少年偵探～推理之絆～ |

### 寫作技法分析
- **推理驅動的少年漫**：把偵探小說的解謎和少年漫的動作融合
- **長線懸念**：失蹤的哥哥串起所有單元事件
- **作者背景**：城平京是綾川哲也獎（重量級推理小說獎）決選入圍者

### 可借鑑的技法
> **「長線懸念串起單元事件」**：
> 我們的長線 = 「資料庫裡的隱藏圖/網路」。
> 每一章的單元 = 一個新的演算法概念。
> 用長線懸念讓讀者一直追下去。

## 推理技法總整理

| 技法 | 說明 | 代表作 |
|------|------|--------|
| **封閉空間** | 隔絕外部幫助，限制問題空間 | Another、館系列 |
| **公平線索** | 解謎所需的所有資訊都已給讀者 | 冰菓、柯南 |
| **不可靠敘事者** | 敘事者的視角隱藏真相 | 東野圭吾《惡意》 |
| **日常之謎** | 沒有犯罪的推理 | 冰菓 |
| **貓鼠遊戲** | 兩個聰明人的策略對決 | Death Note、約定的夢幻島 |
| **恐怖+推理融合** | 恐懼與解謎並存 | Another |
| **社會批判** | 推理作為社會議題的透鏡 | 白夜行 |
| **負空間敘事** | 不直接寫，讓讀者自己推導 | 白夜行 |
| **規則基礎世界觀** | 嚴格定義的規則創造邏輯框架 | Death Note |
