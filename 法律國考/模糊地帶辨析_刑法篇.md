# 刑法易混淆概念深度辨析 — 用工程師的精確思維切割模糊地帶

> 律師／司法官國考一試及二試專用
> 專為理工/程式背景的非法律轉職考生設計。

## 混淆組 1：竊盜(S320) vs 侵占(S335) vs 業務侵占(S336)

### 一眼判斷法

```
function 判斷罪名(行為人, 物品) {
    if (行為人在行為前「沒有持有」該物品) {
        // 物品在別人手上，你去「拿走」
        return "竊盜罪 §320";
    } else if (行為人在行為前「已經持有」該物品) {
        if (持有原因 === "業務上的關係") {
            return "業務侵占 §336";
        } else {
            return "普通侵占 §335";
        }
    }
}
```

| 判斷要素 | 竊盜 S320 | 普通侵占 S335 | 業務侵占 S336 |
|----------|-----------|---------------|---------------|
| 行為前持有關係 | 無（物品在他人支配下） | 有（合法持有他人之物） | 有（因業務而持有） |
| 核心行為 | 破壞他人持有，建立自己持有 | 易持有為所有 | 易持有為所有 |
| 白話 | 從別人那裡偷來 | 借了不還、占為己有 | 公司的東西私吞 |
| 刑度 | 5年以下 | 5年以下 | 6月以上5年以下 |

### 混淆原因

三個罪名的結果看起來一樣：「別人的東西跑到你手上了」。差別在於**起點**不同 -- 行為開始的那一刻，東西在誰手上？這就像 git 裡面的 `initial state` 決定了後續操作的性質。

### 深度解析

**關鍵概念：「持有」(Gewahrsam) = 事實上的支配管領力（physical control），不等於所有權（legal ownership）。**

- 竊盜 = **破壞**他人持有 → **建立**自己持有（像未授權 clone 別人的 private repo）
- 侵占 = **已持有** → 把持有**升級**成所有（像拿 read access 的 repo 改名說是自己的）
- 業務侵占 = 因業務有 admin access → 私吞（像把公司 production repo 賣給競爭對手）

### 案例對比

| 情境 | 罪名 | 判斷理由 |
|------|------|----------|
| 員工偷隔壁桌同事的筆電帶走 | 竊盜 | 筆電在同事支配下，員工無持有關係 |
| 員工把公司配發給他使用的筆電帶回家不還 | 普通侵占 | 員工已合法持有（公司借給他用），但非業務本身 |
| 超商店員把收銀機裡的營業款拿走 | 業務侵占 | 收銀機的錢是「因業務而持有」，店員因為收銀工作而支配那筆錢 |
| 快遞員把客戶託運的包裹據為己有 | 業務侵占 | 因運送業務而持有該包裹 |
| 撿到路邊皮夾放進自己口袋 | 侵占遺失物 S337 | 原持有人已喪失持有（不是從別人手上拿的） |

### 邊界案例

**案例：餐廳服務生把客人遺留在桌上的手機拿走**

分析路徑：
1. 客人離開餐廳後，手機的「持有」歸誰？
2. 如果客人剛走、手機還在桌上 → 客人尚未完全喪失持有 → 服務生拿走 = **竊盜**
3. 如果客人走了很久、餐廳打烊了 → 手機變成「遺失物」→ 服務生拿走 = **侵占遺失物 S337**
4. 如果餐廳規定服務生負責保管客人遺留物品 → 服務生因業務持有 → 拿走 = **業務侵占 S336**

**關鍵：持有關係是動態變化的，要看「行為時」的那一瞬間。**

### 考試陷阱

1. **陷阱題型**：題目說「甲受乙委託保管一幅畫，後來拿去賣掉」→ 學生寫竊盜 → **錯！是侵占**（甲已合法持有）
2. **進階陷阱**：題目說「甲是倉庫管理員，把倉庫裡的貨品偷走」→ 要判斷：甲對貨品有沒有「持有」？如果甲只是看門的（監視占有），持有歸公司 → **竊盜**；如果甲有權調度貨品（管領力）→ **業務侵占**
3. **最狠陷阱**：「甲向乙借車，到期不還」→ 不一定是侵占！如果甲只是單純拖延歸還（沒有「易持有為所有」的意思），可能只是民事糾紛，不成立犯罪

## 混淆組 2：詐欺(S339) vs 背信(S342)

### 一眼判斷法

```
function 判斷罪名(行為人, 被害人) {
    if (行為人與被害人之間有「信任/委任關係」) {
        if (行為人違背任務 && 損害委託人利益) {
            return "背信罪 §342";
        }
    }
    if (行為人對被害人施用「詐術」&& 被害人因此「陷於錯誤」&& 因此「處分財產」) {
        return "詐欺罪 §339";
    }
}
```

| 判斷要素 | 詐欺 S339 | 背信 S342 |
|----------|-----------|-----------|
| 關係基礎 | 不需要信任關係（騙陌生人也算） | 必須有委任/信賴關係 |
| 核心行為 | 施用詐術 → 使人陷於錯誤 → 處分財產 | 違背任務 → 損害本人利益 |
| 被害人角色 | 被害人被「騙」了，做了錯誤的決定 | 被害人根本不知情（或知情但被背叛） |
| 財產流向 | 被害人的財產轉移到行為人手上 | 委託人的財產受到損害（不一定轉移） |
| 白話 | 騙外人的錢 | 出賣老闆/委託人的利益 |

### 混淆原因

兩者都是「用不誠實的方式取得利益」。當「內部人」用欺騙手段取得公司財產時，到底是「騙公司的錢」（詐欺）還是「違背對公司的義務」（背信）？

### 深度解析

```
詐欺因果鏈：施用詐術 → 陷於錯誤 → 處分財產 → 取得利益（每步都不能斷）
背信構造：  委任關係 → 違背任務 → 損害委託人利益

一句話：詐欺=你「騙」別人把錢給你 / 背信=別人「信任你」管錢，你搞砸/私吞
```

### 案例對比

| 情境 | 罪名 | 理由 |
|------|------|------|
| 路人假裝募款騙你捐錢 | 詐欺 | 對陌生人施用詐術 |
| 公司經理把公款轉入自己帳戶 | 背信（可能也構成業務侵占） | 違背受委任管理公司資金的義務 |
| 業務員誇大產品功效賣給客戶 | 詐欺 | 對客戶施用詐術使其購買 |
| 公司經理私下收回扣，把合約給出價較高的廠商 | 背信 | 違背為公司爭取最佳利益的義務 |

### 邊界案例

**經典邊界：公司經理虛報費用（甲每月填假單據讓會計撥款 5 萬）**

```
詐欺分析：假單據=詐術 → 會計陷於錯誤 → 撥款=處分財產 → 成立 ✓
背信分析：甲受委任管理事務 → 虛報=違背任務 → 損害公司 → 成立 ✓
結論：兩者都可能成立 → 想像競合，從一重處斷
```

實務多數以**詐欺罪**為主（詐術是取得財產的核心手段），但也有判決論以背信。考試時兩個方向都要論述。

### 考試陷阱

1. **不是所有「公司內部的不誠實行為」都是背信**：背信要求「為他人處理事務」。如果你只是一個普通員工（不是受委任管理事務的人），就不符合背信的主體要件
2. **背信是「損害」本人利益，不一定要「取得」利益**：經理做了一個明知會虧損的投資決定 → 公司虧損 → 背信（即使經理本人沒有從中獲利）
3. **詐欺需要被害人「自己」處分財產**：如果被害人不是自己交出來的，而是行為人直接拿走的 → 不是詐欺，是竊盜

## 混淆組 3：強盜(S328) vs 搶奪(S325) vs 恐嚇取財(S346)

### 一眼判斷法

```
// 暴力程度光譜 (violence_level: 0 → 100)
function 判斷罪名(violence_level, timing) {
    if (violence_level >= 80) {
        // 暴力已達「至使不能抗拒」的程度
        return "強盜罪 §328";
    } else if (violence_level >= 30 && violence_level < 80) {
        if (timing === "趁不備、突然施力") {
            return "搶奪罪 §325";
        }
    } else if (violence_level >= 1 && violence_level < 30) {
        if (手段 === "威脅未來的惡害") {
            return "恐嚇取財 §346";
        }
    }
}
```

| 判斷要素 | 強盜 S328 | 搶奪 S325 | 恐嚇取財 S346 |
|----------|-----------|-----------|---------------|
| 暴力程度 | 最高：壓制被害人反抗 | 中等：趁不備、快速奪取 | 最低：以言語/行為威脅 |
| 被害人狀態 | 不能抗拒 | 來不及抗拒 | 心生恐懼而交付 |
| 關鍵用語 | 至使不能抗拒 | 趁人不備 | 恐嚇使人交付 |
| 白話 | 打到你不能動再搶 | 衝過來搶了就跑 | 嚇你叫你交出來 |
| 刑度 | 5年以上（重罪） | 6月以上5年以下 | 6月以上5年以下 |
| 財產移轉方式 | 行為人強行取走 | 行為人快速奪走 | 被害人自己交出 |

### 混淆原因

三者都是「用某種強制力取得他人財物」，差別在於**強制力的種類和程度**。這是一個光譜，不是截然二分的，所以邊界特別模糊。

### 深度解析

**強盜 = 「至使不能抗拒」**（依客觀情況，一般人會喪失反抗能力。判斷因素：工具、人數、體格、環境、暴力強度）

**搶奪 = 「乘人不備」**（重點是速度不是暴力。被害人「來不及反應」而非「被壓制」）

**恐嚇取財 = 「以惡害通知使人心生恐懼」**（威脅未來惡害，被害人因恐懼而「自己交出」財物。注意：財物是被害人自己交的，跟強盜「行為人強取」不同）

### 案例對比

| 情境 | 罪名 | 理由 |
|------|------|------|
| 持刀架在脖子上：「把錢拿出來」 | 強盜 | 被害人已不能抗拒 |
| 騎機車從後方搶走路人手上的皮包 | 搶奪 | 趁不備快速奪取 |
| 寫信給商店：「不交 100 萬就炸你店」 | 恐嚇取財 | 以未來惡害威脅，使人交付 |
| 三個大漢圍住一個人：「識相的把錢包拿出來」 | 強盜 | 人數優勢已達不能抗拒 |
| 趁老人在公園打瞌睡，拿走旁邊的手提袋 | 竊盜 | 甚至不是搶奪（沒有「奪取」，而是趁睡偷走） |

### 邊界案例

**經典邊界：搶奪升級為強盜（準強盜 S329）**

情境：甲騎車搶走乙的皮包，乙追上拉住不放，甲推倒乙後逃逸。

```
原始行為 = 搶奪 → 甲當場施強暴以脫免逮捕 → 觸發 §329 準強盜
結論：以強盜罪論處（5年以上！）

§329 觸發條件：竊盜/搶奪 + 當場 + 強暴脅迫 + 目的（防護贓物/脫免逮捕/湮滅罪證）
```

### 考試陷阱

1. **搶奪 vs 竊盜的分界**：如果被害人根本不知道東西被拿走（例如扒手從口袋偷走皮夾）→ 竊盜，不是搶奪。搶奪要有「公然奪取」的動作
2. **恐嚇取財 vs 強盜的分界**：關鍵在於威脅的「現實性」。「我現在就砍你」+ 手持武器 → 強盜。「你如果不給錢，改天讓你好看」→ 恐嚇取財
3. **準強盜的時間限制**：必須是「當場」施強暴脅迫。如果小偷逃走後隔天被警察找到才反抗 → 不符合「當場」→ 不成立準強盜

## 混淆組 4：殺人(S271) vs 傷害致死(S277 II)

### 一眼判斷法

```
function 判斷罪名(行為人的主觀意思, 結果) {
    if (結果 === "被害人死亡") {
        if (行為人主觀上有「殺人故意」) {
            return "殺人罪 §271（既遂）";
        } else if (行為人主觀上僅有「傷害故意」) {
            return "傷害致死 §277 II（加重結果犯）";
        } else if (行為人連「傷害故意」都沒有) {
            return "過失致死 §276";
        }
    }
}
```

| 判斷要素 | 殺人 S271 | 傷害致死 S277 II |
|----------|-----------|-----------------|
| 主觀犯意 | 殺人故意（直接或間接） | 僅有傷害故意 |
| 加重結果 | 死亡本身就是故意的目標 | 死亡是「超出預期」的加重結果 |
| 故意內容 | 「我要他死」或「死了也無所謂」 | 「我只是要打他/教訓他」 |
| 刑度 | 10年以上、無期、死刑 | 無期徒刑或7年以上 |
| 白話 | 蓄意殺人 | 打人打到死（沒想殺他） |

### 混淆原因

結果完全一樣（被害人都死了），差別純粹在於**行為人腦子裡在想什麼**。但你又不是讀心者，怎麼判斷他在想什麼？這就是為什麼這組混淆是考試最愛考的。

### 深度解析

**殺人故意的認定 -- 法院從客觀事實反推主觀犯意：**

```
殺人故意推斷因素（綜合判斷，不是只看一個）：
  攻擊部位 → 要害（頭、頸、心臟）= 傾向殺人故意
  使用工具 → 致命武器（刀、槍）= 傾向殺人故意
  攻擊次數 → 反覆不停手 = 強烈傾向殺人故意
  事後行為 → 棄之不顧逃逸 = 傾向殺人 / 叫救護車 = 傾向非殺人
```

### 案例對比

| 情境 | 罪名 | 判斷關鍵 |
|------|------|----------|
| 持刀對準心臟猛刺三刀 | 殺人 | 要害部位+致命工具+多次攻擊 |
| 推了一把，被害人撞到頭死亡 | 傷害致死 | 推人的力道通常不致死，死亡是意外結果 |
| 拿棒球棍往頭上猛打十幾下 | 殺人 | 攻擊要害+致命工具+反覆攻擊+不停手 |
| 打架時一拳打到太陽穴，對方倒地不治 | 傷害致死 | 拳頭一擊，通常不預期致死 |
| 把安眠藥磨碎放入對方飲料中，劑量足以致命 | 殺人 | 事先準備+致命劑量=預謀殺人 |

### 邊界案例

**經典邊界：酒吧鬥毆，甲拿啤酒瓶往乙頭上打，乙倒地不治。**

```
殺人故意方：頭部=要害 + 玻璃瓶=有殺傷力 → 間接故意
傷害故意方：鬥毆常態≠殺人 + 非典型致命武器 + 只打一下 → 僅傷害故意

實務多數：傷害致死（單次攻擊+非典型武器+鬥毆脈絡）
但：打很多下/倒地後繼續打 → 升級為殺人故意
```

### 考試陷阱

1. **不要只看結果**：被害人死了不代表行為人有殺人故意。考試就是要你分析「故意的內容」
2. **間接故意也是故意**：行為人不需要「我就是要殺你」，只要「死了也無所謂」就夠了
3. **傷害致死是「加重結果犯」**：要求行為人對加重結果（死亡）至少有「過失」。如果死亡結果完全無法預見，連傷害致死都不成立
4. **最常見錯誤**：看到「持刀」就寫殺人。不一定！拿水果刀在手臂上劃一刀 → 可能只是傷害故意。要綜合判斷所有因素

## 混淆組 5：公然侮辱(S309) vs 誹謗(S310)

### 一眼判斷法

```
function 判斷罪名(言論內容) {
    if (言論是「抽象謾罵」，沒有具體事實) {
        // 例如：「你是垃圾」「廢物」「不要臉」
        return "公然侮辱 §309";
    } else if (言論是「指摘或傳述具體事實」) {
        // 例如：「你偷公司的錢」「你跟老闆有一腿」
        return "誹謗罪 §310";
    }
}
// 注意：兩者都要求「公然」（不特定人或多數人得以共見共聞）
```

| 判斷要素 | 公然侮辱 S309 | 誹謗 S310 |
|----------|--------------|-----------|
| 言論性質 | 抽象的價值判斷/謾罵 | 具體的事實指摘 |
| 內容特徵 | 無法驗證真偽 | 可以驗證真偽 |
| 例句 | 「你這個人渣」 | 「你上個月偷了公司十萬塊」 |
| 公然要件 | 需要 | 需要（散布於眾） |
| 真實性抗辯 | 不適用（價值判斷無真偽） | 適用（S310 III：能證明為真且與公益有關→不罰） |

### 混淆原因

兩者都是「用嘴巴傷害別人的名譽」。差別在於你說的內容是「空泛的罵」還是「具體的指控」。但實際上很多話同時包含兩者，這就是模糊地帶。

### 深度解析

**「抽象」vs「具體」的分界線：**

```
可驗證真偽 = 誹謗
不可驗證真偽 = 侮辱

「你是小偷」→ 模糊地帶！
  如果語境是在指控某次具體偷竊 → 誹謗
  如果只是單純罵人 → 侮辱
```

**誹謗罪的特殊防火牆 -- S310 III 真實性抗辯：**

如果能證明你說的是真的，而且跟公共利益有關 → 不罰。這是為了保護言論自由（特別是新聞媒體的監督功能）。

但侮辱罪沒有這個防火牆，因為「垃圾」「人渣」這種話沒有真假可言。

### 案例對比

| 言論 | 罪名 | 理由 |
|------|------|------|
| 在臉書公開寫：「某某人是社會敗類」 | 公然侮辱 | 抽象謾罵、無具體事實 |
| 在臉書公開寫：「某某人上個月挪用公款 50 萬」 | 誹謗 | 指摘具體事實 |
| 在公司群組說：「那個白痴主管又來了」 | 公然侮辱 | 「白痴」是抽象貶抑 |
| 在公司群組說：「主管靠裙帶關係升官的」 | 誹謗 | 指摘具體事實（靠關係升官） |
| 面對面罵一個人「你去死啦垃圾」（只有兩人在場） | 可能不成立 | 不符合「公然」要件（沒有第三人） |

### 邊界案例

**案例：在臉書寫「某某公司老闆是慣老闆，壓榨員工的垃圾」**

```
拆解分析：
  「慣老闆」→ 暗示具體事實（慣性壓榨勞工）→ 偏誹謗
  「壓榨員工」→ 指摘具體行為 → 誹謗
  「垃圾」→ 抽象謾罵 → 侮辱

結論：同時構成公然侮辱 + 誹謗 → 想像競合，從一重（誹謗）處斷
```

### 考試陷阱

1. **「公然」的認定**：不是要求「現場很多人」，而是「不特定人或多數人得以共見共聞」。在網路上公開發文 = 公然。在只有兩人的密室 = 不公然
2. **誹謗罪的阻卻違法**：S311 列舉了四種免責情形（善意發表言論），包括自衛、自辯、保護合法利益的善意評論等。考試愛考「記者報導」是否構成誹謗
3. **憲法法庭 113 年憲判字第 3 號**（2024 年）：宣告公然侮辱罪部分違憲，限縮處罰範圍。考試可能會出相關題目，注意最新發展

## 混淆組 6：偽造文書各罪區分(S210-S215)

### 一眼判斷法

```
function 判斷偽造文書罪名(文書類型, 行為方式) {
    // Step 1: 判斷文書類型
    let docType;
    if (文書製作者 === "公務員職務上製作") docType = "公文書";
    else docType = "私文書";

    // Step 2: 判斷行為方式
    let actType;
    if (行為 === "無中生有，冒用他人名義製作") actType = "偽造";
    else if (行為 === "在真正文書上竄改內容") actType = "變造";

    // Step 3: 對應罪名
    if (docType === "公文書" && actType === "偽造") return "偽造公文書 §211（1-7年）";
    if (docType === "公文書" && actType === "變造") return "變造公文書 §211（1-7年）";
    if (docType === "私文書" && actType === "偽造") return "偽造私文書 §210（5年以下）";
    if (docType === "私文書" && actType === "變造") return "變造私文書 §210（5年以下）";
}
```

| 維度一：文書類型 | 公文書 | 私文書 | 特種文書 |
|-----------------|--------|--------|----------|
| 定義 | 公務員職務上製作 | 一般人製作 | 護照、旅券、免許證等 |
| 對應偽造罪 | S211 | S210 | S212 |
| 刑度 | 1-7年 | 5年以下 | 5年以下 |

| 維度二：行為方式 | 偽造 | 變造 |
|-----------------|------|------|
| 定義 | 無製作權人，冒名製作 | 有權或無權人，竄改真正文書內容 |
| 白話 | 假造一份不存在的文件 | 改動一份真正文件的內容 |
| 程式比喻 | 偽造 = 在別人名下 create new file | 變造 = 未經授權 edit existing file |

| 維度三：有形偽造 vs 無形偽造 |  |  |
|---------------------------|---|---|
| 有形偽造 | 冒用他人名義製作文書 | 文書的「作者」是假的 |
| 無形偽造 | 有製作權限的人，登載不實內容 | 文書的「內容」是假的 |
| 對應條文 | 一般偽造文書 S210/211 | 公務員登載不實 S213 / 使公務員登載不實 S214 |

### 混淆原因

偽造文書是一個「家族」，包含好幾個罪名，排列組合容易搞混。抓住三個維度（文書類型 x 行為方式 x 有形/無形），就能用矩陣定位。

### 深度解析

```
§213 公務員登載不實：公務員自己在職務文書上寫假的（例：戶政人員明知假結婚仍登記）
§214 使公務員登載不實：一般人利用公務員不知情而登載（例：拿假地址證明辦遷入）
注意：§214 的對象必須是「公務員」，保險公司人員不算
```

### 案例對比

| 情境 | 罪名 | 理由 |
|------|------|------|
| 冒用別人的名字簽一張借條 | 偽造私文書 S210 | 冒名製作私文書 |
| 把畢業證書上的「肄業」改成「畢業」 | 變造特種文書 S212 | 竄改真正文書的內容 |
| 自己刻一個假的市政府公章蓋在假文件上 | 偽造公文書 S211 | 冒用公務機關名義製作 |
| 戶政人員收賄後幫假結婚登記 | 公務員登載不實 S213 | 公務員在職務文書上登載不實 |
| 拿假地址證明讓戶政人員辦遷入 | 使公務員登載不實 S214 | 利用公務員不知情而登載 |

### 考試陷阱

1. **偽造私文書需要「足以生損害於公眾或他人」**：如果偽造一張沒有任何法律效力的文件（例如寫一封假的情書），不成立
2. **偽造 vs 變造的區分**：如果改動幅度太大，導致文書的「同一性」喪失 → 不是變造，是偽造
3. **S215 業務上文書登載不實**：主體限定為「從事業務之人」。醫師開不實診斷書 → S215，不是 S213（因為醫師不是公務員）

## 混淆組 8：恐嚇危害安全(S305) vs 恐嚇取財(S346) vs 擄人勒贖(S347)

### 一眼判斷法

```
function 判斷恐嚇類型(行為, 目的) {
    if (行為.includes("擄人") && 目的 === "勒贖") {
        return "擄人勒贖 §347（無期徒刑或7年以上）";
    } else if (目的 === "取得財物或財產上利益") {
        return "恐嚇取財 §346（6月以上5年以下）";
    } else if (目的 === "單純恐嚇、沒有財產目的") {
        return "恐嚇危害安全 §305（2年以下）";
    }
}
```

| 判斷要素 | 恐嚇危安 S305 | 恐嚇取財 S346 | 擄人勒贖 S347 |
|----------|--------------|---------------|---------------|
| 目的 | 無財產目的 | 取得財物 | 取得贖金 |
| 手段 | 以加害生命等事恐嚇 | 恐嚇使人交付財物 | 擄走人質+要求贖金 |
| 白話 | 嚇你讓你害怕 | 嚇你逼你給錢 | 綁架你的人要贖金 |
| 刑度 | 2年以下 | 6月以上5年以下 | 無期或7年以上（極重） |
| 被害人 | 被恐嚇者本人 | 交付財物者 | 被擄者+交付贖金者 |

### 混淆原因與深度解析

三者共通元素是「恐嚇」，差別在**目的**和**手段升級程度**：

- **S305**：純恐嚇，讓你害怕就好（例：前男友傳「讓你不得好死」）
- **S346**：恐嚇 + 取財目的（例：「不給 100 萬就公布私密照」）
- **S347**：手段升級到擄人，用人質安危要脅（刑法中刑度最重的罪名之一）

### 案例對比

| 情境 | 罪名 | 理由 |
|------|------|------|
| 傳訊息：「小心你的家人」 | 恐嚇危安 S305 | 純恐嚇、無財產目的 |
| 傳訊息：「給我 50 萬，不然公布你的祕密」 | 恐嚇取財 S346 | 恐嚇+取財目的 |
| 把富商的小孩帶走，打電話要 3000 萬 | 擄人勒贖 S347 | 擄人+勒贖 |
| 堵住路人：「把錢包給我，不然揍你」 | 看暴力程度 | 若達不能抗拒→強盜；若只是口頭威脅→恐嚇取財 |

### 邊界案例

**案例：甲傳訊息給乙「把 10 萬匯給我，不然我把你外遇的證據寄給你老婆」**

```
分析：
  手段 = 恐嚇（威脅揭露隱私）
  目的 = 取得財物（10 萬元）
  → 恐嚇取財 §346 ✓

  但如果乙確實有外遇，甲的威脅內容是「真實的」→ 影響罪名成立嗎？
  答案：不影響。恐嚇取財看的是「手段」（用恐嚇方式取財），
        不管威脅內容是真是假。
```

### 考試陷阱

1. **擄人勒贖 vs 強盜的區分**：如果綁架的目的不是「勒贖」而是「搶被害人身上的錢」→ 強盜+私行拘禁，不是擄人勒贖
2. **恐嚇取財 vs 詐欺**：如果被害人是因為「被騙」而交錢 → 詐欺；因為「被嚇」而交錢 → 恐嚇取財。兩者的心理狀態不同（錯誤 vs 恐懼）
3. **擄人勒贖的既遂時點**：擄人的行為完成就既遂，不需要真的拿到贖金

## 混淆組 9：故意與過失的四分法

### 一眼判斷法 -- 2x2 矩陣

```
                    認知面（有沒有預見？）
                    ┌─────────────┬─────────────┐
                    │   有預見     │   無預見     │
意志面  ────────────┼─────────────┼─────────────┤
  接受結果          │ 間接故意     │   （不存在） │
  （不違背本意）    │ §13 II      │              │
────────────────────┼─────────────┼─────────────┤
  追求結果          │ 直接故意     │   （不存在） │
  （有意使其發生）  │ §13 I       │              │
────────────────────┼─────────────┼─────────────┤
  排斥結果          │ 有認識過失   │ 無認識過失   │
  （確信不發生）    │ §14 II      │ §14 I        │
                    └─────────────┴─────────────┘
```

用機率和態度的雙軸來理解：

| 類型 | 認知（P = 主觀認為結果發生的機率） | 意志（對結果的態度） | 白話 |
|------|----------------------------------|--------------------|----- |
| 直接故意 | P ≈ 1（確定會發生） | 積極追求 | 「我就是要他死」 |
| 間接故意 | 0 < P < 1（可能會發生） | 接受、不在乎 | 「死了也無所謂」 |
| 有認識過失 | 0 < P < 1（可能會發生） | 排斥、確信不會 | 「應該不會出事吧」 |
| 無認識過失 | P = 0（根本沒想到） | 沒有態度（未認知） | 「完全沒想到會這樣」 |

### 混淆原因

**間接故意 vs 有認識過失** 是刑法中最經典、最難判斷的分界線。兩者在「認知面」完全相同（都有預見結果可能發生），差別純粹在「意志面」（你對這個結果的態度）。

### 深度解析

**間接故意 vs 有認識過失 -- 用程式比喻**

```python
# 間接故意：酒駕上路，「撞到人也無所謂」
driver.assess_risk("可能撞死人")  # 有認知
driver.attitude = "whatever"      # 不違背本意 → 間接故意

# 有認識過失：疲勞駕駛，「應該撐得住」
driver.assess_risk("可能撞到人")  # 有認知
driver.attitude = "不會啦"        # 確信不發生 → 有認識過失
```

**區分的關鍵判斷指標：行為人有沒有採取「避免結果發生」的措施？**
- 有採取合理措施（但失敗）→ 傾向「有認識過失」
- 完全沒有採取任何措施 → 傾向「間接故意」

### 案例對比

| 情境 | 類型 | 分析 |
|------|------|------|
| 甲要殺乙，拿槍對準乙射擊 | 直接故意 | 追求乙死亡的結果 |
| 甲在鬧區對仇人開槍，不管路人死活 | 間接故意（對路人） | 預見可能打到路人，但不在乎 |
| 甲酒駕上路，覺得「應該不會出事」 | 有認識過失 | 預見可能肇事，但確信不會 |
| 甲正常開車，不知道煞車壞了，撞死人 | 無認識過失 | 根本沒預見（但該定期檢查） |

### 邊界案例

**經典邊界：飆車族在市區飆車撞死路人**

```
檢方：市區飆車極可能撞人 + 明知卻不在乎 → 間接故意
辯方：目的是追求刺激 + 自認技術好不會撞到 → 有認識過失

法院看「有沒有採取避免措施」：
  選空曠路段/深夜車少時飆 → 傾向有認識過失
  尖峰時段鬧區完全不減速 → 傾向間接故意
```

### 考試陷阱

1. **不要把「間接故意」和「過失」混為一談**：間接故意仍然是「故意」，適用故意犯的所有規定（包括未遂犯處罰）
2. **有認識過失的「確信」必須有合理基礎**：如果行為人說「我確信不會出事」但客觀上完全不合理 → 法院可能不採信，改認定間接故意
3. **無認識過失的判斷標準**：「應注意、能注意、而不注意」→ 三個要件缺一不可

## 混淆組 10：正當防衛(S23) vs 緊急避難(S24)

### 一眼判斷法

```
function 判斷阻卻違法(行為對象) {
    if (行為對象 === "攻擊者本人") {
        return "正當防衛 §23";
    } else if (行為對象 === "無辜第三人或第三人的財物") {
        return "緊急避難 §24";
    }
}
// 核心區別：你是「反擊壞人」還是「犧牲無辜的人」
```

| 判斷要素 | 正當防衛 S23 | 緊急避難 S24 |
|----------|-------------|-------------|
| 侵害來源 | 不法侵害者 | 危難（人為或自然） |
| 行為對象 | 對抗攻擊者本人 | 犧牲無辜第三人的利益 |
| 手段限制 | 必要性（較寬鬆） | 相當性（較嚴格：利益衡量） |
| 時間要件 | 現在的不法侵害 | 緊急危難 |
| 過當效果 | 得減輕或免除其刑 | 得減輕或免除其刑 |
| 白話 | 正面迎戰壞人 | 拿無辜的人當擋箭牌 |

### 混淆原因

兩者都是「在危急情況下做了通常違法的事，但法律不處罰你」。差別在於**你對付的對象是誰**。

### 深度解析

**正當防衛的要件清單：**
`[x]` 現在的不法侵害 `[x]` 防衛行為針對侵害者 `[x]` 主觀防衛意思 `[x]` 手段必要性

**緊急避難的要件清單：**
`[x]` 緊急危難存在 `[x]` 不得已（無其他合法手段） `[x]` 利益衡量通過 `[x]` 對象通常是第三人

**「利益衡量」是緊急避難最關鍵的限制：**

```
if (避免的損害 > 造成的損害) → 合法的緊急避難
if (避免的損害 ≈ 造成的損害) → 有爭議
if (避免的損害 < 造成的損害) → 不成立（避難過當）
```

### 案例對比

| 情境 | 性質 | 理由 |
|------|------|------|
| 甲拿刀砍你，你拿椅子打回去 | 正當防衛 | 對抗攻擊者本人 |
| 被A追殺，搶了路人B的摩托車逃跑 | 對A是防衛問題，對B是緊急避難 | 損害B的財產來避免自己的生命危險 |
| 大樓失火，你踹開鄰居家的門逃生 | 緊急避難 | 損害鄰居的財產（門）來保全自己的生命 |
| 有人拿槍指著你叫你去打第三人 | 緊急避難 | 你打第三人是為了保全自己的生命 |

### 邊界案例

**經典邊界：被A追殺，搶了路人B的摩托車逃跑**

```
對 A：甲有正當防衛權，但甲選擇逃跑而非反擊 → 未行使防衛
對 B：甲搶走 B 的摩托車 → 構成搶奪，但主張緊急避難
      利益衡量：甲的生命 > B 的財產 → 成立緊急避難 → 阻卻違法
```

### 考試陷阱

1. **正當防衛不能對付合法行為**：警察合法逮捕你，你不能主張正當防衛來反抗
2. **正當防衛沒有「利益衡量」限制**（通說）：對方偷你一顆蘋果，你打他一拳 → 只要是「必要的防衛手段」就可以（但不能過當）
3. **緊急避難有嚴格的「補充性」要件**：必須是「不得已」，如果有其他合法手段可以避免危難，就不能主張緊急避難
4. **「自招危難」不能主張緊急避難**：你自己故意引發危險，不能再犧牲別人來避難

## 混淆組 11：未遂 vs 預備 vs 陰謀

### 一眼判斷法

```
// 用軟體開發生命週期比喻犯罪進程
const crimeLifecycle = {
    "陰謀":  "寫 spec / 討論需求 → 多人合意要做壞事",
    "預備":  "寫 code / 建構開發環境 → 準備犯罪工具或計畫",
    "著手":  "deploy to staging → 開始實行構成要件行為",
    "未遂":  "deploy to production 但 server crash → 已著手但結果沒發生",
    "既遂":  "production running smoothly → 犯罪完成、結果發生"
};
```

| 階段 | 定義 | 處罰原則 | 比喻 |
|------|------|---------|------|
| 陰謀 | 二人以上謀議犯罪 | 原則不罰，例外處罰（重罪） | 一群人在會議室討論要做壞事 |
| 預備 | 為實行犯罪而準備 | 原則不罰，例外處罰（重罪） | 買了工具、勘查地形、寫計畫 |
| 著手 | 開始實行犯罪行為 | 進入「未遂」的領域，可罰 | 開始動手做了 |
| 未遂 | 已著手而不遂 | 依 S25 處罰（得減輕） | 做了但沒成功 |
| 既遂 | 犯罪結果發生 | 依本刑處罰 | 成功了 |

### 混淆原因

**核心難題是「著手」的判斷。** 什麼時候從「預備」跨到「著手」？這條線決定了行為人是否進入可罰的範圍。

### 深度解析

**「著手」的判斷標準（通說：主客觀混合說）：**
行為人依其犯罪計畫，行為已對法益產生直接危險時 = 著手。

```
甲計畫在乙的飲料中下毒：
  查毒物資訊 → 預備 | 購買砒霜 → 預備 | 帶到乙家 → 預備
  把砒霜倒入水杯 → 著手！（法益直接受危險）
  乙喝下毒水 → 實行中 | 乙死亡 → 既遂
```

### 考試陷阱

1. **原則上只有「未遂」以上才可罰**：預備和陰謀只有法律有特別規定才處罰（例如 S271 II 殺人罪的預備犯）
2. **不是所有罪名都處罰未遂**：過失犯沒有未遂的問題（過失的本質就是「不想要結果發生」，不存在「未遂」）
3. **考試愛考的「著手時點」**：什麼時候算「開始入室竊盜」？打開窗戶的時候？伸手進去的時候？進入房間的時候？→ 通說認為是「伸手進去拿東西的時候」

## 混淆組 12：不能未遂 vs 普通未遂 vs 中止未遂

### 一眼判斷法

```
function 判斷未遂類型(失敗原因) {
    if (失敗原因 === "行為人自己放棄") {
        return "中止未遂 §27（必減輕或免除其刑）";
    } else if (失敗原因 === "客觀上根本不可能成功") {
        if (手段或對象在本質上不可能達成結果) {
            return "不能未遂 §26（不罰）";
        }
    } else {
        return "普通未遂 §25（得減輕其刑）";
    }
}
```

| 類型 | 失敗原因 | 法律效果 | 程式比喻 |
|------|---------|---------|---------|
| 普通未遂 | 外在因素阻止（被害人逃走、被路人阻止） | 得減輕其刑 | deploy 失敗因為 server 被擋 |
| 中止未遂 | 行為人出於自己的意思放棄 | 必減輕或免除其刑 | 自己按了 rollback |
| 不能未遂 | 行為本身根本不可能成功 | 不罰 | code 根本編譯不過，不可能跑起來 |

### 深度解析

**中止未遂 S27 = 已著手 + 「因己意」中止。判斷用 Frank 公式：**

```
「能達目的而不欲」→ 中止未遂（✓算了不做了 ✓我下不了手）
「欲達目的而不能」→ 普通未遂（✗警察來了 ✗門鎖打不開）
```

**不能未遂 -- S26：行為不能發生犯罪之結果，又無危險者，不罰。**

```
關鍵：不是所有「不可能成功」都不罰，必須「又無危險」

拿玩具槍搶銀行：外觀像真槍→有危險→普通未遂 / 一看就是假的→無危險→不罰
對稻草人開槍以為是仇人 → 客觀無人在場 → 不能未遂 → 不罰
用糖水以為是毒藥下毒 → 不可能致死又無危險 → 不能未遂 → 不罰
```

### 考試陷阱

1. **中止未遂是「必」減免，普通未遂是「得」減免**：一字之差，效果天差地別
2. **中止未遂的「己意」判斷很主觀**：考題常設計「聽到警車聲嚇跑」→ 是「己意」還是「被迫」？→ 要看行為人是否真的被阻止（只是聽到聲音但警察還沒到 → 可能仍算己意中止，有爭議）
3. **不能未遂的判斷標準**：S26 要求「又無危險」，這裡的「危險」採「客觀危險說」（一般人在事後從客觀角度判斷有無危險）

## 混淆組 13：共同正犯 vs 教唆犯 vs 幫助犯

### 一眼判斷法

```
// 犯罪團隊的角色分配
function 判斷參與角色(行為人) {
    if (行為人與他人「共同實行」犯罪) {
        return "共同正犯 §28（co-founder）";
    } else if (行為人「唆使」原本沒有犯意的人去犯罪) {
        return "教唆犯 §29（投資人叫你去創業做壞事）";
    } else if (行為人「幫助」已有犯意的人犯罪) {
        return "幫助犯 §30（提供工具的 vendor）";
    }
}
```

| 判斷要素 | 共同正犯 S28 | 教唆犯 S29 | 幫助犯 S30 |
|----------|-------------|-----------|-----------|
| 角色定位 | 核心成員、共同決策 | 創造犯意的人 | 提供協助的邊緣角色 |
| 行為 | 共同實行犯罪行為 | 使人「萌生犯意」 | 幫助他人實行犯罪 |
| 犯意來源 | 自己有犯意 | 犯意來自教唆犯的唆使 | 正犯本來就有犯意 |
| 處罰 | 依正犯處罰 | 依其所教唆之罪處罰 | 得減輕其刑 |
| 比喻 | Co-founder | 天使投資人（出主意的人） | Vendor / Contractor |

### 混淆原因

三者都是「參與了犯罪但不是自己一個人做完全部」，差別在於**參與的方式和深度**。特別是「共同正犯」和「幫助犯」的界線容易模糊。

### 深度解析

**共同正犯的核心：「犯意聯絡 + 行為分擔」**

```
共同正犯 = 犯意聯絡（共同的犯罪計畫）+ 行為分擔（各自分工實行）

重要原則：「一部行為、全部責任」
→ 甲負責把風、乙負責行竊 → 兩人都成立竊盜罪的共同正犯
→ 甲不能說「我只是站在外面，沒有偷東西」來脫罪
```

**教唆犯 vs 幫助犯的關鍵區分：「犯意的來源」**

```
if (正犯原本沒有犯意，是因為行為人的唆使才產生犯意) {
    行為人 = 教唆犯;  // 你「創造」了這個犯罪
} else if (正犯本來就有犯意，行為人只是提供協助) {
    行為人 = 幫助犯;  // 你只是「幫忙」已經存在的犯罪
}
```

### 案例對比

| 情境 | 角色 | 理由 |
|------|------|------|
| 甲乙一起規劃搶銀行，甲持槍、乙開車 | 共同正犯 | 犯意聯絡+行為分擔 |
| 甲對乙說：「你去把那個人打一頓，我給你 5 萬」，乙去打了 | 甲=教唆犯，乙=正犯 | 甲使乙萌生犯意 |
| 甲知道乙要去偷東西，借了甲的車給乙當逃跑工具 | 甲=幫助犯 | 乙本來就有犯意，甲只是提供工具 |
| 甲負責調查目標住家作息、乙負責進去偷 | 共同正犯 | 犯意聯絡+行為分擔（甲的調查是犯罪計畫的核心部分） |

### 邊界案例

**共謀共同正犯：甲是黑幫老大，策劃搶劫的所有細節，但自己沒到現場。**

```
只是「叫人去搶」→ 教唆犯
「主導整個犯罪計畫」、掌控全局 → 共謀共同正犯（以正犯論處）
→ 實務通說承認：雖未親自實行，但居犯罪核心地位 = 共同正犯
```

### 考試陷阱

1. **教唆犯的「雙重故意」**：教唆犯必須 (1) 故意唆使他人 + (2) 故意使他人實行特定犯罪。如果你隨口說「把他殺了」但不是認真的 → 可能不成立教唆
2. **幫助犯必須有「幫助故意」**：不知道對方要犯罪而提供工具 → 不成立幫助犯（例如五金行正常賣刀給客人）
3. **教唆犯的處罰依照「正犯所犯之罪」**：教唆殺人，正犯只構成傷害 → 教唆犯的處罰跟著調整

## 混淆組 14：想像競合 vs 實質競合 vs 法條競合

### 一眼判斷法

```
function 判斷競合類型(行為數, 法條關係) {
    if (行為數 === 1) {
        if (法條之間有包含/特別關係) {
            return "法條競合（只適用一個法條）";
        } else {
            return "想像競合 §55（從一重處斷）";
        }
    } else if (行為數 > 1) {
        return "實質競合 §50（合併處罰）";
    }
}
```

| 類型 | 行為數 | 法條數 | 處理方式 | 比喻 |
|------|--------|--------|---------|------|
| 想像競合 S55 | 一個行為 | 觸犯多個法條 | 從一重處斷 | 一個 commit 觸發了多個 CI check failure |
| 實質競合 S50 | 多個行為 | 觸犯多個法條 | 合併處罰（有上限） | 多個 commit 各自觸發不同的 failure |
| 法條競合 | 一個行為 | 看似觸犯多個法條 | 只適用一個（排斥其他） | 多個 rule 匹配但有 priority，只執行最高優先 |

### 深度解析

```
想像競合 S55：一行為觸犯多罪 → 從一重處斷
  例：一槍射穿乙又打中丙 → 兩個殺人罪，從一重

實質競合 S50：多行為各自成罪 → 合併處罰
  例：今天偷皮夾、明天打人 → 竊盜+傷害，合併

法條競合（不是真的競合，是排斥）：法條間有邏輯包含關係 → 只適用一個
  特別關係：§272 排斥 §271（特別法優先）
  補充關係：實害犯排斥危險犯
  吸收關係：既遂吸收未遂
```

### 案例對比

| 情境 | 競合類型 | 處理 |
|------|---------|------|
| 一顆炸彈炸死3人 | 想像競合 | 一個行為觸犯3個殺人罪，從一重 |
| 週一偷東西、週三打人 | 實質競合 | 兩個獨立行為，合併處罰 |
| 殺害自己的父親 | 法條競合 | §272 排斥 §271（特別法優先） |
| 開車撞死人又撞壞路邊車 | 想像競合 | 一個行為同時觸犯過失致死+毀損 |

### 考試陷阱

1. **「一個行為」的判斷是關鍵**：是想像競合還是實質競合，取決於行為人做了幾個行為。同一個動作 = 想像競合；分開的動作 = 實質競合
2. **法條競合不是真正的「競合」**：它的效果是排斥，而非從一重處斷。法條競合只適用一個法條，其他法條完全不出現在判決中
3. **想像競合「從一重處斷」，但輕罪的效果不能被忽略**：例如重罪只有罰金、輕罪有有期徒刑 → 不能只判罰金

## 混淆組 15：作為犯 vs 不作為犯

### 一眼判斷法

```
function 判斷行為類型(行為) {
    if (行為 === "積極做了某件事") {
        return "作為犯";
    } else if (行為 === "該做而沒做") {
        if (法律有「明文規定」不作為即構成犯罪) {
            return "純正不作為犯（法律直接規定）";
        } else {
            return "不純正不作為犯 §15（以不作為實現作為犯的構成要件）";
        }
    }
}
```

| 判斷要素 | 作為犯 | 不作為犯 |
|----------|--------|---------|
| 行為態樣 | 積極行動 | 消極不動 |
| 因果關係 | 行為「造成」結果 | 不作為「未阻止」結果 |
| 例子 | 推人下水 | 看到人溺水不去救 |
| 特殊要件 | 無 | 保證人地位（S15） |

### 深度解析

**不純正不作為犯的核心：保證人地位（S15）**

S15：有防止義務、能防止而不防止者，與積極行為發生結果者同。

```
保證人地位的來源（= 你被 assign 了 on-call duty）：
├── 法令：父母對未成年子女
├── 契約：保姆對受託小孩、救生員對泳客
├── 危險前行為：你把人推倒，就有義務救他
├── 自願承擔：你主動接下救護工作
└── 密切生活關係：同居伴侶之間
```

沒有 on-call duty 的路人甲 → 系統掛了跟你無關（不成立不作為犯）。

### 案例對比

| 情境 | 類型 | 理由 |
|------|------|------|
| 甲把乙推入水中溺死 | 作為犯（殺人） | 積極行為造成死亡 |
| 甲看到陌生人乙溺水，不予施救，乙死亡 | 原則上不成立殺人罪 | 甲對陌生人乙沒有保證人地位 |
| 母親看到自己 3 歲小孩溺水，不予施救 | 不作為犯（殺人/過失致死） | 母親對小孩有法定保護義務（保證人地位） |
| 救生員看到泳客溺水，不予施救 | 不作為犯 | 救生員因契約有保護義務 |
| 甲不小心把乙撞倒在路上，開車離開，乙被後車輾過死亡 | 不作為犯 | 甲的前行為（撞倒乙）產生保證人地位 |

### 考試陷阱

1. **不是所有「不救人」都構成犯罪**：只有具備「保證人地位」的人才有救助義務。一般路人看到陌生人遇難不施救 → 道德上可議，但刑法上不構成犯罪（除非有特別法規定）
2. **不作為犯也需要「能力」**：你有義務救人，但你自己不會游泳 → 「不能防止」→ 不成立不作為犯
3. **不作為的因果關係判斷**：用「假設的因果關係」→ 「如果行為人做了他該做的事，結果有沒有可能避免？」如果即使做了也無法避免 → 不成立

## 混淆組 16：刑法詐欺 vs 民法詐欺

### 一眼判斷法

```
function 判斷適用法域(行為) {
    if (行為構成「犯罪」且需要「國家刑罰」) {
        return "刑法詐欺 §339（國家追訴、可能坐牢）";
    }
    if (行為導致「意思表示瑕疵」且需要「撤銷契約」) {
        return "民法詐欺 §92（被害人撤銷、回復原狀）";
    }
    // 兩者可以同時成立！
}
```

| 判斷要素 | 刑法詐欺 S339 | 民法詐欺 S92 |
|----------|--------------|-------------|
| 目的 | 處罰行為人 | 保護被害人的意思自由 |
| 法律效果 | 刑罰（自由刑、罰金） | 被詐欺者得撤銷其意思表示 |
| 構成要件 | 施用詐術→陷於錯誤→處分財產→取得利益 | 因被詐欺而為意思表示 |
| 追訴方式 | 國家（檢察官）追訴 | 被害人自己主張 |
| 嚴格程度 | 較嚴格（罪刑法定） | 較寬鬆 |
| 可以並行？ | 可以 | 可以 |

### 深度解析

**核心差異：刑法看「行為人該不該被處罰」，民法看「被害人的契約效力」。** 同一個詐騙行為可以同時觸發刑事追訴和民事撤銷。

```
刑法：施用詐術 → 陷於錯誤 → 處分財產 → 取得利益（每一步嚴格證明）
民法：因被詐欺而為意思表示 → 得撤銷（門檻較低）
```

### 案例對比與考試陷阱

| 情境 | 刑法效果 | 民法效果 |
|------|---------|---------|
| 中古車商隱瞞事故車賣你 | 可能成立詐欺罪 | 可撤銷買賣契約 |
| 網拍誇大商品功效 | 不一定成立（誇大≠詐術） | 可能可主張撤銷 |

- **刑法詐欺不成立 ≠ 民法詐欺不成立**（刑法標準較嚴格）
- **民法 S92 但書**：第三人詐欺 → 限相對人明知才可撤銷
- **跨科題型**：同一案例問刑法和民法效果 → 分別論述

## 混淆組 17：刑法竊盜 vs 民法侵權

### 一眼判斷法

| 判斷要素 | 刑法竊盜 S320 | 民法侵權行為 S184 |
|----------|--------------|-------------------|
| 性質 | 犯罪 | 民事責任 |
| 制裁 | 刑罰（坐牢、罰金） | 損害賠償（賠錢） |
| 追訴 | 國家（檢察官） | 被害人自己（民事訴訟） |
| 主觀要件 | 故意（竊盜必須是故意） | 故意或過失皆可 |
| 客體 | 他人之動產 | 權利或利益（範圍更廣） |

### 深度解析與考試陷阱

同一個偷東西行為同時觸發兩套系統：刑事（竊盜罪，最重 5 年）+ 民事（侵權行為，損害賠償+物之返還）。**兩者獨立運作**：刑事判無罪 ≠ 民事不用賠。

- **刑事附帶民事訴訟**：被害人可在刑事審判中附帶提起民事賠償
- **竊盜罪的「動產」限制**：刑法只保護動產，民法侵權對不動產也保護
- **親屬間竊盜（S324）**：配偶、直系血親間竊盜 → 免除其刑，但民事侵權不受此限

## 混淆組 18：正當防衛(刑法S23) vs 自助行為(民法S151)

### 一眼判斷法

```
function 判斷法律依據(情境) {
    if (侵害正在發生 && 目的是「防止侵害繼續」) {
        return "正當防衛（刑法 §23）";
    } else if (侵害已經結束 && 目的是「保全自己的請求權」) {
        return "自助行為（民法 §151）";
    }
}
```

| 判斷要素 | 正當防衛（刑法 S23） | 自助行為（民法 S151） |
|----------|--------------------|--------------------|
| 時間 | 侵害「正在發生」 | 侵害已結束，但需要「事後保全」 |
| 目的 | 防止/排除不法侵害 | 保全請求權（來不及請求公力救濟） |
| 效果 | 阻卻違法（不成立犯罪） | 阻卻違法（不構成侵權） |
| 法域 | 刑法 | 民法 |
| 事後義務 | 無 | 須即時向法院聲請處理（S151 II） |
| 比喻 | 即時防火牆 | 事後緊急 hotfix |

### 深度解析

正當防衛 = A 正在打你，你打回去（侵害進行中 → 防衛）。
自助行為 = A 偷了你的東西跑掉，你追上去搶回來（侵害已結束 → 事後保全）。
混淆原因：「侵害正在發生」和「侵害剛結束」的分界線可能只差幾秒鐘。

### 案例對比

| 情境 | 性質 | 理由 |
|------|------|------|
| 小偷正在翻你皮夾，你一拳打過去 | 正當防衛 | 侵害正在發生 |
| 小偷拿了你皮夾轉身就跑，你追上去搶回來 | 自助行為（或正當防衛延續，有爭議） | 侵害是否已結束？ |
| 小偷昨天偷了你的東西，今天你在路上看到他，搶回來 | 自助行為（如果來不及報警） | 侵害已結束，事後保全 |
| 小偷昨天偷了你的東西，今天你去他家把他打一頓 | 不成立任何阻卻違法 | 這是「報復」，不是防衛也不是自助 |

### 考試陷阱

1. **自助行為有「即時聲請法院」的義務**：如果你搶回東西後不去報警/聲請法院 → 自助行為的合法性可能喪失
2. **正當防衛的時間限制很嚴格**：「現在的侵害」= 侵害正在發生或迫在眉睫。侵害結束就不能再主張正當防衛
3. **跨科考法**：題目可能問「甲的行為在刑法上是否構成犯罪？在民法上是否構成侵權？」→ 分別用刑法正當防衛和民法自助行為來分析
