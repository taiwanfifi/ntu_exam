# 模糊地帶辨析：訴訟法與商事法篇 — 20 組易混淆概念的精確判斷

> **適用考試**：律師／司法官一試＋二試
> **目標讀者**：有理工或程式背景的非法律轉職考生
> **核心理念**：訴訟法的難點在於「程序選擇」，商法的難點在於「制度間的交叉適用」。
> 用工程師的系統思維，把判斷標準化為 if-else 規則，讓每一組混淆概念都能精確區分。

---

## 閱讀指南

每一組混淆概念包含六個模組：

| 模組 | 功能 | 工程類比 |
|------|------|----------|
| 一眼判斷法 | 快速分類的 if-else 表格 | switch-case |
| 混淆原因 | 為什麼搞混 | root cause analysis |
| 深度解析 | 制度底層邏輯 | 讀 source code |
| 案例對比 | 同一情境不同適用 | unit test |
| 邊界案例 | 最容易判斷錯的情境 | edge case |
| 考試陷阱 | 出題者最愛設的坑 | known bugs |

# 壹、民事訴訟法混淆組

---

## 第 1 組：訴之變更 vs 訴之追加

### 一眼判斷法

```
if (原本的訴訟標的或聲明被「替換」了) {
    return "訴之變更";  // 舊的不要了，換新的
} else if (原本的訴訟標的或聲明「還在」，多加新的) {
    return "訴之追加";  // 舊的保留，再加新的
}
```

| 判斷基準 | 訴之變更 | 訴之追加 |
|----------|----------|----------|
| 原訴訟標的/聲明 | 被取代、不再主張 | 仍保留 |
| 新訴訟標的/聲明 | 替換原來的 | 額外增加 |
| 訴訟標的數量 | 不變（A→B） | 增加（A→A+B） |
| 工程類比 | git revert + new commit | git add new file |

### 混淆原因

兩者都是「訴訟中改變起訴內容」，條文都在 §255，適用條件幾乎一樣。但變更意味原請求「消滅」，追加則原請求「還在」。很多考生覺得「反正都是改訴，有差嗎？」——有差，因為變更後法院不再審理原請求，追加後法院要同時審理兩個請求。

### 深度解析

§255 I 規定訴之變更或追加原則需被告同意，但七款例外不需同意：

| 款次 | 例外情形 | 白話翻譯 |
|------|----------|----------|
| ① | 請求之基礎事實同一 | 同一件事換法律主張 |
| ② | 因情事變更以他項聲明代之 | 局勢變了改要求 |
| ③ | 擴張或減縮應受判決事項之聲明 | 多要一點或少要一點 |
| ④ | 基礎事實同一，追加原非當事人 | 同一件事拉新人進來 |
| ⑤ | 訴訟標的對數人必須合一確定 | 法律上必須一起判 |
| ⑥ | 於某法律關係之成立有爭執 | 先確認前提問題 |
| ⑦ | 不甚礙被告之防禦及訴訟之終結 | 改了也不影響對方 |

### 案例對比

**情境：甲被乙車撞傷，起訴請求醫藥費 50 萬元。**

| 後續動作 | 分類 | 理由 |
|----------|------|------|
| 金額從50萬改80萬 | 擴張聲明（§255 I③） | 標的不變，金額增加 |
| 不要醫藥費，改請求精神慰撫金 | 訴之變更 | 原請求被替換 |
| 醫藥費保留，再加精神慰撫金 | 訴之追加 | 原請求保留，多加新的 |
| 除了告乙，再追加告乙的僱主丙 | 訴之追加（主體追加） | 多拉一個被告 |

### 邊界案例

甲原主張侵權（§184），後改主張不當得利（§179）——如果只主張不當得利、不再主張侵權 → 變更；如果同時主張兩者（競合）→ 追加。關鍵：原請求權基礎還在不在。

### 考試陷阱

> 「擴張聲明」不是訴之變更，是 §255 I③ 的獨立類型。
> 「50萬改80萬」只是擴張聲明，無需被告同意。
> 考生最容易犯的錯：看到金額改變就以為是「變更」。

## 第 2 組：既判力 vs 爭點效

### 一眼判斷法

```
if (問的是「訴訟標的」在後訴中的拘束力) {
    return "既判力（§400）";    // 法律明文，強拘束
} else if (問的是「重要爭點」在後訴中的拘束力) {
    return "爭點效（判例法理）"; // 法院創設，弱拘束
}
```

| 判斷基準 | 既判力 | 爭點效 |
|----------|--------|--------|
| 法源 | §400 明文規定 | 判例法理（最高法院96台上2531） |
| 拘束對象 | 訴訟標的之法律關係 | 前訴已充分攻防之重要爭點 |
| 發生前提 | 確定判決當然發生 | 須符合四要件 |
| 效力強度 | 絕對拘束（不可覆寫） | 相對拘束（有條件可推翻） |
| 工程類比 | final 變數 | cached value（可 invalidate） |

### 混淆原因

兩者都是「前訴拘束後訴」，而且實務上常同時討論。但既判力只及於「主文判斷的訴訟標的」，爭點效及於「理由中的重要爭點判斷」。範圍完全不同。

### 深度解析

**既判力範圍（§400）：**
- 客觀範圍：僅及主文所判斷的訴訟標的，判決理由中的判斷不生既判力
- 主觀範圍：原則僅及當事人（§401 例外擴張至繼受人等）
- 時間範圍：以事實審言詞辯論終結時為基準

**爭點效四要件（全部滿足才成立）：**

```
1. 前訴中當事人就該爭點已為充分攻防
2. 法院就該爭點已為實質審理並判斷
3. 該爭點的判斷是判決的重要基礎
4. 前後訴的當事人相同
→ 任一不滿足，爭點效不成立，後訴法院可做不同認定
```

### 案例對比

甲訴乙返還借款100萬，法院認定「有消費借貸關係」，甲勝訴確定：

| 後訴情境 | 既判力 | 爭點效 |
|----------|--------|--------|
| 乙另訴確認借貸不存在 | 被既判力擋（同一標的） | 不用討論 |
| 甲再訴另一筆50萬借款 | 無（不同標的） | 「有借貸關係」可能有爭點效 |
| 甲訴乙侵權，乙抗辯甲才欠錢 | 無（不同標的） | 前訴認定可能拘束後訴 |

### 邊界案例

判決理由認定「甲有過失」→ 既判力不適用（理由不是標的）→ 爭點效要看四要件是否滿足 → 如後訴出現新事證（如行車紀錄器），爭點效可能被推翻。

### 考試陷阱

> 既判力只及於「主文」，不及於「理由」。這是最高頻考點。
> 判決理由中的判斷頂多成立爭點效，且需四要件全部滿足。
> 很多考生以為「法院判過的就不能翻」——理由中的判斷，可能可以翻。

## 第 3 組：假扣押 vs 假處分 vs 定暫時狀態處分

### 一眼判斷法

```
if (怕對方脫產 && 金錢債權)        → 假扣押（§522）→ 凍結財產
if (怕現狀被改變 && 非金錢給付)    → 假處分（§532）→ 維持現狀
if (法律關係有爭議 && 不決定會出事) → 定暫時狀態（§538）→ 暫時定一個狀態
```

| 判斷基準 | 假扣押 | 假處分 | 定暫時狀態處分 |
|----------|--------|--------|----------------|
| 保全目的 | 防止脫產 | 防止現狀變更 | 暫時確定法律關係 |
| 請求類型 | 金錢債權 | 非金錢之特定物 | 有爭執之法律關係 |
| 條文 | §522-528 | §532-535 | §538 |
| 工程類比 | lock 資源 | read-only 模式 | 設定臨時 default 值 |

### 混淆原因

三者都是「暫時保護措施」，目的都是「怕等到判決確定就來不及了」。考生看到「保全」就混在一起，分不清各自適用場景。

### 深度解析

- **假扣押**：你欠我錢 → 我怕你轉移財產 → 先凍住你的財產
- **假處分**：你應該給我某特定物 → 我怕你毀壞或變更 → 先維持現狀
- **定暫時狀態**：我和你對法律關係有爭議 → 不先決定會造成重大損害 → 法院暫時拍板

### 案例對比

| 情境 | 正確選擇 | 理由 |
|------|----------|------|
| 甲欠乙200萬不還，甲正在脫產 | 假扣押 | 金錢債權 + 防脫產 |
| 甲賣房給乙，交屋前甲又要賣給丙 | 假處分 | 特定物交付 + 防現狀變更 |
| 鄰居持續在違建上加蓋 | 假處分 | 非金錢 + 維持現狀 |
| 公司董事長改選有爭議，兩派對立 | 定暫時狀態 | 法律關係爭議 + 公司營運受損 |
| 員工被解僱，認為無效請求暫時復職 | 定暫時狀態 | 僱傭關係有爭執 |

### 邊界案例

甲請求乙交付A車，乙打算把車賣掉 → 不是假扣押（甲要的不是錢）→ 是假處分（特定物）。判斷關鍵：本案請求是「金錢」還是「特定物」。

### 考試陷阱

> 假處分 = 對「物」的保全（維持特定物現狀）。
> 定暫時狀態 = 對「法律關係」的暫時確定（誰是董事長、僱傭是否存在）。
> 快速口訣：看到「物」選假處分，看到「身份/地位」選定暫時狀態。

## 第 4 組：普通共同訴訟 vs 必要共同訴訟（固有 vs 類似）

### 一眼判斷法

```
if (多數當事人的訴訟標的各自獨立) {
    return "普通共同訴訟（§53）";  // 各打各的，只是剛好一起告
} else if (法律上必須全部一起) {
    return "固有必要共同訴訟";     // 少一個人就駁回
} else if (各自可告但結果要統一) {
    return "類似必要共同訴訟";     // 一起告的話結果要一樣
}
```

| 判斷基準 | 普通 | 固有必要 | 類似必要 |
|----------|------|----------|----------|
| 當事人適格 | 各自獨立 | 必須全體一起 | 各自可單獨 |
| 判決結果 | 可以不同 | 必須合一確定 | 必須合一確定 |
| 少一個人 | 沒關係 | 不適格→駁回 | 沒關係，但一起告就統一 |
| 典型案例 | 兩債權人各告同一人 | 全體繼承人分割遺產 | 數股東撤銷股東會決議 |
| 工程類比 | 獨立 thread | 強制 join 的 thread group | merge 結果的 thread pool |

### 混淆原因

「必要」指什麼必要？固有必要 = 人必須到齊；類似必要 = 結果必須統一。類似必要中每人可單獨起訴，但合併起訴就適用合一確定（§56）。

### 深度解析

- **普通共同訴訟**：只是訴訟經濟，合在一起審。A勝B敗完全可能。
- **固有必要**：法律規定必須全體一起。例如共有物分割（§823）、遺產分割（§1164）。少一個 → 法院命補正 → 不補正就駁回。
- **類似必要**：適用 §56 合一確定——有利行為及於全體，不利行為不及於全體。

### 案例對比

| 情境 | 類型 | 理由 |
|------|------|------|
| 甲乙共有土地，甲訴請分割 | 固有必要 | 分割共有物必須全體共有人 |
| 甲乙各借100萬給丙，一起告丙 | 普通共同 | 兩個獨立借貸債權 |
| 甲乙都是股東，分別提撤銷決議 | 類似必要 | 決議有效或無效只能一個結果 |
| 甲被撞，告司機乙和僱主丙 | 普通共同 | 對乙丙的請求權各自獨立 |

### 邊界案例

連帶債務人是固有必要嗎？不是！連帶債務人責任各自獨立，債權人可選擇告一個或全部。一起告 → 普通共同訴訟。這是最常見的陷阱。

### 考試陷阱

> 看到「共有物分割」→ 固有必要。但「連帶債務」→ 普通共同。不要搞混。
> 類似必要的關鍵：§56 有利行為及於全體，不利行為不及於全體。

## 第 5 組：訴訟參加 vs 訴訟告知

### 一眼判斷法

```
if (第三人「主動」加入訴訟) {
    return "訴訟參加（§58 輔助參加 / §54 獨立參加）";
} else if (當事人「通知」第三人有訴訟在進行) {
    return "訴訟告知（§65）";
    // 第三人可以選擇加入或不加入
}
```

| 判斷基準 | 訴訟參加 | 訴訟告知 |
|----------|----------|----------|
| 發動者 | 第三人主動申請 | 當事人通知第三人 |
| 第三人選擇 | 已決定要參加 | 可選擇是否參加 |
| 效果 | 成為訴訟參加人 | 產生參加效（§67） |
| 工程類比 | subscribe event | send notification |

### 混淆原因

兩者常連動：當事人先做訴訟告知 → 第三人收到後決定參加 → 變成訴訟參加。但它們是兩個不同制度。

### 深度解析

**輔助參加（§58）**：第三人就兩造訴訟有法律上利害關係 → 主動加入幫一方打官司。例如保證人輔助主債務人（主債務人輸了，保證人也要賠）。

**訴訟告知（§65）**：當事人通知第三人有訴訟 → 目的是讓第三人知道，將來可對其主張參加效。不論第三人是否參加，都受參加效拘束（§67）。

**參加效 ≠ 既判力**：參加效較弱，只是受告知人不能在後訴中主張「前訴判斷不當」。

### 案例對比

| 情境 | 制度 | 說明 |
|------|------|------|
| 甲告乙，丙認為乙輸了自己也受影響，主動聲請加入幫乙 | 輔助參加 | 第三人主動 |
| 甲告乙，乙通知丙有訴訟進行 | 訴訟告知 | 當事人通知 |
| 買受人甲被丙追奪標的物，甲告知出賣人乙 | 訴訟告知 | 將來可向乙主張瑕疵擔保 |

### 邊界案例

訴訟告知後受告知人沒參加 → 仍產生參加效（§67）！後訴中不能主張前訴判斷不當。這是最容易錯的地方。

### 考試陷阱

> 「丙未參加訴訟，判決效力是否及於丙？」→ 既判力不及於丙，但參加效及於丙。
> 訴訟告知 ≠ 強制參加。但不參加的後果是受參加效拘束。

## 第 6 組：給付之訴 vs 確認之訴 vs 形成之訴

### 一眼判斷法

```
if (要對方「做某件事」或「給某樣東西」) → 給付之訴
if (只要法院「確認法律關係存在或不存在」) → 確認之訴
if (要法院「創設、變更、消滅法律關係」) → 形成之訴
```

| 判斷基準 | 給付之訴 | 確認之訴 | 形成之訴 |
|----------|----------|----------|----------|
| 目的 | 讓對方履行義務 | 釐清法律關係 | 改變法律關係 |
| 判決主文 | 被告應給付... | 確認...存在/不存在 | 撤銷.../解除... |
| 可否強制執行 | 可以（最重要差別） | 不行 | 不行（判決確定即生效） |
| 需確認利益？ | 不需要 | 需要（§247） | 不需要 |
| 需形成權？ | 不需要 | 不需要 | 需要法定形成權 |
| 工程類比 | POST/PUT | GET | DELETE/PATCH |

### 混淆原因

很多訴訟同時涉及多種類型。例如「確認買賣關係存在＋請求交付標的物」同時包含確認之訴和給付之訴。

### 深度解析

- **給付之訴**分為現在給付（已屆清償期）vs 將來給付（§246：尚未屆期但有預為請求必要）
- **確認之訴**需「確認利益」= 法律地位有不安，可透過確認判決除去。消極確認（確認不存在）實務很常見，如確認本票債權不存在。
- **形成之訴**須有法定形成權，例如：撤銷詐害行為（§244）、離婚（§1052）、撤銷股東會決議

### 邊界案例

「確認租約不存在」→ 確認之訴。不能說「判決租約無效」→ 無效是自始當然無效，不需法院判決來形成。確認之訴有補充性（§247但書）：能提給付之訴就不該提確認之訴。

### 考試陷阱

> 形成之訴必須有法定形成權，沒有明文就不能提。
> 「確認 vs 形成」的區分：無效 → 確認之訴（法律關係本來就不存在）；撤銷 → 形成之訴（法律關係存在但要消滅它）。

## 第 7 組：上訴 vs 抗告 vs 再審 vs 第三人撤銷訴訟

### 一眼判斷法

```
if (不服「判決」&& 未確定)              → 上訴（20日）
if (不服「裁定」)                        → 抗告（10日）
if (判決已確定 && 有法定再審事由 && 當事人) → 再審（知悉30日）
if (判決已確定 && 損害權利 && 非當事人)    → 第三人撤銷訴訟（知悉30日）
```

| 判斷基準 | 上訴 | 抗告 | 再審 | 第三人撤銷 |
|----------|------|------|------|------------|
| 針對 | 未確定判決 | 裁定 | 已確定判決 | 已確定判決 |
| 提起人 | 當事人 | 當事人 | 當事人 | 非當事人第三人 |
| 性質 | 通常救濟 | 通常救濟 | 非常救濟 | 非常救濟 |
| 期間 | 20日 | 10日 | 知悉後30日 | 知悉後30日 |

### 混淆原因

四種都是「不服法院決定的救濟」。考生最易混的是再審和第三人撤銷——都針對確定判決，但提起人不同（當事人 vs 第三人）。

### 深度解析

**上訴三級制**：二審覆審制（事實＋法律）；三審法律審（§468 以違背法令為由，且利益逾150萬 §466）。

**再審法定事由（§496 I）**：最重要的是①適用法規顯有錯誤、⑬發現新證據。

**第三人撤銷訴訟（§507-1）**：有法律上利害關係之第三人，非因可歸責於己之事由未參加訴訟，可撤銷原確定判決中對其不利部分。

### 案例對比

| 情境 | 正確救濟 | 理由 |
|------|----------|------|
| 一審判敗訴，20天內 | 上訴 | 判決未確定＋當事人 |
| 法院駁回假扣押聲請 | 抗告 | 裁定不是判決 |
| 確定後發現對方偽造文書 | 再審 | 有法定事由 |
| 甲乙的確定判決把丙的土地判給甲 | 第三人撤銷 | 丙非當事人，權利受損 |

### 考試陷阱

> 上訴20天、抗告10天——期間不同！
> 再審不是「覺得判錯」就能提，必須符合 §496 I 各款法定事由。
> 第三人撤銷只限「有法律上利害關係」的第三人。

# 貳、刑事訴訟法混淆組

## 第 8 組：公訴 vs 自訴

### 一眼判斷法

| 判斷基準 | 公訴 | 自訴 |
|----------|------|------|
| 起訴人 | 檢察官 | 犯罪被害人 |
| 律師代理 | 不需要 | 強制律師代理（§319 II） |
| 偵查程序 | 有（先偵查再起訴） | 無（直接起訴） |
| 撤回 | 不得任意撤回 | 可撤回（§325） |
| 案件範圍 | 所有犯罪 | 限被害人，排除特定案件 |

### 混淆原因

考生常問「都可以追訴犯罪，有何不同？」公訴是常態，自訴是例外補充。自訴有很多限制。

### 深度解析

**為何自訴強制律師代理？**
1. 防止濫訴（避免被害人情緒化提告）
2. 訴訟品質（被害人不懂法律程序）
3. 武器平等（律師對抗辯護律師）

**公訴優先（§323 I）**：檢察官已開始偵查 → 不得再自訴。

### 案例對比

| 情境 | 結果 | 理由 |
|------|------|------|
| 被詐騙100萬，去警局報案 | 公訴 | 由檢察官偵查起訴 |
| 被鄰居打傷，想自己告 | 自訴（須委任律師） | 被害人直接起訴 |
| 被打傷，已報警偵辦中 | 只能等公訴 | 公訴優先原則 |
| 被害人死亡，家屬想告 | 家屬可自訴（§319 I但書） | 仍須律師代理 |

### 邊界案例

被害人死亡 → 配偶、直系血親、三親等旁系血親、二親等姻親可獨立自訴。

### 考試陷阱

> 「甲一邊報警一邊自訴」→ 如果檢察官已開始偵查，自訴不合法（§323 I）。
> 自訴人在法庭上是「原告」，但不能自己進行訴訟——必須透過律師。

## 第 9 組：告訴 vs 告發 vs 自首

### 一眼判斷法

```
被害人去報案   → 告訴（§232）
路人去報案     → 告發（§240）
犯人自己去投案 → 自首（刑法§62，注意規定在刑法不在刑訴）
```

| 判斷基準 | 告訴 | 告發 | 自首 |
|----------|------|------|------|
| 行為人 | 被害人等特定人 | 任何人 | 犯罪人本人 |
| 法律依據 | 刑訴§232-§236 | 刑訴§240-§241 | 刑法§62 |
| 法律效果 | 告訴乃論罪的訴追條件 | 僅供偵查線索 | 得減輕其刑 |
| 可否撤回 | 告訴乃論罪可撤回（§238） | 不可 | 不適用 |
| 期間 | 告訴乃論罪6個月（§237） | 無限制 | 無限制 |

### 混淆原因

三者都是「讓司法機關知道有犯罪」，但法律效果天差地別。告訴是告訴乃論之罪的「訴追條件」——沒有告訴，檢察官不能起訴。

### 深度解析

```
if (該罪是「告訴乃論之罪」) {
    if (有合法告訴) → 檢察官可以起訴;
    else → 不能起訴（欠缺訴追條件）;
} else {
    告訴只是偵查線索，跟告發效果一樣;
}
```

常見告訴乃論之罪：普通傷害（§277 I）、過失傷害（§284）、公然侮辱（§309）、誹謗（§310）、毀損（§354）。

自首要件：犯罪「未發覺」前（含犯人身份未發覺）。效果：「得」減輕其刑（非「應」）。

### 案例對比

| 情境 | 分類 | 理由 |
|------|------|------|
| 甲被乙打傷，去派出所報案 | 告訴 | 被害人申告 |
| 路人丙看到乙打甲，去報案 | 告發 | 第三人申告 |
| 乙打完人後悔，自己去派出所 | 自首 | 犯罪人自行申告（犯罪未發覺前） |
| 乙被監視器拍到，警察上門後承認 | 不是自首 | 犯罪已發覺，僅「自行到案」 |

### 邊界案例

犯罪已被發覺但犯人身份未被發覺 → 犯人去投案仍算自首。「未發覺」包括犯人未被發覺。

### 考試陷阱

> 「告訴」不等於「起訴」！告訴 = 報案，起訴 = 檢察官提起公訴。
> 撤回告訴須在第一審辯論終結前（§238 I），過了就不能撤。
> 自首是「得」減輕，法官有裁量權。

## 第 10 組：逮捕 vs 拘提 vs 羈押

### 一眼判斷法

```
現場直接抓人（不需令狀）    → 逮捕（§87/§88）
拿法院/檢察官令狀去抓人到案 → 拘提（§71-§78）
到案後關在看守所等審判      → 羈押（§101，需法官裁定）
```

| 判斷基準 | 逮捕 | 拘提 | 羈押 |
|----------|------|------|------|
| 時間點 | 即時（當場） | 按令狀執行 | 到案後 |
| 令狀 | 不需要（現行犯） | 需拘票 | 需法官裁定 |
| 決定者 | 任何人(現行犯)/警察 | 法官/檢察官 | 只有法官 |
| 持續時間 | 短暫（解送） | 短暫（到案） | 長期（可達數月） |

### 混淆原因

三者都是「限制人身自由」的強制處分，在時間軸上有先後。考生容易搞不清什麼時候用哪個。

### 深度解析

**時間軸**：犯罪發生 → 逮捕(現場) → 24hr內解送 → 檢察官訊問 → 聲請羈押 → 法官裁定。

**羈押三要件（§101 I）：**
1. 犯罪嫌疑重大
2. 有逃亡之虞 / 串供之虞 / 所犯為重罪
3. 非予羈押顯難進行追訴審判（羈押必要性）

**預防性羈押（§101-1）**：特定犯罪＋有事實足認有反覆實施之虞。

### 邊界案例

「準現行犯」（§88 II）：犯罪後即時發覺、持兇器、身有犯罪痕跡、經追呼為犯罪人等 → 以現行犯論，可逮捕。

### 考試陷阱

> 只有「法官」才能裁定羈押，檢察官只能「聲請」。
> 逮捕後24小時內必須移送（§93 I），超過 → 違法。
> 重罪只是羈押原因之一，仍需審查羈押必要性。不能「因為重罪就直接關」。

## 第 11 組：搜索 vs 扣押

### 一眼判斷法

| 判斷基準 | 搜索 | 扣押 |
|----------|------|------|
| 性質 | 發現、尋找的「過程」 | 占有、取走的「結果」 |
| 對象 | 身體、物件、住宅 | 可為證據或得沒收之物 |
| 令狀 | 原則需搜索票（§128） | 附隨搜索或獨立扣押 |
| 先後 | 通常先搜索 | 找到後扣押 |
| 工程類比 | find / grep | cp / mv |

### 混淆原因

搜索和扣押常同時發生（先搜再扣），考生容易當成一件事。但法律上是兩個獨立強制處分。

### 深度解析

**無令狀搜索的例外：**

| 類型 | 條文 | 情境 |
|------|------|------|
| 附帶搜索 | §130 | 逮捕時搜被逮捕人身體和立即可觸及處所 |
| 緊急搜索 | §131 I | 有相當理由認為情況急迫 |
| 同意搜索 | §131-1 | 受搜索人自願同意 |

扣押的獨立性：不一定要有搜索在前。被告主動交出證物 → 可直接扣押。

### 邊界案例

搜索票寫「搜索住所」可否看手機？趨勢認為手機數位資料涉及高度隱私，需搜索票上明確記載。

### 考試陷阱

> 同意搜索必須「出於自願」，威脅下的同意等於沒同意。
> 違法搜索取得的證據 → §158-4 權衡法則判斷，不是絕對排除。
> 搜索票範圍不能太廣泛，必須記載「應搜索之處所、身體、物件」（§128 II）。

## 第 12 組：傳聞證據 vs 非傳聞

### 一眼判斷法

```
庭外陳述 → 用來證明「陳述內容為真」 → 傳聞（§159，原則排除）
庭外陳述 → 用來證明「陳述行為本身」 → 非傳聞（有證據能力）
```

| 證據 | 分類 | 分析 |
|------|------|------|
| A說「我聽B說他殺了人」→ 證明B殺人 | 傳聞 | 轉述內容當證據 |
| A說「B對我說要殺你」→ 證明B恐嚇 | 非傳聞 | B的話本身就是恐嚇行為 |
| 監視器錄影畫面 | 非傳聞 | 機械紀錄，不是人的陳述 |
| LINE恐嚇訊息 | 非傳聞 | 訊息本身就是犯罪行為 |
| LINE對話拿來證明某事實 | 傳聞 | 用陳述內容證明事實為真 |
| 被害人診斷證明書 | 傳聞例外 | §159-4 特信性文書（業務文書） |

### 混淆原因

關鍵在「使用目的」不同。同一句話，用來證明「內容真實」→ 傳聞；用來證明「有這個行為」→ 非傳聞。

### 深度解析

**傳聞例外（§159-1 至 §159-5）：**

| 條文 | 例外 | 說明 |
|------|------|------|
| §159-1 I | 法官面前陳述 | 有保障的環境 |
| §159-1 II | 檢察官面前陳述 | 被告可爭執 |
| §159-2 | 先前不一致陳述 | 翻供 |
| §159-3 | 不能到庭 | 死亡、失蹤、國外 |
| §159-4 | 特信性文書 | 公務文書、業務文書 |
| §159-5 | 當事人同意 | 明示或擬制同意 |

### 考試陷阱

> 傳聞的判斷是「使用目的」不是「陳述形式」。同一句話可能是傳聞也可能不是。
> 機器產出的紀錄（監視器、測速照相、ATM紀錄）都不是傳聞。
> §159-5「同意」可以是明示或擬制（知悉而未聲明異議）。

## 第 13 組：緩起訴 vs 不起訴 vs 緩刑

### 一眼判斷法

| 判斷基準 | 緩起訴 | 不起訴 | 緩刑 |
|----------|--------|--------|------|
| 決定者 | 檢察官 | 檢察官 | 法官 |
| 階段 | 偵查終結 | 偵查終結 | 審判終結 |
| 意思 | 有嫌疑但先觀察 | 確定不起訴 | 有罪但先觀察 |
| 觀察期 | 1-3年 | 不適用 | 2-5年 |
| 可附條件 | 可以（賠償、勞務等） | 不可 | 可以 |
| 違反後果 | 撤銷，繼續起訴 | 不適用 | 撤銷，執行原刑 |
| 工程類比 | feature flag 暫關 | reject PR | merge 但暫不 deploy |

### 混淆原因

「緩起訴」和「緩刑」都是「先觀察」，但一個在偵查階段（檢察官），一個在審判階段（法官）。

### 深度解析

**緩起訴限輕罪**：最輕本刑三年以上之罪「以外」才能緩起訴。
**不起訴法定事由（§252）**：一事不再理、時效完成、大赦、除罪化、告訴乃論未告訴/撤回、被告死亡、犯嫌不足等。

### 邊界案例

緩起訴期滿未被撤銷 → 效果等同不起訴確定。被害人不服緩起訴 → 聲請再議 → 駁回可聲請交付審判。

### 考試陷阱

> 觀察期不同：緩起訴1-3年，緩刑2-5年。
> 重罪不能緩起訴，但重罪仍可能緩刑（§74限制不同）。
> 緩起訴可附「向被害人賠償」等條件，這是與不起訴最大的差異。

## 第 14 組：簡式審判 vs 簡易程序 vs 協商程序

### 一眼判斷法

| 判斷基準 | 簡式審判 | 簡易程序 | 協商程序 |
|----------|----------|----------|----------|
| 啟動時點 | 審判中 | 偵查終結/審判中 | 起訴後審判中 |
| 啟動者 | 法官裁定 | 檢察官聲請/法院職權 | 檢察官與被告協商 |
| 被告態度 | 自白認罪 | 不需自白 | 認罪且同意 |
| 審理方式 | 仍開庭，簡化證據調查 | 不開庭，書面審理 | 依協商結果判決 |
| 適用案件 | 非重罪 | 輕罪（宜判緩刑/拘役/罰金/6月以下） | 非重罪 |
| 上訴 | 可以 | 可以（範圍受限） | 原則不可（§455-10） |
| 工程類比 | 簡化 code review | CI 自動通過 | 談好條件後 merge |

### 混淆原因

三者都是「比通常程序更快的審判」，但啟動條件、審理方式、上訴限制完全不同。

### 深度解析

```
簡式審判 = 通常程序的「簡化版」→ 還是開庭，只是證據調查簡化
簡易程序 = 不開庭的「書面程序」→ 法院根據卷證直接判決
協商程序 = 檢察官和被告的「交易」→ 認罪換較輕量刑，法院依協商結果判
```

三者都排除重罪（死刑/無期/最輕三年以上）。

### 邊界案例

簡易判決不服 → 上訴到同法院合議庭（不是高院），準用二審通常程序。

### 考試陷阱

> 簡式審判「有開庭」，簡易程序「不開庭」——不要搞反！
> 協商程序原則不可上訴（§455-10），除非有特定違法事由。
> 三種程序都不能用於重罪。

# 參、商事法混淆組

---

## 第 15 組：公司法 vs 證交法（上市公司雙重適用）

### 一眼判斷法

```
if (一般公司)   → 只適用公司法
if (上市櫃公司) → 公司法 + 證交法雙重適用
if (兩者衝突)   → 證交法優先（特別法優先原則）
```

| 面向 | 公司法 §23 | 證交法 §171 |
|------|-----------|-------------|
| 義務內容 | 忠實義務＋善良管理人注意 | 不得損害公司利益以圖私利 |
| 責任性質 | 民事賠償 | 刑事（3-10年有期徒刑） |
| 構成要件 | 違反注意義務即可 | 需不法意圖＋損害公司 |
| 適用對象 | 所有公司董事 | 上市櫃公司董監事/經理人 |

### 混淆原因

上市公司同時受兩部法律規範，很多規定看似重複。答題時必須分別分析。

### 案例對比

甲上市公司董事長知悉併購消息，提前買股獲利 5000 萬：

| 法律依據 | 評價 |
|----------|------|
| 公司法 §23 | 違反忠實義務→民事賠償 |
| 證交法 §157-1 | 內線交易→刑事3-10年 |
| 證交法 §157-1 III | 對善意相對人民事賠償 |
| 證交法 §171 | 特別背信罪→加重處罰 |

### 邊界案例

公開發行但未上市的公司 → 部分適用證交法。§157-1 適用對象包括所有「公開發行公司」，不限上市櫃。

### 考試陷阱

> 不要以為證交法只管上市公司——很多規定適用「公開發行公司」。
> 考題問「法律責任」→ 要同時寫公司法和證交法，不能只寫一個。
> 公司法 §23 是民事，證交法 §171 是刑事——同一行為可能同時觸犯。

## 第 16 組：背書轉讓 vs 交付轉讓（票據法）

### 一眼判斷法

```
記名票據   → 必須背書轉讓（§30 I）：背面簽名 + 交付
無記名票據 → 交付即可（§30 I但書）
空白背書後 → 視為無記名，交付即可轉讓
```

| 判斷基準 | 背書轉讓 | 交付轉讓 |
|----------|----------|----------|
| 適用票據 | 記名票據 | 無記名票據 / 空白背書票據 |
| 背書人責任 | 負擔保付款之責（§29） | 無背書人無擔保 |
| 追索權 | 可向所有背書人追索 | 無法向前手追索 |
| 工程類比 | 有 commit log 的移轉 | 直接 copy，無版本紀錄 |

### 深度解析

**空白背書（§32）的三種處理：**
1. 持票人填入被背書人名 → 變回記名票據
2. 持票人空白背書再轉讓 → 維持無記名流通
3. 持票人記名背書再轉讓 → 回到記名

### 邊界案例

「禁止背書轉讓」（§30 III）→ 轉讓仍有效！只是記載人免除對後手的擔保責任。這是最常見的誤解。

### 考試陷阱

> 「禁止背書轉讓」≠「轉讓無效」。只免擔保責任。
> 空白背書人仍負擔保付款責任。
> 支票沒有承兌制度——不要把匯票規定套用到支票。

---

## 第 17 組：要保人 vs 被保險人 vs 受益人（保險法）

### 一眼判斷法

```
付保費的人 → 要保人（§3）：契約當事人、繳費義務人
被保障的人 → 被保險人（§4）：事故發生於其身上
領保險金的 → 受益人（§5）：被指定領取保險金
※ 三者可以是同一人，也可以是三個不同人
```

| 角色 | 核心功能 | 義務 | 工程類比 |
|------|----------|------|----------|
| 要保人 | 簽約付錢 | 繳保費、據實告知 | 付費訂閱者 |
| 被保險人 | 被保障 | 配合體檢等 | 被監控的系統 |
| 受益人 | 領錢 | 無 | 收到通知的人 |

### 案例對比

| 情境 | 要保人 | 被保險人 | 受益人 |
|------|--------|----------|--------|
| 甲替自己買壽險，受益人指定太太乙 | 甲 | 甲 | 乙 |
| 爸甲幫兒子乙買保險，受益人指定媽丙 | 甲 | 乙 | 丙 |
| 公司甲幫員工乙買團體險 | 甲 | 乙 | 依約定 |
| 甲替自己買車險 | 甲 | 甲 | 甲 |

### 深度解析

**保險利益**：人身保險中要保人對被保險人需有保險利益（§16）——本人、家屬、債務人、管理財產之人等。無保險利益 → 契約無效。

**道德危險防範**：以他人為被保險人的死亡保險需被保險人「書面同意」（§105）。

**未指定受益人**：保險金作為被保險人之遺產（§113）。

**受益人指定權**：要保人有權指定和變更受益人（§111），需通知保險人。

### 邊界案例

要保人欠繳保費 → §116 保險人應催告「要保人」→ 30日內仍未繳 → 契約停效（非直接終止）。催告對象是要保人，不是被保險人。

### 考試陷阱

> 「要保人」才是契約當事人和繳費義務人。
> §105 書面同意是道德危險防範的核心規定。
> 保險利益不存在 → 契約自始無效（§17）。

## 第 18 組：發票人 vs 背書人 vs 承兌人 vs 保證人（票據法）

### 一眼判斷法

| 角色 | 功能 | 出現票據 | 責任性質 |
|------|------|----------|----------|
| 發票人 | 創設票據 | 所有 | 本票=主債務人；匯票/支票=擔保人 |
| 背書人 | 轉讓票據 | 所有 | 第二次債務人（擔保付款） |
| 承兌人 | 承諾付款 | 僅匯票 | 第一次（主）債務人 |
| 保證人 | 擔保付款 | 所有 | 與被保證人負同一責任（§61） |

### 深度解析

**依票據種類的主債務人：**

| 票據 | 主債務人 | 擔保債務人 |
|------|----------|------------|
| 匯票 | 承兌人 | 發票人、背書人 |
| 本票 | 發票人 | 背書人 |
| 支票 | 無（銀行非票據債務人） | 發票人、背書人 |

**追索權**（§96、§97）：持票人可向任何票據債務人追索，不限順序，可跳過。所有背書人連帶負責。

### 案例對比

甲簽發匯票 → 乙背書 → 丙背書 → 丁(持票人)，戊為付款人且已承兌，己為甲的保證人：
- 到期丁提示付款，戊拒付 → 丁可向甲、乙、丙、戊、己任何人追索
- 承兌人戊是主債務人，第一順位承擔
- 保證人己負與甲同一責任

### 考試陷阱

> 支票沒有承兌制度！「支票的承兌人」= 錯誤選項。
> 本票發票人 = 主債務人（跟匯票完全不同！匯票發票人只是擔保人）。
> 保證人是「同一責任」（§61）不是「補充責任」——債權人可直接向保證人追索。

## 第 19 組：股東會普通決議 vs 特別決議 vs 假決議（公司法）

### 一眼判斷法

| 類型 | 出席門檻 | 表決門檻 | 效力 |
|------|----------|----------|------|
| 普通決議（§174） | ≥ 已發行股份 1/2 | 出席 ≥ 1/2 同意 | 直接生效 |
| 特別決議（§185等） | ≥ 已發行股份 2/3 | 出席 ≥ 1/2 同意 | 直接生效 |
| 假決議（§175） | ≥ 1/3 但未達法定 | 出席 ≥ 2/3 同意 | 暫時，需再確認 |

### 混淆原因

「假決議」不是「假的」決議，是「暫時的、附條件的」決議——出席人數不夠時的替代機制。

### 深度解析

**需特別決議的事項（法定列舉）：**

| 條文 | 事項 |
|------|------|
| §185 | 營業或財產重大變更、讓與全部或主要營業/財產 |
| §199 | 董事之解任 |
| §277 | 變更章程 |
| §316 | 合併、分割 |
| §316-1 | 股份轉換 |

**假決議程序（§175）：**
1. 第一次股東會：出席不足1/2但達1/3 → 出席2/3同意 → 做成假決議
2. 通知各股東
3. 一個月內再開第二次股東會 → 出席不論多少，1/2同意 → 正式決議

### 邊界案例

特別決議事項可否用假決議？通說認為不行。特別決議的高門檻是立法者對重大事項的保護，不能用假決議繞過。

### 考試陷阱

> 假決議 ≠ 無效決議，只是需要再確認。
> 普通決議出席門檻章程可以提高但不能降低（§174但書：「章程有較高之規定者從其規定」）。
> 假決議只適用普通決議場合，不適用特別決議。

## 第 20 組：內線交易 vs 操縱市場 vs 短線交易（證交法）

### 一眼判斷法

```
利用未公開重大消息買賣 → 內線交易（§157-1）：偷跑
用人為手段影響股價     → 操縱市場（§155）：炒作
六個月內反向交易獲利   → 短線交易（§157）：短進短出
```

| 判斷基準 | 內線交易 §157-1 | 操縱市場 §155 | 短線交易 §157 |
|----------|-----------------|---------------|---------------|
| 禁止行為 | 利用未公開重大消息 | 人為操控股價 | 六個月內反向交易 |
| 主觀要件 | 知悉重大消息 | 有操控意圖 | 不問主觀（純客觀） |
| 行為人 | 內部人及消息受領人 | 任何人 | 董監事、經理人、大股東 |
| 刑事責任 | 3-10年 | 3-10年 | 無 |
| 民事效果 | 賠償善意相對人 | 賠償 | 歸入權（公司請求吐出獲利） |

### 混淆原因

三者都是證交法「禁止行為」，內部人可能同時觸犯多項。最易混的是內線交易和短線交易——前者看「消息」，後者看「時間差」。

### 深度解析

**內線交易構成要件（§157-1）：**
1. 行為人為內部人（董監事、經理人、10%大股東）或消息受領人
2. 知悉涉及公司財務業務之重大消息
3. 在消息明確後、公開前（或公開後18小時內）買賣股票

**操縱市場主要類型（§155）：**
- 沖洗買賣（左手賣右手買，製造假交易量）
- 相對委託（跟特定人串通交易）
- 連續買賣（大量操控價格，炒股）
- 散布流言（散布不實消息影響股價）

**短線交易（§157）：**
- 純客觀認定，不問動機、不問是否知悉消息
- 六個月內有差價 → 公司可請求歸入（吐出獲利）
- 只有民事歸入權，沒有刑事責任

### 案例對比

| 情境 | 適用 | 分析 |
|------|------|------|
| 董事長知道即將被併購，提前買股 | §157-1 內線交易 | 利用未公開重大消息 |
| 某大戶連續大量買進拉抬股價 | §155 操縱市場 | 連續買賣操控價格 |
| 董事1月買進，4月賣出獲利500萬 | §157 短線交易 | 六個月內反向操作 |
| 董事知內幕且2月內買賣獲利 | §157-1 + §157 | 同時構成，分別分析 |

### 邊界案例

董事配偶買股 → 如從董事獲悉消息即為「消息受領人」→ 構成內線交易。實務上配偶主張「不知道」通常不被採信。

短線交易不問動機 → 即使董事因急需用錢才賣，六個月內有差價仍要歸入。

### 考試陷阱

> 短線交易沒有刑事責任，只有民事歸入權——別和內線交易搞混。
> 內線交易的時間限制：消息公開後18小時內仍不能交易。
> 操縱市場的行為人不限內部人——任何人都可能構成。
> 同一行為可能同時構成內線交易和短線交易——考題要分別分析。

# 肆、跨組綜合決策樹

## 一、民事訴訟程序選擇總流程

```
民事紛爭發生
├── 需要「暫時保護」？
│   ├── 怕對方脫產（金錢）→ 假扣押
│   ├── 怕現狀改變（特定物）→ 假處分
│   └── 法律關係有爭議 → 定暫時狀態處分
│
├── 選擇「訴之類型」？
│   ├── 要對方做事/給東西 → 給付之訴
│   ├── 只要確認權利存在 → 確認之訴
│   └── 要改變法律關係 → 形成之訴
│
├── 訴訟進行中「改訴」？
│   ├── 換掉原來的請求 → 訴之變更
│   └── 保留原請求再加新的 → 訴之追加
│
├── 多數當事人？
│   ├── 各自獨立 → 普通共同訴訟
│   ├── 法律上必須全部一起 → 固有必要共同訴訟
│   └── 各自可告但結果統一 → 類似必要共同訴訟
│
├── 判決效力？
│   ├── 訴訟標的的拘束力 → 既判力
│   └── 重要爭點的拘束力 → 爭點效
│
└── 不服判決/裁定？
    ├── 判決未確定 → 上訴
    ├── 裁定 → 抗告
    ├── 判決已確定（當事人）→ 再審
    └── 判決已確定（第三人）→ 第三人撤銷訴訟
```

## 二、刑事訴訟強制處分強度階梯

```
強度由低到高：
傳喚 → 拘提 → 逮捕 → 羈押
搜索（找）→ 扣押（拿）
```

## 三、商法責任人交叉表

```
公司法：董事 §23 忠實義務（民事賠償）
證交法：董監事 §171 特別背信（刑事3-10年）
        內部人 §157-1 內線交易（刑事3-10年）
        內部人 §157 短線交易（民事歸入權）
        任何人 §155 操縱市場（刑事3-10年）
票據法：發票人/背書人/承兌人/保證人（各負不同擔保責任）
保險法：要保人(付費+契約當事人)/被保險人(被保障)/受益人(領錢)
```

| 遇到什麼問題 | 怎麼判斷 | 答案 |
|--------------|----------|------|

| 前訴結果拘束後訴 | 拘束的是標的還是爭點？ | 標的→既判力；爭點→爭點效 |

| 多人共同訴訟 | 必須一起嗎？ | 不必→普通；人要齊→固有；結果統一→類似 |
| 不服法院決定 | 判決or裁定？確定否？ | 判決未確定→上訴；裁定→抗告；確定→再審 |

# 陸、答題思考框架

## 遇到混淆概念時的 SOP

```
Step 1：辨識 — 這題在問哪一組混淆概念？
Step 2：分類 — 用 if-else 判斷法快速歸類
Step 3：驗證 — 用構成要件逐一核對
Step 4：排除 — 想一下「不是另一個」的理由是什麼
Step 5：邊界 — 這個案例有沒有踩到邊界？
Step 6：陷阱 — 出題者想騙我什麼？
```

## 工程師考法律的四大優勢

1. **邏輯拆解**：法律概念的區分就是 if-else 條件判斷
2. **系統化思考**：零散法條組織成決策樹和流程圖
3. **邊界測試意識**：直覺地去想 edge case，這正是考點
4. **除錯思維**：逐步驗證答案，不被似是而非的選項騙到

善用這些優勢，把每組混淆概念轉化為你熟悉的技術文件來理解和記憶。
