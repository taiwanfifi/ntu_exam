## 貳、AB 線品質排名（逐演算法詳細分析）

### 第 1 名：DP / Knapsack ★★★★★

**為什麼是 Vol 2 最佳？**

DP / Knapsack 在 Vol 2 中達到了 AB 線融合的最高境界——演算法概念與劇情選擇完全不可分離。讀者在閱讀 ch147-149 時，不是在「學背包問題」然後「看嚴柏翰做選擇」——他們是在看同一件事。

| 維度 | 分析 |
|------|------|
| **A 線純度** | 背包問題的「重量」和「價值」被精確映射為記憶的「神經容量」和「對人格的重要性」。超出極限 34% = 必須放棄。0/1 約束（保留或放棄，不能只保留一半）= 記憶的不可分割性 |
| **B 線深度** | 嚴柏翰逐一秤量十年的破案紀錄、榮譽、同事情誼。手在發抖。最終選擇只保留三樣：友誼（真的東西）、方法論（可帶走的工具）、茉莉花的味道（愛的殘留）|
| **同步品質** | ch149 的「三樣東西」= 背包問題的最優解。但讀者不會想到演算法——他們只會想到「如果是我，我保留什麼」。這正是教學的最高境界：概念內化為人生問題 |
| **跨卷呼應** | Vol 1 ch77 維倫犧牲所有記憶 = 清空背包。Vol 2 ch149 嚴柏翰精選三樣 = 在有限空間中做最優選擇。兩次「選擇」的哲學層級不同：ch77 是「全部放棄」，ch149 是「選擇保留什麼」——後者更成熟 |
| **米亞比喻** | 「你的大腦是一個背包。只能裝這麼多。你必須決定：帶走什麼，留下什麼。」——全卷最佳。簡單、直覺、催淚 |

**結論**：DP / Knapsack 是 Vol 2 的標竿場景，品質與 Vol 1 的 Binary Search（ch35 茉莉花）和 Linked List（ch27-29 循環記憶）並駕齊驅。

### 第 2 名：Union-Find ★★★★★

**為什麼是「最佳演算法即劇情」？**

Union-Find 只用了 2 章（ch131-132），卻完成了全書最大的身份揭露之一。這是因為 Union-Find 的演算法邏輯與劇情揭露的邏輯是同構的——Find 操作「追溯到根」= 偵探追查「追溯到幕後黑手」。

| 維度 | 分析 |
|------|------|
| **A 線純度** | ch131 的歸併操作——把每一個篡改事件、每一條被切斷的邊、每一個幽靈帳號合併到同一集合。ch132 的 Find 操作——找到根。A 線邏輯清晰、步驟明確 |
| **B 線深度** | 所有的陰謀、篡改、斷聯全部指向一個人。W = 韻 = 邱韻如。技術揭露的瞬間 = 情感炸彈爆炸的瞬間 |
| **同步品質** | AB 線不是「同步」——是「同一」。Find(root) = 找到織網者。演算法操作本身就是劇情高潮。這是全書少數「演算法 = 劇情」而非「演算法 + 劇情」的場景 |
| **米亞比喻** | 「如果你追蹤每一條線索到最後，它們全部收束到同一個點——那個點就是根。」——偵探感十足 |

**不足之處**：教學密度偏低。只有 2 章，歸併過程的視覺化不夠充分。建議精修時增加米亞在月台牆上投射「合併動畫」的場景——讓讀者看到集合一個一個合併的過程。

**結論**：Union-Find 是 Vol 2 中「演算法即劇情」設計的極致案例。如果 DP / Knapsack 是「讀者不知道自己在學」，Union-Find 是「讀者學完了才發現自己學了」。

### 第 3 名：Graph / CC ★★★★★

**為什麼是 Vol 2 最核心？**

Graph / CC 不是一個「場景演算法」——它是 Vol 2 的隱喻骨架。從 ch100（三方勢力 = 圖結構）到 ch121（城市碎裂為 123 座島）再到 ch125（碎片重建），Graph 的概念貫穿了角力篇和揭面篇的全部主線。

| 維度 | 分析 |
|------|------|
| **A 線純度** | 7 次出場，覆蓋節點/邊/連通分量/碎裂/重建的完整概念鏈。從政治關係圖到記憶孤島到城市碎裂，抽象層級逐步提高 |
| **B 線深度** | 「人是節點，記憶連結是邊，邊斷了就變孤島」——這不只是比喻，是 Vol 2 的社會寓言。47 個家庭 → 23,000 人 → 123 座島 = 文明的崩潰 |
| **同步品質** | ch104（母親記憶中孩子是孤島）和 ch121（月台投影）是兩個高品質同步點。但因為分布在 7 章中，單次的同步衝擊力不如 DP 或 Union-Find 集中 |
| **米亞比喻** | 「想像新台北是一張巨大的網。每個人是一個點，每段記憶連結是一條線。有人在剪線。」——全卷最佳米亞比喻 |

**不足之處**：出場次數多（7 次），但部分出場是輕觸式的概念引用（如 ch100 白板畫圖），不是深度教學。核心教學集中在 ch104-105 和 ch120-121，其餘章節是鋪排和回收。

**結論**：Graph / CC 的品質不在於單一場景的衝擊力，而在於它作為「全卷隱喻」的一致性和深度。它是 Vol 2 的骨架，如同 Binary Search 是 Vol 1 的靈魂。

### 第 4 名：BFS ★★★★

**三次遞進 = 同一工具的三種人性**

BFS 的設計亮點不在於任何單一場景，而在於三次使用的情感遞進：

| 出場 | 章節 | 情感色彩 | 功能 |
|------|------|---------|------|
| 第一次 | ch90 | 中性（工具使用） | 概念建立：橫向掃描關聯碎片 |
| 第二次 | ch103 | 溫暖 / 心碎（母愛） | 情感高峰：「一個母親在自己的腦海裡找不到自己的孩子」 |
| 第三次 | ch110 | 恐怖（病毒） | 反面應用：病毒以 BFS 模式從圓心向外擴散 |

**設計哲學**：讀者通過三次 BFS 學到的不只是「廣度搜尋」——是「同一個演算法可以救人也可以傷人，工具是中性的，意圖決定一切」。這是 Vol 2 在教學哲學上最精妙的設計之一。

**ch103 品質**：有潛力成為 Vol 2 前半的催淚場景。「漣漪」意象（每擴一圈 = 更多的空白 = 更大的心碎）是視覺化教學與情感渲染的完美結合。

**不足**：ch90 的首次出場偏功能性，缺乏情感鉤子。建議精修時在 ch90 加入一個微小的情感細節（例如維倫在橫向掃描時意外碰到議員年輕時的善意記憶），讓第一次 BFS 不只是工具展示。

### 第 5 名：Tree / BST ★★★★

**「根被砍但枝葉還在搖晃」——最詩意的意象**

Tree 是 Vol 2 第一個正式登場的新演算法，承擔了建立「深潛空間」視覺語言的重任。

| 維度 | 分析 |
|------|------|
| **意象品質** | 「記憶之樹倒生結構」= 直觀、美麗、易記。根在最深處（身份/最深的愛），葉在表層（感官碎片）。這個意象在後續所有深潛場景中持續使用 |
| **鋼琴師案** | ch93-94 是完整的教學場景。「技能記憶之樹被從根部砍斷，身體記得怎麼彈但大腦空白」= 把 Tree 的根/葉概念與「身體記得」母題結合 |
| **同步品質** | ch94 維倫找到被切斷的根節點 = 「為什麼開始彈琴」的記憶。修復根 = 找到起源。概念清晰，情感真摯 |

**不足**：米亞的 Tree 比喻（「一棵樹被砍了，但根還在地下」）偏直白，不如 BFS 的「漣漪」或 DP 的「背包」有衝擊力。建議精修時升級為更有層次的比喻，例如將「倒生巨樹」與「潛水越深光越少」的感官體驗結合。

### 第 6 名：Backtracking ★★★★

**兩案跨接 = 概念從恐怖到心碎的進化**

Backtracking 的獨特之處在於它跨越了兩個委託案（退役探員案 ch115-117 + 獵犬記憶案 ch150-151），在兩種完全不同的情感基調中展現同一概念：

| 使用 | 章節 | 基調 | 教學重點 |
|------|------|------|---------|
| 第一次 | ch115-117 | 恐怖（動態迷宮） | Backtracking 基礎：嘗試所有路徑，碰壁回退。ch116 的「會動的牆」= 動態環境的回溯 |
| 第二次 | ch150-151 | 心碎（記憶恢復） | Backtracking 進階：在多層覆蓋記憶中逐層剝離。ch151 回到「研究生」= 回到起點 |

**ch151 的同步點**：嚴柏翰看到 24 歲的自己不是探員而是研究生——Backtracking「回到起點」= 他回到了真正的自己。這是非常好的同步設計：演算法的「回退」= 人生的「回歸」。

**與 Vol 1 Stack 的呼應**：Vol 1 ch19-20 的 Stack 是「逐層剝開」（pop），Vol 2 的 Backtracking 是「系統性地逐層嘗試與回退」。從「剝開」到「探索」，概念升級自然。

### 第 7 名：TopSort ★★★★

**邏輯清晰，情感耦合度中等**

TopSort 的 A 線設計非常出色——用「依賴關係排序」還原篡改的執行順序，讀者跟著維倫一步一步拼出「死亡紀錄 → 遊行紀錄 → 目擊者記憶 → 新聞報導 → 全城市民」的完整鏈條。ch127 揭示「最先被篡改的是周明哲的死亡紀錄」= 脊背發涼。

| 維度 | 分析 |
|------|------|
| **A 線純度** | 優秀。依賴關係的概念非常具體：「催淚彈在衝突升級之後」= 時間順序 = 拓撲排序 |
| **B 線深度** | 中等。ch127 的「一切起點是偽造死亡」有衝擊力，但 TopSort 本身偏邏輯操作，情感場景不如 DP 或 BFS 催淚 |
| **ch142 回收** | 鸚鵡計畫文件中的完整依賴鏈 = TopSort 的第二次應用。冰冷恐怖，但不催淚 |

**不足**：米亞的比喻（「先打地基再蓋房子」）太日常化，缺乏記憶駭客世界的獨特感。建議精修時改為與篡改歷史相關的比喻：「要讓一座城市遺忘，你不能從屋頂開始拆——你得先拔掉地基。」

**品質調整說明**：在藍圖文件中 TopSort 被預估為 ★★★，但實際細綱中 ch127 的「死亡紀錄是起點」揭露具有相當的敘事衝擊力，ch142 的鸚鵡計畫回收也很紮實。上調至 ★★★★。

### 第 8 名：DFS ★★★

**恐怖感出色，教學密度不足**

DFS 的問題不在於品質差，而在於它的概念太直觀——「一直走到走不通，走不通就退一步」——讀者在 ch86 就理解了，之後沒有太多深化空間。

| 維度 | 分析 |
|------|------|
| **A 線純度** | 清晰但簡單。「在記憶迷宮中不斷向更深層推進」= DFS 的字面意思。回溯（ch87）是唯一的進階概念 |
| **B 線深度** | 恐怖感很好。ch87 的防禦體（由恐懼凝聚的巨大眼球）是優秀的視覺意象。但恐怖 > 情感，缺少催淚的同步點 |
| **同步品質** | ch87「退一步不是失敗——是記住了一條不通的路」= 有哲學層次，但比喻感偏強，情感耦合度偏弱 |
| **ch115 變體** | 動態迷宮中的 DFS 是有趣的變體，但此時 DFS 已經被 Backtracking 覆蓋，概念邊界模糊 |

**核心問題**：DFS 和 Backtracking 的概念邊界在 ch115 處不夠清晰。ch115 同時標註了 DFS 和 Backtracking——讀者可能不清楚兩者的區別。建議精修時在 ch115 由米亞明確區分：「DFS 是走迷宮的策略，Backtracking 是走迷宮時記住每一步的策略。DFS 只管走，Backtracking 還知道哪裡不能走。」

**為什麼相對較低？** 不是因為差，而是因為其他 8 個演算法的 AB 線融合太好了。DFS 的場景（恐怖迷宮）在 Vol 2 的整體格局中更像是「氣氛營造」而非「教學+情感」的雙重突破。

### 第 9 名：Dijkstra ★★★

**動作場景的節奏 > 教學深度**

Dijkstra 在 Vol 2 中被定位為「動作場景的演算法」——潛入記憶銀行的七層安全防護。這個定位決定了它的教學深度有限：讀者跟著維倫「像幽靈一樣穿過數位防線」，興奮有餘，理解不足。

| 維度 | 分析 |
|------|------|
| **A 線純度** | 中等。「加權圖 + 最短路徑」的概念在 ch137 建模、ch138 執行、ch139 即時調整中有完整展現。但「權重 = 被偵測概率」的設定比較功能性，缺乏情感維度 |
| **B 線深度** | 動作場景。節奏快、張力高，但情感層較薄。唯一的哲學閃光是 ch139：「最安全的路徑不一定通向最重要的真相」|
| **同步品質** | ch139 的意外——觸發十年前遊行完整錄像——是一個好的劇情轉折。但同步的力度更多來自劇情（遊行錄像的震撼）而非演算法概念本身 |
| **米亞比喻** | 「每條路都有代價。Dijkstra 找的不是最短的路——是代價最低的路。但有時候，最重要的東西偏偏在代價最高的那條路上。」——簡潔有力 |

**核心問題**：Dijkstra 的「加權」概念沒有被充分情感化。建議精修時在 ch137 加入情感權重的暗示——例如某些路徑的「權重」不只是被偵測概率，還包含「經過這條路會碰到維倫自己的記憶碎片」的心理代價。這樣能讓 Dijkstra 從「純工具」提升到「工具 + 選擇」的層次。

**為什麼相對較低？** 與 DFS 相同——不是差，而是定位限制了深度。Dijkstra 被分配給「動作場景」（潛入任務），天然地偏向節奏感而非教學感。如果要提升，需要在 ch137-139 中加入更多「每一步選擇都有情感代價」的細節。

## 肆、Vol 1 → Vol 2 教學風格銜接

### 一、從線性到非線性到最佳化的難度遞進

```
Vol 1 讀者的演算法理解曲線：
  ch1-2:   「排序。把亂的東西排好。」            → 線性思維
  ch8-9:   「窗口掃描。在流動中找模式。」        → 掃描思維
  ch14-15:  「比對。兩個東西是否指向同一人。」   → 比對思維
  ch19-20:  「層層剝開。最後一層可能是空白。」   → 層次思維
  ch27-29:  「偵測循環。你可能在重複的路。」     → 非線性思維萌芽
  ch33-35:  「對折搜尋。不需要從頭找。」         → 效率思維

Vol 2 讀者的演算法理解曲線：
  ch85:     「記憶像一棵樹。有根有葉。」         → 結構思維

  ch90:     「不深入。展開。像漣漪。」            → 廣度搜索思維
  ch100:    「三方勢力。點和線。」                → 圖論思維
  ch109:    「碎裂的網。孤島。」                  → 連通性思維
  ch126:    「順序。先做什麼後做什麼。」           → 依賴關係思維
  ch131:    「所有線索歸併到同一人。」             → 集合溯源思維
  ch137:    「代價最低的路。」                     → 最優路徑思維
  ch147:    「背包只有這麼大。選擇帶走什麼。」    → 最優化決策思維
  ch150:    「走錯了退回來。知道了哪條不通。」     → 系統探索思維
```

**遞進邏輯**：

| 階段 | Vol 1 | Vol 2 | 升級方向 |
|------|-------|-------|---------|
| 基礎 | Array（排列） | Tree（結構化） | 從一維到多維 |
| 搜索 | Sliding Window（線性掃描） | DFS / BFS（深度 / 廣度） | 從單向到多向 |
| 比對 | HashMap（一對一比對） | Graph（多對多關係） | 從配對到網絡 |
| 層次 | Stack（剝開） | Backtracking（系統性剝開） | 從被動到主動 |
| 結構 | Linked List（環形） | Connected Components（碎裂） | 從單一結構到全局結構 |
| 效率 | Binary Search（搜索效率） | Dijkstra（路徑效率） | 從線性到加權 |
| 綜合 | 綜合運用（組合） | DP（最優化決策） | 從運用到決策 |

**品質評估**：遞進邏輯自然、清晰。讀者從 Vol 1 的「線性」世界過渡到 Vol 2 的「網絡」世界時，不會感到跳躍——因為 Vol 1 的圖論萌芽（ch21 有向圖 / ch30 匯聚 / ch56 分散式）已經為 Vol 2 的 Graph 系列做了鋪墊。

### 三、米亞比喻風格的演變

| Vol 1 | Vol 2 | 演變方向 |
|-------|-------|---------|
| 「河流 + 漁網」（Sliding Window） | 「撒網。漣漪。」（BFS） | 水的意象延續 |
| 「跑道上的兩個人」（Floyd 判圈法） | 「走迷宮的規則」（DFS / BT） | 從跑道到迷宮：空間升級 |
| 「對折再對折」（Binary Search） | 「代價最低的路」（Dijkstra） | 從重複操作到路徑選擇 |
| 「指紋 + 手印」（HashMap） | 「人是點，記憶是線」（Graph） | 從配對到網絡 |

**品質評估**：米亞的比喻風格從 Vol 1 的「物理類比」（河流、跑道、對折）進化到 Vol 2 的「人生類比」（背包 = 人生的取捨、漣漪 = 母愛的擴散方式）。這與米亞自身的成長弧線一致——她在 Vol 2 更加「理解人類」，所以比喻也更接近人類的情感經驗。

**待加強**：Tree（「根在最深處，葉子在外面」）和 TopSort（「先打地基再蓋房子」）的比喻相對平淡，缺乏米亞在 Vol 2 應有的「成長後的洞察力」。建議精修時升級。

## 陸、教學改進建議

### 一、哪些演算法的 AB 線可以更好？

#### DFS（★★★ → 潛力 ★★★★）

**現狀**：恐怖感出色，但情感同步點偏弱。「退一步不是失敗」是好的哲學，但缺少催淚的個人故事。

**改進方案**：在 ch87 的回溯場景中，加入維倫在退回分岔點時意外碰到的一段記憶——不是議員的，是維倫自己的（從深潛後遺症滲入的碎片）。他看到一個模糊的畫面：語青的笑臉。然後碎片消失。他不知道那是誰。但他的手心發燙了一下。

**效果**：DFS 的「回溯」= 維倫在找語青的記憶中「退回又碰到」。教學概念不變，但增加了情感鉤子。

#### Dijkstra（★★★ → 潛力 ★★★★）

**現狀**：純動作場景。「加權」概念缺乏情感維度。

**改進方案**：在 ch137 的建模階段，讓某些路徑的「權重」不只是被偵測概率，還包含「路徑上殘留的記憶碎片」帶來的心理干擾。例如第三層有一段議員案的殘留記憶（ch92 的外來碎片效應），維倫經過時會短暫失去方向感。米亞必須把這個「心理干擾」也計入權重。

**效果**：Dijkstra 的「加權」不再只是數學概念，而是「每條路都有情感代價」。

#### TopSort（★★★★，米亞比喻可升級）

**現狀**：A 線邏輯清晰，但米亞的「先打地基再蓋房子」太日常。

**改進方案**：「要讓一座城市遺忘一場遊行，你不能從觀眾的記憶開始——你得先刪掉那場遊行存在過的證據。殺死領袖，然後刪除遊行，然後清除目擊者，最後覆蓋全城。每一步都依賴前一步。」

**效果**：米亞用 Vol 2 的具體劇情解釋 TopSort，而不是用建築比喻。更沉浸、更有衝擊力。

### 二、哪些章節的教學太重？

| 章節 | 問題 | 建議 |
|------|------|------|
| ch85-90 | 6 章引入 4 個新演算法（Tree, DFS, BST, BFS） | 考慮將 ch85 的 Tree 和 ch88 的 BST 更緊密地整合（同一章的兩個階段），或在 ch89 之後再引入 BFS |
| ch126 | 全篇教學密度最高的章節（TopSort 完整概念 + 碎片依賴排序） | 用偵探追查的緊張感包裹教學。確保米亞的比喻在此章節足夠有力——這是讀者在此段唯一的「翻譯器」|
| ch131-132 | Union-Find 歸併 + Find 操作 + 邱韻如揭露同時發生 | 教學不重——但讀者的注意力可能完全被揭露吸引而忽略 UF 的教學。建議在 ch131 加入更多「歸併過程」的視覺化描寫（米亞在月台牆上投射合併動畫）|

### 三、整體教學節奏的優化方向

**現狀節奏**：
```
深潛篇：密集啟動 → 鋼琴師案教學 → 尾部輕觸
角力篇：母親案教學 → 政治谷（零演算法）→ 迷宮案教學 → 篇末核彈
揭面篇：追查教學 → 揭露高潮（UF = 劇情）→ 動作段（Dijkstra）
碎裂篇：情感段 → DP 核心教學 → BT 恢復 → 情感核彈（零演算法）
```

**優化方向**：

1. **深潛篇的「演算法洪水」問題**：ch85-90 的密集程度需要在精修時通過感官寫作來緩衝——讓讀者沉浸在深潛的恐怖/奇幻體驗中，演算法概念隱藏在體驗的後面。

2. **角力篇的「零演算法長谷」問題**：ch107-114 的 8 章零演算法在節奏上是正確的，但需要確保這 8 章的政治燒腦足夠精彩，否則讀者可能覺得「演算法小說突然變成了政治小說」。建議在 ch109（語青的地圖）加強 Graph/CC 的視覺化，讓它成為政治段中的「技術錨點」。

3. **揭面篇的「教學 + 高潮」雙峰問題**：ch126-132 同時承擔教學和揭露。目前的設計讓教學服務於懸念推進（TopSort = 追查順序，UF = 找到幕後黑手），這是正確的。但精修時需要確保讀者在情感高潮中仍然「學到了」——建議在 ch132 揭露後加入米亞的一句總結：「所有的路。同一個根。」讓讀者在情感衝擊後有一個概念錨點。

4. **碎裂篇的「DP 孤峰」問題**：ch147-149 的 DP 教學是全卷最佳，但它是碎裂篇唯一的教學段——前後都是情感場景。建議在 ch155（DP 變體：碎片重建序列）加強教學成分，讓 DP 在碎裂篇有「兩個教學點」而非「一個教學點 + 一次輕觸」。

## 捌、寫作檢查清單（10 條）

在 Vol 2 每個演算法教學場景的精修中，逐項確認以下 10 條標準：

### AB 線融合類

- [ ] **1. AB 線同步點是否明確？** 技術突破的瞬間 = 情感釋放的瞬間。如果兩者不同步，找到原因並修復。Vol 2 的標竿：ch132（Union-Find 的 Find = 織網者揭露）和 ch149（背包最優解 = 「三樣東西」）。

- [ ] **2. 讀者是否「不覺得在學」？** 在精修後朗讀每個教學段落。如果聽起來像教科書，重寫。演算法概念必須隱藏在場景、對話、行動中——不是米亞的獨白。

- [ ] **3. 演算法概念是否可以被移除？** 如果把演算法從場景中拿掉，劇情是否依然成立？如果是——融合不夠深。好的融合 = 拿掉演算法，劇情就塌了。例：ch149 拿掉背包問題，嚴柏翰的選擇就失去了「必須選擇」的物理約束。

### 教學品質類

- [ ] **4. 米亞的比喻是否足夠有力？** 每個演算法都有一句米亞的「翻譯」。這句話必須（a）非工程讀者聽得懂、（b）有情感共鳴、（c）在全卷中不重複。目前待加強：Tree 和 TopSort。

- [ ] **5. 概念是否有至少兩次出場？** 一次引入，一次深化或變體。避免「教了就忘」。目前所有 9 個演算法都至少出現 2 次。但 Union-Find 只有 2 次，建議在精修時考慮是否需要在碎裂篇加入一次輕觸回收。

- [ ] **6. 教學密度是否與情感段平衡？** 連續 3 章以上的高密度教學 = 認知過載風險。目前 ch85-90 是唯一的紅區。

### 銜接類

- [ ] **7. 與 Vol 1 對應演算法的升級是否清晰？** 每個 Vol 2 演算法都應該讓讀者感到「這比 Vol 1 的東西更複雜，但我能跟上」。銜接表（參照「壹」總覽表）中的每一對升級關係都需要在文本中有一個「提示點」——例如維倫或米亞說一句「這比以前的掃描更複雜」。

- [ ] **8. Vol 3 的演算法種子是否已種下？** Greedy、Heap、Trie 三個 Vol 3 新概念需要在 Vol 2 的精修中植入微小的伏筆。不需要教學——只需要一句話的暗示。

### 風格類

- [ ] **9. 恐怖場景是否有教學功能？** Vol 2 的深潛篇和退役探員案有大量恐怖場景（防禦體、會動的牆）。每個恐怖場景必須同時承載教學功能——恐怖不是目的，「用恐怖驅動好奇心」才是。確認每個恐怖意象都對應一個演算法概念。

- [ ] **10. 「身體記得」母題是否在教學場景中出現？** Vol 2 的核心母題是「身體記得」（維倫手心發燙 / 嚴柏翰的手抖 / 阿嬤的豆漿）。至少 3 個演算法教學場景中應該有角色的「身體先於大腦做出反應」——這與「讀者直覺學會演算法」是同構的。目前 ch149（嚴柏翰的手在發抖）和 ch94（鋼琴師身體記得怎麼彈）已達標。建議在 ch103（BFS 母親案）加入一處：母親的手在摸到孩子照片時微微收緊——邊斷了，但身體還記得。
