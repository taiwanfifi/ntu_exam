# 輕小說寫作原則 — 核心法則

> 從日本輕小說產業、角川動漫教育課程、以及多年來的評論分析中提煉的原則。

---

## 一、輕小說 vs 一般小說的核心差異

| 維度 | 一般小說 | 輕小說 |
|------|----------|--------|
| 文字風格 | 可以複雜、文學性高 | **淺白、清爽、讀起來快** |
| 角色設定 | 複雜、灰色地帶 | **鮮明、一眼就記住** |
| 敘事節奏 | 可以慢燉 | **快節奏、短章節** |
| 目標讀者 | 廣泛 | **青少年~30 歲** |
| 視覺感 | 可有可無 | **如漫畫分鏡般的場景描寫** |
| 題材限制 | 有文學傳統包袱 | **完全自由，什麼題材都可以** |

## 二、五大核心原則

### 原則 1：有趣至上

```
輕小說的第一使命是「有趣」。
不是「有教育意義」、不是「文筆優美」、不是「主題深刻」。
是「有趣」。

如果讀者不覺得有趣，其他一切都不存在。
因為他已經關掉了。
```

**對演算法小說的意義**：
> 教學是隱藏目標。表面目標永遠是「好看」。
> 讀者是來看故事的，不是來上課的。
> 他學到東西是副作用，不是主菜。

---

### 原則 2：角色鮮明度 > 一切

```
輕小說的角色必須在 3 秒內被記住。

好的角色設定：
✓ 「ICPC 金牌學姊，說話像寫程式，零冗餘」 ← 一句話就記住了
✓ 「信奉迴圈能解決一切的暴力美學家」 ← 一句話就記住了
✓ 「平時靠直覺活著，偶爾靠邏輯」 ← 一句話就記住了

壞的角色設定：
✗ 「一個很厲害的工程師」 ← 記不住
✗ 「一個聰明的學生」 ← 記不住
```

**對演算法小說的意義**：
> 我們目前的角色設定已經很好了。
> 繼續強化每個角色的「演算法性格」：
> - 學姊 = O(1) 的人（直接算出答案，不搜尋）
> - 阿翔 = O(n²) 的人（暴力但真誠）
> - 林凱 = O(n log n) 的人（在學姊和阿翔之間找到平衡）

---

### 原則 3：漫畫式分鏡寫作

```
寫輕小說 = 用文字導演一部動畫。

每個場景都應該能被畫成漫畫分鏡：
- 清晰的空間感（誰在哪裡、面對誰）
- 動態的動作描寫（不是「她很快地打字」，而是「手指在鍵盤上的速度讓我想起我媽打麻將」）
- 表情和肢體語言（「那個眼神的溫度大概跟新竹的風差不多」）
```

**對演算法小說的意義**：
> 我們已經在做了！「學姊看了他一眼。就一眼。」這就是漫畫分鏡。
> ASCII art 圖表 = 漫畫中的信息面板。繼續用。

---

### 原則 4：「鉤子」密度

```
每 2-3 頁至少要有一個「鉤子」（hook）。

鉤子的類型：
├── 懸念鉤：「然後背脊發涼。」
├── 幽默鉤：「你不懂，我這學期全靠加分過的。」
├── 知識鉤：「你知道嗎？O(1) 是有代價的。」
├── 情感鉤：「今天你的推理——不錯。」
└── 翻轉鉤：「這些學號不是隨機的。」

如果連續 3 頁沒有鉤子，讀者就會開始滑手機。
```

**對演算法小說的意義**：
> 檢查每一段：有沒有鉤子？
> 特別是「知識鉤」——把「要教的概念」包裝成「讓人好奇的問題」。
> 不是「現在我們來學 Binary Search」，
> 而是「八萬六千行日誌，你要怎麼在十七步之內找到那條分界線？」

---

### 原則 5：對話驅動

```
輕小說的主要敘事載體是對話，不是描述。

好的比例：
對話 : 描述 : 內心獨白 = 5 : 3 : 2

對話的功能：
├── 推進劇情
├── 展現角色性格
├── 傳遞知識（教學！）
└── 製造幽默/張力

壞的做法：
「Binary Search 的原理是......」（教科書式描述）

好的做法：
「如果我們不從第一行開始，而是——直接看中間那行呢？」
「看中間？為什麼？」
「想像日誌是一本書。前半本是白色的頁面，後半本是黑色的......」
```

**對演算法小說的意義**：
> 我們已經掌握了這個原則！「演算法偵探筆記」中的知識傳遞幾乎全部透過對話。
> 繼續保持。永遠不要退回教科書模式。

## 三、節奏控制法則

### 起承轉合（日本敘事結構）

```
起（き）：問題登場。「系計中被入侵了。」
承（しょう）：調查展開。用已知工具嘗試。
轉（てん）：意外發現。「這些學號不是隨機的。」
合（けつ）：收束總結。偵探手札。

每個「演算法單元」都應該走完一輪起承轉合。
```

### 序破急（進階節奏）

```
序（じょ）：緩慢展開，鋪墊世界觀。（第一夜：前 1-2 幕）
破（は）：突破平衡，衝突加劇。（第一夜：第 3-5 幕）
急（きゅう）：快速收束，高潮迭起。（第一夜：第 6 幕 + 手札）

整個系列也可以用序破急：
序 = 前 5 章：基礎概念（Binary Search, Hash Map, Two Pointers）
破 = 中間 10 章：進階概念（Graph, Tree, DP）+ 劇情複雜化
急 = 最後 5 章：終極挑戰（Google 面試等級）+ 揭開真相
```

### 張弛有度

```
不能一直緊繃。人的注意力需要呼吸。

緊：案件推理、演算法推導
鬆：阿翔的笑話、角色日常、吃飯場景

理想的節奏：
緊-緊-鬆-緊-鬆-緊-緊-緊-鬆-大爆發
```

## 四、懸疑小說的特殊技法

### 來自日本推理大師的核心技法

| 技法 | 說明 | 應用到演算法小說 |
|------|------|------------------|
| **封閉空間** | 隔絕外部幫助 | 伺服器室、斷網的電腦 |
| **公平線索** | 解謎資訊已全給讀者 | 每個演算法的前提條件都在文中 |
| **不可靠敘事者** | 敘事者的視角可能有偏差 | 阿翔以為暴力解是對的 |
| **倒敘** | 先看結果再找原因 | 先看到最優解的效果，再回推原理 |
| **紅鯡魚** | 假線索 | 看似 O(n) 的解法其實有 edge case |
| **時鐘炸彈** | 倒數計時的壓力 | 伺服器要在天亮前恢復 |
