# 方案二：機率預言師（The Probability Oracle）

> *「我不能預測未來。但我能告訴你每一種未來的機率。」*

---

## 一句話概念

近未來台灣，腦晶片讓人能「看見」事件的機率分布。
當主角發現有人在竄改這些分布時，一場統計學的革命開始了。

---

## 故事設定

### 世界觀
2045 年，台灣。一家叫 Oracular 的公司開發了「機率透鏡」——
一種植入視覺皮層的腦晶片，能讓使用者在日常生活中看到機率。

- 看著天空 → 腦中浮現「降雨機率 73%，分布：Beta(7.3, 2.7)」
- 看著一個人 → 「此人說謊的後驗機率：P(lie|evidence) = 0.82」
- 走在路上 → 「右轉遇到車禍的條件機率比左轉高 3.7 倍」

這個技術改變了一切。保險、醫療、司法、愛情——
所有人都在用機率做決定。台灣變成全球最「理性」的社會。

但有一天，主角發現她的透鏡算出來的數字，跟現實不一樣了。

### 主角
**陳貝葉**（名字致敬 Bayes），28 歲，Oracular 的機率校準工程師。
工作是確保透鏡的計算正確。
某天她注意到：某個地區的犯罪率分布，不符合 Poisson——
太整齊了。像是被人手動調過的。

### 核心張力
有人在操控「機率透鏡」的底層模型。
他們不是在改數據——他們在改 **先驗分布（prior）**。
當所有人的認知都被扭曲的先驗引導，社會的決策就會往特定方向偏移。

貝葉必須用真正的統計推論，揭穿這場騙局。

---

## 章節結構（預計 55-65 章）

### 第一篇：校準（Ch 1-12）—— 機率的基礎

| 章 | 劇情 | 機率概念 |

| 章 | 劇情 | 機率概念 |

### 尾聲（Ch 61-65）
貝葉面臨選擇：摧毀透鏡系統（讓人回到「不確定」的世界），
還是修正它（用正確的先驗取代錯誤的）。

---

## 為什麼機率在這裡很自然？

整個世界觀就是建立在機率之上的。角色的日常生活就是機率計算。
「分布被竄改」是推動劇情的核心懸念，不是硬塞的教學。

- 透鏡看到的畫面 = 分布的視覺化（讀者自然學會辨認分布）
- 找出異常 = 假設檢定（有劇情動機）
- 先驗被改 = 貝氏定理的核心（全書主題）
- 法庭攻防 = p-value 和 Type I/II error 的戲劇化

---

## 調性與風格

- **氣氛**：近未來科幻、乾淨的城市、透明的數據、暗處的操控
- **對標**：心靈判官（Psycho-Pass）× 攻殼機動隊 × 統計學
- **節奏**：懸疑推理節奏。每一篇揭開一層真相。
- **互動**：「透鏡視角」的 ASCII 框，顯示機率計算過程

---

## 範例場景

> 我走在信義區的人行道上。透鏡自動啟動。
>
> ```
> ┌─ 機率透鏡 v4.2 ─────────────────────────────┐
> │ 前方十字路口                                  │
> │ 下 30 秒內有車輛通過的機率：                  │
> │                                               │
> │ P(車輛到達) ~ Poisson(λ = 2.3/min)           │
> │ P(至少一輛在 30 秒內) = 1 - e^(-2.3×0.5)     │
> │                       = 1 - e^(-1.15)         │
> │                       = 1 - 0.3166            │
> │                       = 0.6834                │
> │                                               │
> │ 建議：等待 █████████░ 68%                     │
> └───────────────────────────────────────────────┘
> ```
>
> 正常的數字。
>
> 然後我把目光轉向那棟大樓——Oracular 的總部。
>
> 透鏡閃了一下。
>
> ```
> ┌─ 機率透鏡 v4.2 ─────────────────────────────┐
> │ 目標建築：Oracular HQ                        │
> │ 未來 24 小時內發生重大資安事件的機率：        │
> │                                               │
> │ P(事件) = 0.0001                              │
> │                                               │
> │ 建議：無需擔憂 ░░░░░░░░░░ 0.01%              │
> └───────────────────────────────────────────────┘
> ```
>
> 0.01%。幾乎是零。
>
> 但我知道這個數字是假的。
>
> 因為三天前，我親手算過同一棟大樓的 Poisson 到達率。
> 根據過去六個月的資安事件紀錄，真實的 λ 應該是 0.3/月。
>
> P(至少一次事件在 24 小時內) = 1 - e^(-0.3/30) ≈ 0.01 = 1%
>
> 1%，不是 0.01%。差了一百倍。
>
> 有人把先驗壓低了。
